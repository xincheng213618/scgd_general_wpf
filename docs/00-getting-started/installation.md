# 安装指南

本页只保留用户真正需要的安装步骤与安装后验证方法。安装器内部实现、打包脚本和 Advanced Installer 配置，已经转移到开发与部署文档中。

## 安装前确认

开始之前，先阅读 [系统要求](./prerequisites.md)，重点确认以下几项：

- 当前环境是 Windows 10 1903+ 或 Windows 11
- 使用 x64 环境
- 拥有安装目录的写权限；若涉及服务配置，建议使用管理员权限
- 如果需要在线更新或访问远程服务，网络环境可用

## 方式一：通过安装包安装

这是面向终端用户的推荐方式。

### 步骤 1：获取安装包

- 从团队分发渠道、发布页面或内部交付包获取 ColorVision 安装程序
- 若同时提供主安装包和增量更新包，首次安装请优先使用完整安装包

### 步骤 2：启动安装程序

- 双击安装程序启动安装向导
- 若系统弹出权限确认，按需允许安装继续执行
- 如果安装器提示缺少先决条件，请先按提示完成安装

### 步骤 3：选择安装位置与组件

- 按安装向导选择目标目录
- 根据需要决定是否创建桌面快捷方式、开始菜单入口等
- 如果交付包中包含插件或项目资源，保持默认安装选项通常更稳妥

### 步骤 4：完成安装

- 等待文件复制和初始化完成
- 完成后可直接勾选启动主程序
- 若环境存在旧版本，安装器可能提示升级、覆盖或卸载旧版本后继续

## 方式二：从源码构建并运行

这是面向开发者和调试场景的路径。

### 最小步骤

```powershell
dotnet restore
dotnet build .\ColorVision\ColorVision.csproj -p:Platform=x64
dotnet run --project .\ColorVision\ColorVision.csproj -p:Platform=x64
```

### 说明

- 当前仓库以 Windows WPF 和 x64 为主，不建议把主程序当作跨平台项目处理
- 如果只想验证构建是否成功，可以先执行 `dotnet build`，再进入运行与调试
- 若输出目录被正在运行的程序占用，先关闭旧进程，或改用临时输出目录验证构建

## 安装完成后的最小验证

无论是安装包安装还是源码运行，建议至少做以下检查：

1. 主程序可以正常启动并显示主窗口
2. 日志目录或日志窗口中没有明显的启动级错误
3. 插件扫描完成后，帮助或插件相关菜单能够正常打开
4. 可以打开一张图片，确认主界面基础交互正常

如果这几步都正常，说明安装链路基本可用，下一步看 [首次运行指南](./first-steps.md) 或 [快速上手](./quick-start.md)。

## 常见安装问题

### 安装程序无法启动

- 先确认安装包未损坏，必要时重新获取安装包
- 检查当前系统版本是否满足 [系统要求](./prerequisites.md)
- 如果被系统策略拦截，尝试使用具有足够权限的账户重新执行

### 安装完成但程序启动失败

- 先查看日志或错误提示
- 确认安装目录和用户文档目录具有读写权限
- 若是源码运行，优先检查输出目录是否缺少运行时依赖文件

### 升级后行为异常

- 优先确认是否存在旧版本残留配置或文件占用
- 必要时先关闭主程序与相关后台服务，再重新安装或重新启动
