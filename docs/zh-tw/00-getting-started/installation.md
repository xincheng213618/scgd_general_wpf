# 安裝指南

本頁只保留使用者真正需要的安裝步驟與安裝後驗證方法。安裝器內部實現、打包指令碼和 Advanced Installer 配置，已經轉移到開發與部署文件中。

## 安裝前確認

開始之前，先閱讀 [系統要求](./prerequisites.md)，重點確認以下幾項：

- 當前環境是 Windows 10 1903+ 或 Windows 11
- 使用 x64 環境
- 擁有安裝目錄的寫權限；若涉及服務配置，建議使用管理員權限
- 如果需要線上更新或訪問遠端服務，網路環境可用

## 方式一：透過安裝包安裝

這是面向終端使用者的推薦方式。

### 步驟 1：獲取安裝包

- 從團隊分發渠道、釋出頁面或內部交付包獲取 ColorVision 安裝程式
- 若同時提供主安裝包和增量更新包，首次安裝請優先使用完整安裝包

### 步驟 2：啟動安裝程式

- 雙擊安裝程式啟動安裝嚮導
- 若系統彈出權限確認，按需允許安裝繼續執行
- 如果安裝器提示缺少先決條件，請先按提示完成安裝

### 步驟 3：選擇安裝位置與元件

- 按安裝嚮導選擇目標目錄
- 根據需要決定是否建立桌面快捷方式、開始選單入口等
- 如果交付包中包含外掛或專案資源，保持預設安裝選項通常更穩妥

### 步驟 4：完成安裝

- 等待檔案複製和初始化完成
- 完成後可直接勾選啟動主程式
- 若環境存在舊版本，安裝器可能提示升級、覆蓋或解除安裝舊版本後繼續

## 方式二：從原始碼建置並執行

這是面向開發者和除錯場景的路徑。

### 最小步驟

```powershell
dotnet restore
dotnet build .\ColorVision\ColorVision.csproj -p:Platform=x64
dotnet run --project .\ColorVision\ColorVision.csproj -p:Platform=x64
```

### 說明

- 當前倉庫以 Windows WPF 和 x64 為主，不建議把主程式當作跨平台專案處理
- 如果只想驗證建置是否成功，可以先執行 `dotnet build`，再進入執行與除錯
- 若輸出目錄被正在執行的程式佔用，先關閉舊程序，或改用臨時輸出目錄驗證建置

## 安裝完成後的最小驗證

無論是安裝包安裝還是原始碼執行，建議至少做以下檢查：

1. 主程式可以正常啟動並顯示主視窗
2. 日誌目錄或日誌視窗中沒有明顯的啟動級錯誤
3. 外掛掃描完成後，幫助或外掛相關選單能夠正常開啟
4. 可以開啟一張圖片，確認主介面基礎互動正常

如果這幾步都正常，說明安裝鏈路基本可用，可以繼續閱讀 [首次執行指南](./first-steps.md) 和 [快速上手](./quick-start.md)。

## 常見安裝問題

### 安裝程式無法啟動

- 先確認安裝包未損壞，必要時重新獲取安裝包
- 檢查當前系統版本是否滿足 [系統要求](./prerequisites.md)
- 如果被系統策略攔截，嘗試使用具有足夠權限的帳戶重新執行

### 安裝完成但程式啟動失敗

- 先檢視日誌或錯誤提示
- 確認安裝目錄和使用者文件目錄具有讀寫權限
- 若是原始碼執行，優先檢查輸出目錄是否缺少執行時依賴檔案

### 升級後行為異常

- 優先確認是否存在舊版本殘留配置或檔案佔用
- 必要時先關閉主程式與相關後臺服務，再重新安裝或重新啟動
