# 系統要求

本頁只說明安裝和執行 ColorVision 前需要準備的環境，不再展開安裝器實現、倉庫結構或架構設計。

## 適用物件

- 想安裝現成程式包並開始使用的終端使用者
- 想從原始碼建置並執行主程式的開發者
- 想先確認系統條件是否滿足，再繼續閱讀安裝步驟的同事

## 執行環境要求

### 作業系統

- Windows 10 1903 及以上版本，或 Windows 11
- 建議使用 x64 系統環境
- 首次安裝、升級或涉及系統服務配置時，建議使用具有管理員權限的帳戶

### 硬體與顯示

- 建議使用 1920x1080 及以上解析度
- 建議為影像處理和流程執行預留充足記憶體與磁碟空間
- 若需要連線相機、光譜儀、電機等裝置，應提前確認對應驅動和通訊環境已經準備好

### 網路與權限

- 首次安裝後若需要檢查更新、訪問外掛市場或連線遠端服務，請保證網路可用
- 如果部署環境限制較嚴，請提前確認程式目錄、日誌目錄和使用者文件目錄具有可寫權限

## 從安裝包執行時需要注意什麼

- 安裝程式會按自身流程檢查並部署所需元件；如果提示缺少先決條件，請按安裝器提示完成補齊
- 某些服務管理、裝置驅動配置或系統級寫入操作，可能需要管理員權限
- 若環境中存在舊版本，建議先完成升級或解除安裝，再做新安裝

## 從原始碼建置時需要準備什麼

當前倉庫以 Windows WPF 和 x64 為主，建議準備以下環境：

- .NET 8.0 SDK 或 .NET 10.0 SDK
- Visual Studio 2022 或更新版本（推薦）
- Git 與 PowerShell（用於獲取倉庫和執行指令碼）

建議使用 x64 平台建置主程式：

```powershell
dotnet restore
dotnet build .\ColorVision\ColorVision.csproj -p:Platform=x64
```

## 執行時依賴說明（原始碼場景）

如果你是從原始碼直接執行，而不是透過安裝包部署，還需要注意執行輸出中應包含以下依賴：

- OpenCvSharp Windows 執行時
- `DLL/CVCommCore.dll`
- `DLL/MQTTMessageLib.dll`
- 主程式需要的 `log4net.config` 與相關資原始檔

這些內容通常由專案引用和複製規則處理；如果程式可以建置但啟動失敗，優先檢查輸出目錄是否缺少上述依賴。
