# 相機服務

本頁說明“相機服務”這一層在日常使用中的作用，以及它和物理相機、參數配置、校準模板之間的關係。

## 相機服務和物理相機不是一回事

- 物理相機：真實硬體、許可證、資源和基礎能力
- 相機服務：專案裡真正被裝置列表、檢視視窗和流程呼叫的邏輯裝置
- 一臺相機服務通常要先繫結一個物理相機，之後才能穩定使用採圖、影片、校準和模板功能

## 你通常會在這裡做什麼

- 編輯相機服務配置
- 開啟相機檢視或實時畫面
- 繫結或切換物理相機
- 進入自動曝光、自動對焦、相機參數模板
- 開啟校準參數編輯視窗
- 檢視本地除錯入口和相機日誌

## 推薦使用順序

1. 先在 [物理相機管理](./camera-management.md) 中確認硬體、許可證和資源可用。
2. 回到相機服務，把當前服務例項繫結到正確的物理相機。
3. 在 [相機參數配置](./camera-configuration.md) 中調整曝光、增益、通道和模式。
4. 需要時再進入自動曝光、自動對焦或校準模板。
5. 用實時畫面、本地視窗或流程執行驗證結果。

## 這頁對應的能力範圍

當前程式碼裡，相機服務這一層已經包含了幾類常用入口：

- 編輯相機配置
- 重新整理裝置 ID 和物理相機繫結
- 自動曝光模板
- 自動對焦模板
- 相機參數模板
- 校準模板
- 相機日誌和本地視窗

因此遇到“相機能不能被系統正常使用”的問題，通常先看相機服務，而不是直接去看底層相機資源。

## 常見問題

### 相機服務存在，但打不開或採不到圖

- 先檢查是否已經繫結物理相機
- 再檢查許可證、連線狀態和相機日誌
- 若剛改過配置，重新開啟相關視窗或重啟服務後再試

### 改了物理相機配置後，相機服務沒同步

- 儲存後重新進入相機服務配置
- 檢查當前服務是否繫結到了預期的物理相機
- 必要時重新整理裝置 ID 或重新繫結一次

### 影片或本地模式異常

- 先關閉當前影片或實時模式，再做重連或重啟
- 優先排查是連線問題還是參數問題
- 需要時檢視相機日誌確認後臺服務狀態
