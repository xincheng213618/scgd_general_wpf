# 新增與配置裝置

本頁只講使用者操作路徑：怎麼新增一個裝置服務、怎麼儲存、改參數時先看哪裡。

## 常見場景

- 新接入一臺裝置，需要先在系統裡建立對應服務。
- 裝置已經存在，但需要修改名稱、通訊參數或繫結物件。
- 專案切換後，需要調整裝置順序、禁用舊裝置或補充新裝置。

## 基本流程

1. 開啟裝置管理視窗。
2. 進入裝置設定或編輯介面。
3. 新增對應型別的裝置服務，或選中已有裝置進行修改。
4. 填寫必要參數後儲存。
5. 返回裝置列表，確認裝置已經出現在可用列表中。

如果後續流程、相機視窗或結果介面仍然看不到該裝置，先回到裝置列表確認這一步是否真正生效。

## 配置時通常要確認的內容

### 基本資訊

- 裝置名稱是否易於區分
- 裝置型別是否選對
- 當前專案是否引用了正確的裝置例項

### 通訊或繫結資訊

- 網路地址、埠、主題或通訊標識是否正確
- 是否繫結了對應的物理裝置或資源物件
- 本機驅動、服務端或第三方程式是否已準備好

### 儲存後的可見性

- 裝置是否已經出現在裝置列表
- 裝置狀態是否正常
- 相關視窗或流程節點裡是否能選到該裝置

## 常見問題

### 新增後列表裡沒有裝置

- 檢查是否真的儲存成功
- 檢查是否在錯誤的專案或錯誤的配置上下文裡操作
- 重新開啟裝置視窗確認是否只是列表沒有重新整理

### 參數改了但裝置行為沒變化

- 檢查是否需要重新連線、重新生成控制項或重新開啟相關視窗
- 檢查是否改的是當前實際使用的那一臺裝置例項

### 配置頁看不懂該填什麼

- 先去對應裝置專題頁找參數說明
- 如果該裝置依賴物理裝置物件，先完成物理裝置管理或標定步驟
