# SMU 服務

SMU 服務用於做源測量一體化操作，例如給定電壓測電流、給定電流測電壓，以及按模板執行一段掃描。它適合在需要點測、IV 掃描或把電學結果聯動到後續流程時使用。

## 什麼時候應該看這一頁

- 你要新增或配置一個 SMU 裝置服務
- 你要在 A/B 通道之間切換測量
- 你要做一次手動點測
- 你要執行一組掃描參數，例如 IV 掃描
- 你想確認結果會顯示在哪裡、如何被後續視窗使用

## 這個服務裡最關鍵的幾個概念

### 通道

當前實現支援至少按通道區分配置和結果，常見是：

- Channel A
- Channel B

手動操作和結果顯示都要先確認當前通道，不要把 A 通道的結果當成 B 通道在看。

### 源型別

SMU 可以在兩種常見模式之間切換：

- 以電壓為源，測電流
- 以電流為源，測電壓

顯示配置裡儲存的是當前通道下、當前源型別對應的手動點測參數；掃描時用的不是這組顯示參數，而是單獨的掃描模板參數。

### 手動點測和掃描不是一回事

- 手動點測通常使用當前顯示配置裡的測量值和限值
- 掃描使用 `SMUParam` 模板裡的起點、終點、點數和限值

如果你改了顯示配置，但掃描結果沒有變化，先確認你實際執行的是不是模板掃描。

## 常見使用順序

1. 先在裝置列表裡確認 SMU 服務已經建立並能正常連線。
2. 進入 SMU 視窗，先確認當前通道和源型別。
3. 做手動點測時，先設定測量值和限值，再執行一次讀數。
4. 做掃描時，先確認使用的 `SMUParam` 模板，再執行掃描。
5. 結果出來後，先看當前視窗中的結果檢視，再決定是否交給流程或其他服務繼續使用。

## 手動點測時要注意什麼

- 先確認當前是電壓源還是電流源
- 再確認輸入的是測量值還是限值
- 如果啟用了限值保護，超過裝置限制的參數不會繼續傳送
- 讀數出來後，要確認它屬於當前通道

## 掃描時要注意什麼

掃描通常依賴模板參數，而不是臨時手輸的顯示值。常見會關心的內容包括：

- 起始值
- 結束值
- 限值
- 點數
- 當前通道
- 當前源型別

如果想做 IV 掃描，先確認模板裡的源型別和上下限是否匹配，再執行掃描，不要只改顯示面板裡的當前值。

## 結果會出現在哪裡

- 單次測量結果會回到 SMU 結果檢視
- 掃描結果也會在 SMU 檢視中新增結果項
- 當前選中通道的電壓、電流讀數會同步回顯示配置
- 一部分電壓、電流結果還會聯動給依賴這兩個值的其它視窗或服務

## 常見問題

### 點測沒有返回結果

- 先確認裝置已經開啟並且通道選對
- 再確認當前源型別、測量值和限值是否合理
- 如果啟用了限值保護，先檢查是不是超過了裝置允許範圍

### 掃描結果和手動設定不一致

- 先確認執行的是掃描模板而不是手動點測
- 再確認模板裡的起止值、限值和點數
- 不要把顯示配置裡的當前值誤認為掃描參數

### 兩個通道的資料看起來串了

- 先確認當前顯示的是 A 還是 B
- 再確認最近一次操作針對的是哪個通道
- 看結果時優先按通道理解，不要只盯最後一條讀數

### 結果出來了，但後續流程沒有用到

- 先確認後續流程或視窗讀取的是不是當前通道的值
- 再確認是否真的需要從 SMU 結果檢視而不是顯示配置裡取值
- 必要時聯動檢查 [工作流程概覽](../workflow/README.md) 和 [流程執行與除錯](../workflow/execution.md)
