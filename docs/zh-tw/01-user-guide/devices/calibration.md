# 校準服務

校準服務用於把校準流程和實際相機繫結起來，管理校準模板、觸發校準資料生成，並在需要時處理原始檔案和快取。它不是單純的“參數頁”，而是連線物理相機、模板編輯和校準結果的一層服務。

## 什麼時候應該看這一頁

- 你要新建或配置一個校準服務
- 你要把校準服務繫結到某個物理相機
- 你要編輯校準模板
- 你要觸發一次校準相關的資料獲取
- 你在排查“為什麼不能開始校準”“為什麼模板打不開”這類問題

## 使用前先確認兩件事

### 先繫結物理相機

當前實現裡，校準服務會直接依賴物理相機程式碼。如果沒有先配好對應物理相機，很多校準相關操作根本不會繼續。

### 再確認資料庫可用

如果當前環境啟用了 MySQL，但資料庫沒有連通，校準模板編輯不會正常進入。遇到這種情況，先修資料庫連線，不要反覆點模板視窗。

## 常見使用順序

1. 先在裝置列表裡建立或確認校準服務。
2. 把它繫結到正確的物理相機。
3. 開啟校準模板編輯入口，確認模板內容和當前相機對應。
4. 需要取校準資料時，再執行相關校準動作。
5. 如果物理相機配置變了，記得回頭確認校準服務是否仍然繫結正確。

## 這個服務裡最常見的操作

### 管理物理相機

如果當前校準服務對應的相機還沒準備好，先去物理相機管理頁處理，而不是隻停留在校準視窗裡排查。

可繼續閱讀：

- [相機管理](./camera-management.md)
- [相機參數配置](./camera-configuration.md)

### 編輯校準模板

校準模板編輯依賴當前繫結的物理相機。模板視窗能否正常開啟，本身就是一個很有價值的健康檢查：

- 打不開時，優先查相機繫結和資料庫連線
- 能開啟但內容不對時，優先查當前是不是正確的相機和模板物件

### 處理原始檔案和快取

校準服務還會涉及原始檔案列表、檔案下載和快取清理。遇到“舊資料干擾當前校準”這類問題時，先從快取和原始檔案範圍入手，而不是直接懷疑整個服務不可用。

## 常見問題

### 點了編輯校準模板，但沒有進入編輯器

- 先確認是否已經繫結了對應物理相機
- 如果啟用了 MySQL，再確認資料庫連線是否正常
- 再確認當前開啟的是正確的校準服務例項

### 相機已經換了，但校準結果看起來還是舊的

- 先確認校準服務繫結的 `CameraCode` 是否已經更新
- 再確認物理相機切換後有沒有重新儲存當前配置
- 必要時重新進入模板編輯器確認當前模板物件

### 校準過程中拿到的原始檔案不對

- 先確認當前裝置程式碼和裝置型別是不是目標裝置
- 再確認請求的是不是原始檔案列表而不是舊快取
- 如果懷疑快取干擾，先做一次快取清理再複查

### 校準服務存在，但流程裡不生效

- 先確認繫結的相機和模板是否有效
- 再聯動檢查 [工作流程概覽](../workflow/README.md) 和 [流程執行與除錯](../workflow/execution.md)
- 同時結合 [日誌檢視器](../interface/log-viewer.md) 看具體失敗階段
