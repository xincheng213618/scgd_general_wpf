# 電機服務

電機服務用於把電機裝置納入 ColorVision 的統一裝置體系，並提供開啟、關閉、移動、回原點和讀取當前位置等常見控制動作。它適合在手動除錯和自動化流程裡都需要位置控制的場景使用。

## 什麼時候應該看這一頁

- 你要新增或配置一個電機服務
- 你要手動移動電機到指定位置
- 你要確認當前位置是否已經更新
- 你要排查“電機能看到但不動”或“位置沒有重新整理”的問題

## 常見使用順序

1. 先在裝置列表裡建立並儲存電機服務。
2. 檢查通訊配置和超時參數是否合理。
3. 先執行一次開啟動作，確認裝置處於可操作狀態。
4. 再做移動、回原點或讀位置等動作。
5. 完成後如果業務要求關閉裝置，再執行關閉動作。

## 這個服務裡常見的操作

### 開啟和關閉

很多後續動作都預設裝置已經處於可操作狀態。遇到“命令發了但沒反應”時，先確認是否真的已經成功開啟，而不是直接懷疑移動命令。

### 移動到指定位置

常見會關心這幾個參數：

- 目標位置
- 絕對移動還是相對移動
- 超時時間

如果是除錯階段，先用較小範圍移動確認方向和行為，再做大步長調整。

### 回原點

噹噹前位置不可信，或者需要重新建立統一起點時，優先先回原點再繼續做後續動作。

### 獲取當前位置

當前位置不是靠猜的，應該主動讀回來確認。特別是在自動化流程或手動多次移動之後，先讀位置再做下一步通常更穩。

## 常見問題

### 電機服務存在，但一直打不開

- 先檢查通訊配置是否正確
- 再確認裝置是否線上
- 如果開啟動作失敗，後續移動和讀位置通常也不會可靠

### 傳送了移動命令，但位置沒變化

- 先確認裝置是否已經成功開啟
- 再確認目標位置、絕對/相對模式和超時時間
- 操作後主動讀一次當前位置，不要只看介面是否重新整理

### 當前位置顯示不對

- 先重新執行一次獲取位置
- 再確認是不是上一次移動失敗但介面仍保留舊值
- 如果涉及回原點，先確認回原點動作是否真正完成

### 在流程裡呼叫電機動作失敗

- 先用手動方式驗證這臺電機當前能否正常開啟和移動
- 再檢查流程裡的參數是不是和手動測試一致
- 必要時聯動看 [流程執行與除錯](../workflow/execution.md) 和 [日誌檢視器](../interface/log-viewer.md)
