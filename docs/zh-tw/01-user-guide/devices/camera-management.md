# 物理相機管理

本頁說明物理相機這一層負責什麼。它更像“硬體資源臺賬”，負責發現、授權和維護真實相機，而不是直接代替相機服務。

## 什麼時候應該先來這裡

- 首次接入新相機
- 匯入或更新許可證
- 重新整理可建立的相機 ID
- 管理校準資源、分組和物理配置
- 邏輯相機服務找不到可繫結物件

## 物理相機層負責什麼

- 建立物理相機物件
- 匯入 `.lic` 或 `.zip` 許可證
- 維護物理相機配置和資源樹
- 為相機服務提供可繫結的相機程式碼和基礎參數

這意味著“硬體有沒有被系統真正識別”和“許可證是否有效”這類問題，通常都應該先在這一層排查。

## 常見操作順序

1. 接好裝置並確認系統能夠看到空閒相機 ID。
2. 在物理相機管理中建立相機，或匯入許可證。
3. 檢查相機名稱、程式碼、模式和基礎配置。
4. 如有需要，準備校準、分組和相關資源。
5. 返回 [相機服務](./camera.md)，把邏輯服務繫結到這個物理相機。

## 配置時要特別確認什麼

- 許可證是否有效
- 是否真的存在未建立的相機 ID
- 物理相機資源是否完整
- 相機模式是否和實際硬體一致
- 修改後是否需要回到邏輯相機服務重新儲存或重新整理

## 常見問題

### 找不到未建立的相機

- 確認硬體已經連線並被系統識別
- 先重新整理裝置 ID 再重試
- 如果剛插拔過裝置，重新開啟視窗後再看一次

### 匯入許可證後仍不可用

- 確認匯入檔案格式正確
- 確認許可證對應的相機程式碼與當前硬體一致
- 檢查資料庫或本地資源是否已經同步

### 改了物理相機配置，現有相機服務行為異常

- 檢查邏輯服務是否需要重新儲存或重新繫結
- 關注配置變更後是否需要重啟相關服務
- 若涉及路徑和資源調整，確認相關檔案已經落到新位置
