# 檔案服務

檔案服務用於把遠端檔案列表、檔案下載和檔案上傳納入統一裝置體系。它適合在“結果檔案在服務端”“需要從裝置端取圖”“需要把檔案送到遠端”這類場景下使用。

## 什麼時候應該看這一頁

- 你要新增或配置一個檔案服務
- 你要檢視遠端可用檔案列表
- 你要下載某個原始檔到當前系統
- 你要上傳檔案到遠端服務
- 你在排查“檔案看得到但打不開”“上傳後沒反應”這類問題

## 這個服務通常怎麼用

### 先完成配置

先保證裝置程式碼、傳送主題和訂閱主題這些基礎配置是正確的。檔案服務很多問題看起來像“檔案操作失敗”，實際更早就卡在連線或主題配置了。

### 再看檔案列表

如果你都不知道遠端現在有哪些檔案，先取檔案列表，再決定要下載還是上傳，不要直接猜檔名。

### 需要內容時再開啟或下載

當前實現裡常見檔案動作包括：

- 獲取全部檔案列表
- 開啟或下載指定檔案
- 上傳指定檔案

## 常見使用順序

1. 先在裝置列表裡建立並儲存檔案服務。
2. 確認連線相關配置無誤。
3. 先拉一次檔案列表，確認遠端確實有目標檔案。
4. 再執行下載或開啟動作。
5. 需要回傳檔案時，再執行上傳。

## 使用時要注意什麼

### 檔名要先確認

檔案服務是按檔名發起操作的。檔名錯了，後面再看顯示視窗意義不大。

### 列表和內容是兩步

能看到列表，不代表具體檔案一定能正常開啟；同樣，上傳成功也不代表你已經在正確目錄或正確型別下檢視到了它。

### 先排除主題和裝置程式碼問題

當前實現對裝置程式碼和訂閱主題有明確匹配。遇到沒有響應時，優先先查這些基礎條件。

## 常見問題

### 檔案列表拉不回來

- 先確認傳送主題和訂閱主題是否正確
- 再確認當前裝置程式碼是否和返回訊息匹配
- 如果服務端有心跳但沒有檔案事件，優先查具體檔案操作配置

### 檔案能列出來，但開啟失敗

- 先確認檔名是否正確
- 再確認請求的是不是目標檔案型別
- 如果問題持續，結合 [日誌檢視器](../interface/log-viewer.md) 看返回訊息裡有沒有更明確的錯誤資訊

### 上傳後看不到檔案

- 先確認上傳動作是否真的發出並完成
- 再重新拉一次檔案列表，而不是隻依賴舊列表
- 如果遠端有目錄或型別區分，確認你檢視的是正確範圍

### 檔案服務在流程裡不穩定

- 先用手動方式驗證檔案列表、下載或上傳能否成功
- 再回到 [流程執行與除錯](../workflow/execution.md) 看流程裡用的是不是同一臺檔案服務
- 必要時同時檢查裝置程式碼和主題配置是否與手動測試一致
