# 日誌檢視器

日誌檢視器用於定位啟動異常、裝置通訊問題、流程執行錯誤和外掛裝載失敗。遇到“介面沒反應、功能沒生效、裝置連不上”這類問題時，先看日誌通常比盲猜更快。

## 什麼時候應該開啟它

- 主視窗或某個功能開啟失敗
- 裝置連線狀態異常
- 流程執行報錯但介面提示不夠明確
- 外掛、服務或模板載入行為異常

## 你通常會用到的功能

- 實時檢視新日誌
- 搜尋關鍵詞
- 按日誌級別過濾
- 檢視一定範圍的歷史日誌
- 在自動滾動和手動閱讀之間切換

## 基本使用方式

1. 從幫助選單或對應入口開啟日誌視窗。
2. 先看最近一段日誌，確認錯誤發生的大致時間點。
3. 用關鍵詞搜尋模組名、裝置名、異常關鍵字或檔名。
4. 如果日誌太多，再把級別提高到 `Warn` 或 `Error` 縮小範圍。

## 搜尋時怎麼更高效

- 先用簡單關鍵詞，例如裝置名、模板名、`error`、`timeout`
- 結果過多時，再組合多個關鍵詞
- 只有確定需要時再用正規表示式

如果輸入後搜尋框變紅，通常說明正則寫錯了；此時先退回普通關鍵詞搜尋更穩。

## 常見問題

### 開啟日誌視窗後幾乎沒有內容

- 先檢查自動重新整理是否開啟
- 再檢查當前日誌級別是否過高
- 如果只想看新日誌，確認當前載入策略不是空

### 日誌很多，介面不容易看

- 先把級別切到 `Warn` 或 `Error`
- 再用關鍵詞縮小範圍
- 若只是臨時排障，先不要追求保留全部歷史日誌

### 搜尋結果不對

- 先確認是否把多個關鍵詞都寫成了必須同時匹配
- 如果用了正則，先確認特殊字元是否需要轉義
- 搜尋不到時，回到未過濾檢視重新找時間點

## 排查建議

### 裝置問題

- 先搜裝置名、相機程式碼、服務名
- 再聯動看 [裝置服務概覽](../devices/overview.md)

### 流程問題

- 先搜模板名、節點名或執行失敗關鍵字
- 再聯動看 [流程執行與除錯](../workflow/execution.md)

### 外掛或啟動問題

- 先看應用啟動階段的最近日誌
- 再關注是否有 DLL、版本、路徑或載入失敗提示
