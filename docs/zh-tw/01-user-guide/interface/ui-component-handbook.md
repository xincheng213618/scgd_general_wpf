# UI 元件使用手冊

本頁從操作員、測試工程師和現場交付人員視角說明 ColorVision 常見 UI 元件。它回答：什麼時候打開、從哪裡進入、能做什麼、完成標準是什麼、失敗時先查什麼。

如果你要發布 DLL 或修改源碼，請轉到 [UI DLL 元件手冊](../../04-api-reference/ui-components/component-handbook.md) 和 [UI 元件目錄](../../04-api-reference/ui-components/control-catalog.md)。本頁只保留使用手冊層面的操作說明。

## 元件總覽

| UI 元件 | 何時使用 | 常見入口 | 主要操作 | 完成標準 | 失敗先查 |
| --- | --- | --- | --- | --- | --- |
| 主視窗工作台 | 進入系統後的日常起點 | 程式啟動後自動進入 | 找菜單、搜尋、工作區、狀態列 | 能找到裝置、流程、圖像、日誌、插件入口 | 插件是否載入、權限、語言、布局 |
| 頂部菜單 | 打開全域工具、裝置、插件、幫助 | 主視窗頂部 | 按業務選功能或插件 | 對應視窗或功能打開 | 菜單權限、插件狀態、熱鍵衝突 |
| 搜尋框 | 不知道功能在哪時快速定位 | 主視窗搜尋區 | 輸入關鍵字並打開命令/頁面 | 搜到目標並可打開 | 關鍵字、插件載入、搜尋索引 |
| 狀態列 | 查看服務、插件、Socket、調度狀態 | 主視窗底部 | 查看提示或點狀態圖示 | 狀態與現場一致 | Provider、服務配置、日誌 |
| 設定視窗 | 修改全域或模組配置 | 設定/選項菜單 | 查找設定、修改、保存、重啟驗證 | 修改值重啟後仍生效 | 配置路徑、權限、欄位型別、只讀項 |
| 屬性編輯器 | 編輯裝置、模板、流程節點、配置物件 | 屬性面板或配置彈窗 | 按分類看參數，修改保存 | 物件行為符合新參數 | 元資料、校驗、只讀欄位、保存路徑 |
| 日誌檢視器 | 排查啟動、裝置、流程、插件異常 | 幫助 -> 日誌，預設 `Ctrl+L` | 按時間、等級、關鍵字過濾 | 找到第一條有意義錯誤 | 日誌等級、時間點、模組名 |
| 終端 | 現場執行命令、腳本或查看檔案 | 終端入口或工作區 | 執行命令、查看輸出 | 命令能返回清晰結果 | 目前目錄、權限、環境變數 |
| 影像編輯器 | 查看圖像、overlay、ROI、影片、3D、偽彩 | 結果圖、檔案打開、工作區 | 縮放、標註、量測、匯入/匯出註釋 | 圖像和疊加層對齊 | 檔案路徑、圖像仍在寫入、座標映射 |
| 資料庫瀏覽器 | 查看 MySQL/SQLite 表資料 | 工具 -> 資料庫瀏覽器 | 選資料源、表、搜尋、分頁 | 找到目標記錄或確認未落庫 | 連線配置、搜尋條件、頁碼、主鍵 |
| Socket 管理視窗 | 查看本機 TCP server 和訊息 | 狀態列 Socket 圖示 | 查啟用、端口、連線、訊息歷史 | 外部系統能連線並收發 | 端口占用、協議模式、Handler 載入 |
| 調度任務視窗 | 查看和維護 Quartz 任務 | 工具 -> 任務管理 | 新建、暫停、恢復、立即執行、看歷史 | 下次執行時間和記錄正確 | Cron、任務程序集、執行異常 |
| 工作區/檔案樹 | 管理 `.cvsln`、專案檔和編輯器 | Solution 工作區 | 新建、打開、編輯檔案、切換布局 | 檔案用正確編輯器打開 | 檔案型別、編輯器註冊、布局快取 |
| 插件市場 | 安裝、查看、更新插件/DLL | 幫助 -> 插件市場 | 查看版本、下載或更新 | 下載後宿主能載入 | 管理員權限、網路、manifest、版本 |
| 下載器 | 下載插件、更新包或外部資源 | 插件市場或下載視窗 | 新增任務、看進度、重試 | 檔案完整落到目標目錄 | aria2c、路徑權限、網路 |
| 向導視窗 | 分步初始化或配置 | 向導入口 | 填寫、下一步、完成 | 每步校驗通過並生成配置 | 必填項、裝置連線、輸出路徑 |

## 基礎介面元件

第一次打開主程式時，先確認四件事：菜單是否完整、搜尋是否能定位命令、工作區是否能打開視窗/圖像/流程、狀態列是否顯示服務狀態。主視窗只負責組織入口，不代表某條業務流程；功能沒有出現時，先判斷是菜單未註冊、插件未載入、權限不足，還是專案包未啟用。

| 元件 | 使用建議 | 交接提醒 |
| --- | --- | --- |
| 菜單 | 固定入口和管理員工具優先走菜單 | 菜單可能來自主程式、UI 模組、插件或專案包 |
| 搜尋 | 新使用者不知道入口時優先用搜尋 | 搜不到不一定是功能不存在，可能是插件未載入 |
| 狀態列 | 服務和背景任務優先看狀態列 | Socket、調度等圖示可直接打開管理視窗 |

如果按鈕沒有反應，先打開 [日誌檢視器](./log-viewer.md)，按點擊時間查最近的 `Error` 或 `Warn`。

## 配置和參數元件

設定視窗面向全域配置和模組配置。現場修改配置時，先確認要改的是全域、裝置、流程還是專案配置；修改前記錄原值，尤其是端口、路徑、資料庫、Socket、檔案伺服器；修改後保存，按需要重啟應用或刷新服務；最後用狀態列、日誌、裝置頁或專案頁驗證。

屬性編輯器用於編輯裝置、模板、流程節點、圖元、插件配置和專案配置。若沒有出現預期控件，開發側應檢查屬性的 `Category`、`DisplayName`、`Description`、自訂 editor 類型和可見性標記。

## 診斷元件

| 問題類型 | 日誌關鍵字 | 下一頁 |
| --- | --- | --- |
| 啟動失敗 | `Error`、`DllNotFoundException`、插件名、配置檔 | [常見問題](../troubleshooting/common-issues.md) |
| 裝置連不上 | 裝置名、端口、IP、`timeout`、服務名 | [裝置服務概覽](../devices/overview.md) |
| 流程失敗 | 流程名、節點名、模板名、`failed` | [流程執行與除錯](../workflow/execution.md) |
| 插件未出現 | 插件目錄名、`manifest`、`deps.json`、DLL 名 | [現有插件能力說明](../../04-api-reference/plugins/README.md) |
| 資料未寫入 | 表名、批次號、SN、匯出路徑 | [資料管理概覽](../data-management/README.md) |

終端適合交付工程師和開發支持做現場環境檢查，例如列目錄、測網路、跑腳本。普通操作員不應依賴終端完成日常流程。

## 資料、通信和調度

| 元件 | 適合做什麼 | 完成標準 | 失敗先查 |
| --- | --- | --- | --- |
| 資料庫瀏覽器 | 查結果是否落庫、記錄是否可查 | 能按 SN/時間/批次找到記錄 | 連線、條件、主鍵、只讀 |
| Socket 管理視窗 | 查 TCP server 狀態和訊息歷史 | 外部系統能連線並收發 | 端口、防火牆、協議模式、Handler |
| 調度任務視窗 | 維護 Quartz 定時任務 | 下次執行時間和歷史正確 | Cron、調度器狀態、任務程序集 |

若專案包透過 Socket 觸發，還要看 [專案說明](../../00-projects/README.md) 和對應專案頁，確認事件名、欄位和返回格式。

## 影像和工作區

影像編輯器不只是看圖，它承載結果查看、ROI/POI、標註、overlay、影片、偽彩、直方圖、3D 和 CIE。圖像異常時先確認檔案是否寫完，再查座標映射和 overlay 來源。

工作區負責 `.cvsln`、檔案樹、編輯器標籤、終端和多圖瀏覽。它適合組織檔案，不負責執行 Engine Flow。若目標是跑客戶專案，應打開對應專案視窗或 [專案說明](../../00-projects/README.md)。

## 元件邊界

| 你看到的問題 | 先歸到哪裡 |
| --- | --- |
| 視窗打不開、菜單沒有、按鈕無反應 | UI 操作和日誌 |
| 裝置狀態異常、相機不能採集、電機不動 | 裝置服務 |
| 流程節點失敗、結果沒有生成 | 工作流程和 Engine |
| Socket 能連但欄位不對 | 專案包協議或 Socket handler |
| 插件沒有出現或版本不相容 | 插件載入和插件市場 |
| UI DLL 或 native DLL 缺失 | UI DLL 發布和安裝包 |
| 資料庫查不到結果 | 資料管理、流程寫庫、專案匯出 |

## 交接檢查清單

- 從主視窗打開日誌、設定、資料庫瀏覽器、調度任務和插件市場。
- 修改一個安全配置項，確認保存和重啟後仍生效。
- 打開一張圖像，完成縮放、標註、屬性查看和註釋匯出。
- 用資料庫瀏覽器按時間或 SN 查一條結果。
- 如果啟用 Socket，確認端口、連線和訊息歷史。
- 如果交付插件或專案包，確認對應菜單、狀態圖示或專案視窗出現並可打開。
