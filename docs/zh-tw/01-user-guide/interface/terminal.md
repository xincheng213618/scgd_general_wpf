# 終端

終端面板適合做兩類事情：執行指令碼，和直接觀察命令列輸出。它更像主程式裡的工作輔助區，而不是單獨的開發文件入口。

## 什麼時候應該開啟它

- 需要直接執行 PowerShell 或 CMD 命令
- 想在當前工程上下文裡執行指令碼
- 需要觀察指令碼輸出、報錯或互動提示
- 想快速做一次命令列檢查，而不切出主程式

## 你通常會怎麼用

### 選擇 Shell

終端通常支援至少兩種常見 Shell：

- PowerShell
- CMD

如果當前命令依賴 PowerShell 語法，就不要在 CMD 裡直接重試；先確認 Shell 型別再執行。

### 執行指令碼

常見做法包括：

- 直接輸入命令執行指令碼
- 從資源管理或相關入口觸發“在終端中執行”
- 在排查時先跑最小命令，再逐步追加參數

### 看輸出和報錯

- 優先先看第一條錯誤
- 不要只盯最後一行
- 如果指令碼還會繼續輸出，先確認它是不是仍在執行，而不是假設已經卡死

## 建議的使用順序

1. 先確認當前要用的是 PowerShell 還是 CMD。
2. 再確認當前工作目錄是不是你預期的工程位置。
3. 先執行最小命令，確認環境、路徑和權限都正常。
4. 命令失敗時，優先結合 [日誌檢視器](./log-viewer.md) 和終端輸出一起判斷。

## 常見問題

### 命令能執行，但結果和預期不一致

- 先確認當前 Shell 型別
- 再確認當前目錄、環境變數和指令碼參數
- 如果指令碼依賴外部程式，先確認這些程式是否真的可用

### 終端沒有輸出，或看起來沒反應

- 先確認命令是不是還在等待輸入
- 再確認是否切到了錯誤的 Shell
- 如果是長任務，先給它一點時間輸出第一批結果，不要馬上重複提交命令

### 執行指令碼時報路徑或權限錯誤

- 先檢查路徑是否正確
- 再確認指令碼檔案和依賴檔案是否存在
- 涉及 PowerShell 指令碼時，優先確認執行策略和呼叫方式是否正確

### 互動式命令表現異常

- 先新建一個新的終端會話重試
- 再嘗試改用更簡單的命令確認是否是互動命令本身的問題
- 如果只是想看結果，不一定要在終端裡完成全部互動
