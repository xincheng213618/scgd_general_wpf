# 常見問題

本頁不是程式碼分析，而是一個排查入口。遇到問題時，先確定它屬於哪一類，再跳到對應頁面繼續縮小範圍。

## 先按這個順序排查

1. 先看 [日誌檢視器](../interface/log-viewer.md)，確認錯誤發生在啟動、裝置、流程還是外掛階段。
2. 再確認當前專案、裝置例項、模板或視窗是不是你以為的那個物件。
3. 再檢查路徑、許可證、資料庫連線、服務狀態這些外部條件。
4. 最後再考慮是否需要重開視窗、重新儲存配置或重啟相關服務。

## 常見場景

### 主視窗裡找不到功能

- 先回到 [主視窗導覽](../interface/main-window.md)
- 優先使用搜尋框定位入口
- 如果是外掛功能，確認外掛是否已經正確載入

### 裝置存在，但不能工作

- 先看 [裝置服務概覽](../devices/overview.md)
- 相機場景繼續看 [相機服務](../devices/camera.md)
- 若是繫結或硬體接入問題，再看 [物理相機管理](../devices/camera-management.md)

### 參數改了，但效果沒變化

- 先確認改的是當前正在使用的物件
- 再確認是否已經儲存，並且相關視窗或服務已經重新載入
- 屬性和模板相關場景，優先回到 [屬性編輯器](../interface/property-editor.md) 或對應裝置參數頁複查

### 流程執行失敗

- 先看 [流程執行與除錯](../workflow/execution.md)
- 日誌裡優先搜節點名、模板名和失敗關鍵字
- 不要一開始就懷疑整條流程，先縮到第一個失敗節點

### JSON 或模板內容校驗失敗

- 先用編輯器裡的校驗按鈕確認是不是格式問題
- 再看是否把註釋文字、說明文字或舊模板內容誤貼進了 JSON
- 如果切換了不同編輯模式，先回到當前實際儲存的 JSON 再判斷

### 日誌太多，看不出重點

- 先把日誌級別切到 `Warn` 或 `Error`
- 再用關鍵詞縮小範圍
- 如果搜尋框變紅，先放棄正則，退回普通關鍵詞

## 如果還是沒定位到

- 記錄出問題的時間點
- 記下當前專案、裝置名、模板名或外掛名
- 附帶對應日誌片段再繼續排查，效率會明顯高很多
