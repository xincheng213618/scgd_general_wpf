# 資料匯出與匯入

當前倉庫裡並沒有一個統一的“所有資料都從這裡匯入匯出”的總控中心。更接近真實情況的是：不同物件走不同入口，設定、流程模板和具體結果資料各自有自己的匯入匯出方式。

## 先怎麼理解這一頁

如果你要做匯入匯出，先回答三個問題：

1. 你要處理的是設定、流程模板，還是某個結果模組的資料？
2. 你需要的是全域性配置遷移，還是單個業務物件匯出？
3. 這個功能是不是本來就屬於某個具體視窗，而不是資料管理中心？

## 當前能明確證實的入口

### 設定匯入匯出

當前有明確選單入口：

- 工具 → 匯入匯出設定

這裡至少包含兩類動作：

- 匯出設定到 `.cvsettings`
- 從 `.cvsettings` 匯入設定

如果你的目標是遷移軟體配置，而不是遷移結果資料，優先從這裡開始。

### 流程模板匯入匯出

流程模板的匯入匯出不在資料管理頁統一處理，而是在流程設計器中單獨提供：

- 匯出當前流程
- 匯入流程
- 匯入模組

如果你要遷移流程內容，優先去 [流程設計](../workflow/design.md)。

### 模組內結果匯出

一些業務視窗會提供自己的匯出功能，例如：

- 流程節點分析視窗可匯出 CSV
- 部分外掛或影像/測量視窗有自己的 CSV 或影像匯出入口

這類匯出通常跟具體業務物件強繫結，不應該再被描述成統一的全域性“資料匯出中心”。

## 常見使用順序

1. 先確認你要匯出的到底是什麼物件。
2. 如果是全域性設定，走“匯入匯出設定”。
3. 如果是流程內容，走 [流程設計](../workflow/design.md) 裡的匯入匯出。
4. 如果是結果資料或業務資料，先回到對應模組視窗找它自己的匯出入口。
5. 真正涉及庫內資料時，再配合 [資料庫操作](./database.md) 確認源資料範圍。

## 當前這頁不再承諾什麼

以下這些能力，當前使用者指南頁不再預設宣稱為統一可用能力，除非你已經在具體模組視窗裡看到了它：

- 統一 Excel 匯出中心
- 統一 JSON 匯出中心
- 統一 XML 匯出中心
- 統一 PDF 報告匯出中心
- 通用列對映匯入嚮導
- 通用批次資料夾匯入嚮導

如果某個外掛或視窗確實支援這些格式，應在那個模組自己的頁面說明，而不是放在這裡泛講。

## 常見問題

### 我想匯出資料，但找不到統一入口

- 先不要繼續找總選單
- 先確認物件屬於設定、流程還是某個業務結果視窗
- 再回到對應模組頁查詢匯出入口

### 匯出了設定，但業務結果沒有跟著遷移

- `.cvsettings` 主要用於配置遷移，不等於資料庫結果遷移
- 真正的結果資料要結合 [資料庫操作](./database.md) 或對應業務模組單獨處理

### 流程匯入成功了，但執行結果不對

- 先確認匯入的是不是正確流程版本
- 再回到 [流程執行與除錯](../workflow/execution.md) 驗證依賴裝置和模板
- 必要時重新檢查模組匯入和流程參數
