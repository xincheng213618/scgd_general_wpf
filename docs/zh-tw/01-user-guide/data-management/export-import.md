# 資料匯出與匯入

當前倉庫裡並沒有一個統一的“所有資料都從這裡匯入匯出”的總控中心。更接近真實情況的是：不同物件走不同入口，設定、流程模板和具體結果資料各自有自己的匯入匯出方式。

## 先怎麼理解這一頁

如果你要做匯入匯出，先回答三個問題：

1. 你要處理的是設定、流程模板，還是某個結果模組的資料？
2. 你需要的是全域性配置遷移，還是單個業務物件匯出？
3. 這個功能是不是本來就屬於某個具體視窗，而不是資料管理中心？

## 當前能明確證實的入口

### 設定匯入匯出

當前有明確選單入口：

- 工具 → 匯入匯出設定

這裡至少包含兩類動作：

- 匯出設定到 `.cvsettings`
- 從 `.cvsettings` 匯入設定

如果你的目標是遷移軟體配置，而不是遷移結果資料，優先從這裡開始。

### 流程模板匯入匯出

流程模板的匯入匯出不在資料管理頁統一處理，而是在流程設計器中單獨提供：

- 匯出當前流程
- 匯入流程
- 匯入模組

如果你要遷移流程內容，優先去 [流程設計](../workflow/design.md)。

### 模組內結果匯出

一些業務視窗會提供自己的匯出功能，例如：

- 流程節點分析視窗可匯出 CSV
- 部分外掛或影像/測量視窗有自己的 CSV 或影像匯出入口

這類匯出通常跟具體業務物件強繫結，不應該再被描述成統一的全域性“資料匯出中心”。

## 物件和入口對照

| 要交付的物件 | 優先入口 | 交付前確認 |
| --- | --- | --- |
| 軟體設定 | 工具 → 匯入匯出設定 | `.cvsettings` 能匯入，重啟後關鍵設定仍在 |
| 流程模板 | 流程設計器匯入/匯出 | 匯入後起始節點、裝置綁定、模板參數正確 |
| 資料庫記錄 | 資料庫瀏覽器或業務結果頁 | 能按 SN、時間、批次查到同一輪記錄 |
| CSV/Excel | 對應業務視窗或插件匯出 | 欄位順序、單位、PASS/FAIL、編碼符合客戶要求 |
| PDF/報告 | 專案視窗或插件自身報告入口 | 頁眉、客戶標識、結果截圖、判定項正確 |
| 圖片/overlay | 影像編輯器或結果視窗 | 原圖、ROI/POI、標註座標和檔名能對應 |
| Socket/MES 回應 | 專案視窗、SocketProtocol 或外部聯調工具 | 請求和回應樣例已保存，狀態碼和 Data 欄位正確 |

## 匯出交付前驗收

匯出功能最容易出現「按鈕能點，但交出去的內容不對」。交付前至少做一次端到端核對。

| 步驟 | 操作 | 通過標準 |
| --- | --- | --- |
| 1 | 用一組明確的 SN 或測試批次跑一次最小流程 | 後續查詢、匯出和外部回應都使用同一組標識 |
| 2 | 在資料庫或結果視窗確認源資料存在 | 源資料不是空表，也不是舊批次 |
| 3 | 從目標視窗執行匯出 | 檔案成功生成，路徑和檔名可解釋 |
| 4 | 打開匯出檔案核對欄位 | 欄位順序、單位、判定、時間、SN 與客戶要求一致 |
| 5 | 保存一組樣例檔案和截圖 | 後續版本升級後可按樣例複測 |

## 匯出失敗分層定位

| 現象 | 先查 | 再查 |
| --- | --- | --- |
| 找不到匯出按鈕 | 物件是否屬於設定、流程或某個業務視窗 | 插件/專案文件是否聲明支援匯出 |
| 檔案生成但為空 | 源資料是否已落庫，批次/SN 是否選對 | 匯出篩選條件和欄位映射 |
| 欄位缺失或順序不對 | 當前視窗匯出的物件是否正確 | 專案 exporter、客戶格式版本 |
| 圖片或 overlay 不對齊 | 結果圖和原圖是否同一輪 | ROI/POI 座標、圖像縮放或模板版本 |
| 外部系統收不到 | ColorVision 是否已完成並生成結果 | 協議、端口、專案 handler、回應欄位 |
| 遷移後行為變化 | 匯出的是否只是設定而不是結果資料 | 舊配置、流程模板、資料庫備份是否同步 |

## 常見使用順序

1. 先確認你要匯出的到底是什麼物件。
2. 如果是全域性設定，走“匯入匯出設定”。
3. 如果是流程內容，走 [流程設計](../workflow/design.md) 裡的匯入匯出。
4. 如果是結果資料或業務資料，先回到對應模組視窗找它自己的匯出入口。
5. 真正涉及庫內資料時，再配合 [資料庫操作](./database.md) 確認源資料範圍。

## 當前這頁不再承諾什麼

以下這些能力，當前使用者指南頁不再預設宣稱為統一可用能力，除非你已經在具體模組視窗裡看到了它：

- 統一 Excel 匯出中心
- 統一 JSON 匯出中心
- 統一 XML 匯出中心
- 統一 PDF 報告匯出中心
- 通用列對映匯入嚮導
- 通用批次資料夾匯入嚮導

如果某個外掛或視窗確實支援這些格式，應在那個模組自己的頁面說明，而不是放在這裡泛講。

## 常見問題

### 我想匯出資料，但找不到統一入口

- 先不要繼續找總選單
- 先確認物件屬於設定、流程還是某個業務結果視窗
- 再回到對應模組頁查詢匯出入口

### 匯出了設定，但業務結果沒有跟著遷移

- `.cvsettings` 主要用於配置遷移，不等於資料庫結果遷移
- 真正的結果資料要結合 [資料庫操作](./database.md) 或對應業務模組單獨處理

### 流程匯入成功了，但執行結果不對

- 先確認匯入的是不是正確流程版本
- 再回到 [流程執行與除錯](../workflow/execution.md) 驗證依賴裝置和模板
- 必要時重新檢查模組匯入和流程參數
