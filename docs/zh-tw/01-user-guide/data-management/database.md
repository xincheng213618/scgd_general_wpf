# 資料庫操作

當前最明確、最穩定的資料庫入口是“資料庫瀏覽器”。它適合做表級瀏覽、搜尋、分頁檢視，以及在滿足條件時做新增、修改、刪除。

## 什麼時候應該看這一頁

- 你想確認某批資料有沒有真的落庫
- 你要檢視 MySQL 或 SQLite 裡的實際表資料
- 你要按關鍵字搜尋記錄
- 你要做少量表資料維護
- 你在排查“資料寫入了沒”“為什麼列表裡看不到”這類問題

## 當前真實入口

資料庫瀏覽器當前有明確選單入口：

- 工具 → 資料庫瀏覽器

從當前實現看，它不是 SQL 編輯器，而是一個面向表瀏覽和基礎維護的視窗。

## 資料庫瀏覽器裡能做什麼

### 瀏覽資料來源、庫和表

左側樹會先顯示資料來源，再進入庫和表。當前預設能註冊和顯示的來源至少包括：

- MySQL
- SQLite 日誌庫

### 按表檢視資料

選中某個表後，右側會顯示：

- 表標題和來源
- 列資訊
- 資料網格
- 搜尋框
- 分頁控制元件
- 狀態列

### 搜尋和分頁

當前視窗支援按關鍵字搜尋，並在表資料較多時分頁瀏覽。遇到“明明有資料但沒看到”時，先查搜尋條件和頁碼，而不是直接判斷沒寫入。

### 基礎寫操作

如果當前資料來源和表都允許寫入，並且表有主鍵，介面會提供這些動作：

- 新增
- 儲存
- 撤銷
- 刪除
- 重新整理

如果表沒有主鍵，或者當前資料來源不允許寫入，就不要期待所有編輯按鈕都可用。

## 常見使用順序

1. 先從工具選單開啟資料庫瀏覽器。
2. 在左側展開正確的資料來源、庫和表。
3. 先看錶名、來源和記錄數，確認沒進錯庫。
4. 資料太多時，先用搜尋再翻頁。
5. 需要維護時，再確認當前表是否允許寫入。

## 使用時要特別注意什麼

### 當前視窗是表瀏覽器，不是萬能查詢臺

它適合看錶、篩記錄和做少量維護，但當前文件不再把它描述成統一 SQL 控制檯。若某個模組有專門查詢視窗，應優先使用那個模組自己的入口。

### 沒有主鍵的表不適合隨便改

當前實現會依賴主鍵決定修改和刪除的可靠性。沒有主鍵的表，更適合只讀瀏覽或謹慎新增。

### 寫入能力取決於資料來源和表

就算你已經開啟了資料庫瀏覽器，也不代表所有表都能寫。看到按鈕不可用時，先判斷是當前表不可寫，還是資料來源本身只讀。

## 常見問題

### 找不到想看的資料表

- 先確認當前展開的是正確的資料來源和庫
- 再重新整理資料來源樹
- 如果是 MySQL，確認連線配置和資料庫名是否正確

### 表能開啟，但資料為空

- 先確認當前是不是被搜尋條件過濾了
- 再確認頁碼和分頁大小
- 如果仍為空，回到 [流程執行與除錯](../workflow/execution.md) 或 [日誌檢視器](../interface/log-viewer.md) 看是否真的寫入成功

### 按鈕是灰的，不能編輯

- 先確認當前表是否允許寫入
- 再確認表是否存在主鍵
- 如果沒有主鍵，不要把“不可編輯”誤判為資料庫異常

### 修改後結果不對

- 先確認儲存是否成功
- 再重新整理當前表重新檢視
- 對關鍵業務資料，優先先查讀路徑是否正確，再做人工修改
