# 資料管理

本章節只回答三個問題：資料通常在哪裡看，什麼時候需要進資料庫頁，什麼時候應該走匯入匯出。

## 這一章適合什麼時候看

- 你想確認測試結果或業務資料存到了哪裡
- 你需要檢視、整理或清理資料庫內容
- 你需要做批次匯入或匯出
- 你在排查“資料沒儲存”“匯出失敗”“看不到歷史記錄”這類問題

## 本章節怎麼讀

### 先看資料庫

- [資料庫操作](./database.md)

適合的場景：

- 想確認資料有沒有真的落庫
- 想按記錄、時間或條件查結果
- 想排查連線、寫入或查詢問題

### 再看匯入匯出

- [資料匯出與匯入](./export-import.md)

適合的場景：

- 需要把結果交給外部系統或人工分析
- 需要遷移一批配置或資料
- 想確認當前支援的匯出格式和處理順序

## 常見使用順序

1. 先確認當前專案、裝置或流程已經正確執行完成。
2. 如果懷疑資料沒儲存，先進入 [資料庫操作](./database.md) 查記錄。
3. 只有在確認資料已經存在後，再進入 [資料匯出與匯入](./export-import.md) 做後續處理。
4. 如果匯出內容不對，先回頭核對源資料，而不是反覆重試匯出按鈕。

## 常見問題

### 看不到想要的歷史資料

- 先確認當前連線的資料庫例項是不是正確的
- 再確認篩選條件、時間範圍和專案物件是否選對
- 如果是剛執行完流程，先結合 [日誌檢視器](../interface/log-viewer.md) 看是否真的寫入成功

### 資料存在，但匯出結果不對

- 先確認匯出的物件是不是當前正在檢視的那批資料
- 再確認匯出格式是否符合你的後續使用場景
- 如果欄位缺失，優先回到源資料頁確認，而不是直接判斷匯出功能異常

### 不確定問題出在流程還是資料層

- 先看 [流程執行與除錯](../workflow/execution.md)
- 再用 [日誌檢視器](../interface/log-viewer.md) 確認失敗發生在執行、寫入還是匯出階段
