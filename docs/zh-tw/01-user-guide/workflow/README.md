# 工作流程

本章節面向日常使用，不解釋 FlowEngine 架構本身，而是幫助你判斷什麼時候該設計流程，什麼時候該去執行和除錯。

## 工作流程適合解決什麼問題

在 ColorVision 裡，工作流程通常用來把裝置動作、模板處理和結果輸出串成一條可重複執行的業務鏈路。

常見場景包括：

- 按固定步驟呼叫多個裝置或服務
- 複用一套檢測或處理順序
- 在執行失敗時快速定位卡在哪個節點
- 把人工操作儘量收斂成可儲存、可重複的流程

## 本章節怎麼讀

### 要搭流程時

- [流程設計](./design.md)

適合的場景：

- 新建一條流程
- 調整節點順序、連線關係或參數
- 想確認某一步應該放在哪個位置

### 要跑流程或排錯時

- [流程執行與除錯](./execution.md)

適合的場景：

- 手動執行已有流程
- 觀察執行狀態和結果
- 定位第一個失敗節點

## 建議的使用順序

1. 先明確要串起來的是裝置、模板還是資料處理步驟。
2. 進入 [流程設計](./design.md) 搭出最小可執行版本。
3. 再到 [流程執行與除錯](./execution.md) 驗證每一步是否真的按預期執行。
4. 如果流程涉及裝置服務，回頭核對對應裝置頁，而不是隻在流程視窗裡反覆重試。

## 常見問題

### 流程能開啟，但跑不通

- 先不要同時改很多節點
- 先定位第一個失敗節點
- 再去核對該節點依賴的裝置、模板或輸入資料

### 流程執行結果和預期不一致

- 先確認當前執行的是不是正確的流程版本
- 再確認節點參數和輸入物件是否對應當前專案
- 如果結果落庫或匯出異常，再聯動看 [資料管理](../data-management/README.md)

### 不確定問題出在流程還是裝置

- 先看 [流程執行與除錯](./execution.md)
- 再回到相關裝置專題頁確認服務狀態和配置
- 同時結合 [日誌檢視器](../interface/log-viewer.md) 縮小範圍
