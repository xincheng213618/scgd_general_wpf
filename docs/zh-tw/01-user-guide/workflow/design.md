# 流程設計

流程設計頁的重點不是抽象地介紹“什麼是視覺化編輯器”，而是幫助你把一條流程建出來、儲存下來，並且在改亂之後還能快速回到可執行狀態。

## 當前設計器裡能直接做什麼

從當前實現看，流程設計器至少支援這些常見動作：

- 新建流程
- 開啟已有流程模板
- 儲存流程
- 刪除流程
- 匯入流程
- 匯出流程
- 匯入模組
- 自動佈局
- 自動適配畫布
- 撤銷和重做
- 透過屬性面板編輯節點相關內容

## 常見設計順序

1. 先新建或開啟一條流程。
2. 把需要的節點放到畫布裡。
3. 連線節點關係，並在屬性面板裡補齊關鍵參數。
4. 先儲存一次，再做自動佈局或細調位置。
5. 改動較大後，再儲存一次並進入 [流程執行與除錯](./execution.md) 做驗證。

## 設計時最常用的幾個動作

### 新建、開啟和儲存

流程設計最容易出問題的不是“不會拖節點”，而是改了很久卻沒有儲存到正確模板。設計時優先先確認當前編輯的是哪條流程，再做儲存。

### 自動佈局和自動適配

當節點越來越多、畫布越來越亂時，先用自動佈局和自動適配把結構拉回可讀狀態，再繼續調參數，通常比純手工拖動穩得多。

### 撤銷和重做

當前實現裡有明確的撤銷和重做棧。做大改動時，不要怕先試；但試完發現方向不對，要儘快撤回，而不是在錯誤狀態上繼續補。

### 匯入、匯出和模組複用

如果你不是從零開始搭流程，優先先看有沒有可複用的流程或模組。匯入現成內容後再做小改，通常比完全手工重建更快，也更不容易漏參數。

## 設計時要特別注意什麼

### 畫布上的內容不等於已經儲存

你在設計器裡看見的內容，如果還沒儲存，就不一定是後續執行真正會跑到的版本。執行前一定先儲存。

### 選中的流程模板要和編輯目標一致

當前實現會維護當前選中的流程模板。如果選中物件和你以為的不一致，後面儲存、刪除、匯出都會落到錯誤物件上。

### 參數優先在屬性面板裡確認

節點連好以後，不要直接假設參數也對了。很多問題不是流程結構錯，而是節點參數仍然沿用了舊模板值。

## 常見問題

### 流程能開啟，但畫布是空的

- 先確認當前選中的模板是不是正確的
- 再確認該模板是否真的已有流程資料
- 如果模板本身還沒有內容，先建立對應流程再繼續

### 改了流程，但執行結果像沒變

- 先確認是否已經儲存
- 再確認執行頁選中的是不是同一條流程模板
- 如果匯入過模組或流程，檢查是不是改錯了物件

### 畫布太亂，已經看不清關係

- 先用自動佈局或自動適配
- 再把關鍵節點重新命名或重新組織
- 不要在看不清結構的狀態下直接進入執行排障

### 不確定問題在設計還是執行

- 如果連節點關係和參數都還沒確認，先留在設計頁整理
- 如果結構已經穩定，再去 [流程執行與除錯](./execution.md) 定位執行時問題
