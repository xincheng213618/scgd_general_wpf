# 流程執行與除錯

本頁只保留當前程式碼裡能證實的執行入口和排查路徑。對於大多數使用者，流程執行最關鍵的不是“高階除錯術語”，而是先確認選對了流程、服務是否線上、起始節點是否存在，以及失敗時怎麼縮小範圍。

## 當前可直接使用的執行入口

從當前實現看，流程執行至少有這兩類明確入口：

- 開始執行：`F6`
- 停止執行：`F7`

如果你看到舊文件裡寫的是 `F5`、`F10`、斷點或單步執行，請以當前介面和程式碼繫結為準，不要繼續按舊快捷鍵排查。

## 執行前先確認什麼

### 已選中有效流程模板

如果當前沒有選中有效模板，執行不會真正開始。先確認下拉框裡選中的就是你要跑的那條流程，而不是隻開啟了流程視窗。

### 流程裡存在起始節點

當前執行前會檢查起始節點。沒有起始節點時，流程會直接失敗，不會進入真正的執行階段。

### 註冊中心和服務令牌可用

當前實現會先檢查註冊中心連線和可用服務列表。服務令牌為空時，介面會先提示重新整理服務後再重試。

### 預處理透過

流程正式開始前還有一層預處理。預處理失敗時，流程會被取消，不會繼續往下跑。

## 常見執行順序

1. 先在 [流程設計](./design.md) 中確認當前流程內容和起始節點。
2. 在執行視窗選中正確的流程模板。
3. 確認相關裝置服務已經線上。
4. 按 `F6` 或點選執行入口開始執行。
5. 執行過程中觀察日誌區、當前執行節點和進度變化。
6. 需要中止時按 `F7` 或使用停止入口。

## 執行時你能看到什麼

### 當前執行節點

執行過程中，介面會持續顯示當前正在執行的節點名稱。如果一條流程看起來“卡住了”，先看停在哪個節點，而不是立刻懷疑整條流程。

### 執行進度和耗時

當前實現會記錄流程耗時、上次執行時間，並根據上次耗時估算當前進度。這個估算值適合快速判斷大致階段，但不應該當成嚴格的業務完成度。

### 結果和狀態

流程完成後，會寫入這次批次的狀態、耗時和結果摘要；停止執行時，會把狀態記為取消。

## 除錯時應該怎麼縮小範圍

### 先找第一個失敗節點

比起“整條流程失敗了”這種判斷，更有價值的是先找出第一個變紅或沒有繼續推進的節點，再回到對應裝置、模板或輸入資料去查。

### 先分清是沒啟動，還是中途失敗

常見兩類問題：

- 執行前就被攔住：模板未選中、起始節點缺失、註冊中心未連線、服務令牌為空、預處理失敗
- 已經開始執行，但中途失敗：某個節點返回錯誤、超時或訊息不匹配

### 日誌區優先於猜測

當前執行視窗會持續更新日誌文字。看到失敗時，先看最後停在哪個節點、前後是否出現了預處理失敗、取消執行或狀態訊息，再決定回哪一層排查。

## 流程執行交接要記錄什麼

流程交接不是只交一份流程檔。接手人員需要知道它依賴哪些裝置、模板、輸入、結果和外部系統。

| 記錄項 | 要寫清楚什麼 | 用途 |
| --- | --- | --- |
| 流程模板 | 模板名稱、版本、匯入來源、最後修改人 | 防止跑錯舊流程 |
| 起始條件 | 起始節點、SN/批次輸入、專案視窗或外部觸發方式 | 判斷流程為什麼沒有開始 |
| 裝置依賴 | 相機、電機、SMU、檔案服務等節點綁定 | 排查裝置層失敗 |
| 模板依賴 | 圖像處理模板、校準模板、判定閾值 | 排查結果偏差 |
| 資料去向 | 資料庫表、匯出檔案、圖片目錄、Socket/MES 回應 | 確認流程完成後結果在哪裡 |
| 失敗證據 | 第一個失敗節點、日誌時間點、錯誤訊息 | 讓後續人員能復現而不是重猜 |

## 最小複測腳本

現場複測或升級後，不要一開始就跑完整生產鏈路。先用下面順序確認流程能穩定進入和退出。

1. 打開流程設計頁，確認流程模板和起始節點存在。
2. 打開執行頁，選擇同一條流程模板。
3. 確認相關裝置服務在線，必要時先做一次裝置最小動作。
4. 準備一組不會影響正式產線的 SN、圖片或測試輸入。
5. 按 `F6` 執行，記錄開始時間、當前節點、最終狀態和耗時。
6. 查詢同一輪結果：日誌、圖像、資料庫、匯出檔案或外部回應至少要能對應上。
7. 如需停止，按 `F7` 後確認狀態被記錄為取消，而不是停在未知狀態。

## 失敗分叉表

| 失敗位置 | 典型表現 | 優先排查 |
| --- | --- | --- |
| 執行前 | 按執行沒有反應、提示刷新服務、起始節點缺失 | 註冊中心、服務列表、流程模板、起始節點 |
| 預處理 | 一開始就取消或提示預處理失敗 | 輸入參數、模板合法性、專案視窗上下文 |
| 裝置節點 | 等待超時、裝置無回應、返回碼異常 | 對應裝置頁、硬體連線、裝置 Code、MQTT/串口/IP |
| 模板節點 | 執行完成但結果不對 | 模板版本、閾值、圖片來源、校準資料 |
| 資料節點 | 流程完成但看不到結果 | 資料庫寫入、批次/SN、匯出對象、權限 |
| 外部系統 | ColorVision 完成但 MES/Socket 沒收到 | 協議配置、端口、專案 handler、回應欄位 |

## 常見問題

### 按了執行，但流程沒有開始

- 先確認註冊中心是否已連線
- 再確認服務列表是否已經重新整理出來
- 檢查是否選中了有效流程模板
- 檢查流程裡是否真的存在起始節點

### 流程開始了，但很快停止

- 先看是不是預處理失敗
- 再看日誌裡最後一個節點和對應狀態
- 如果是裝置相關節點，回到對應裝置頁確認連線和配置

### 進度不動，看起來像卡住

- 先看當前執行節點名稱
- 再判斷是不是某個節點正在等待裝置或訊息返回
- 必要時按 `F7` 停止後，先單獨驗證那個節點依賴的裝置服務

### 手動執行成功，流程裡卻失敗

- 先確認流程引用的是不是同一套裝置和模板
- 再確認執行前的環境是否和手動測試一致
- 需要時聯動看 [裝置服務概覽](../devices/overview.md) 和 [日誌檢視器](../interface/log-viewer.md)
