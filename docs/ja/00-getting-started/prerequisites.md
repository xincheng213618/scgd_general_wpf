# システム要件

このページでは、ColorVision をインストールして実行する前に準備する必要がある環境についてのみ説明し、インストーラーの実装、ウェアハウス構造、アーキテクチャ設計については詳しく説明しません。

## 適用可能なオブジェクト

- 既製のパッケージをインストールして使い始めたいエンドユーザー
- メインプログラムをソースからビルドして実行したい開発者
- インストール手順を読み続ける前に、システム条件が満たされているかどうかを確認したい同僚

## 動作環境要件

### オペレーティング システム

- Windows 10 バージョン 1903 以降、または Windows 11
- x64システム環境の使用を推奨します
- 初めてシステム サービスをインストール、アップグレード、または構成する場合は、管理者権限を持つアカウントを使用することをお勧めします。

### ハードウェアとディスプレイ

- 1920x1080 以上の解像度を使用することをお勧めします
- 画像処理とプロセス実行のために十分なメモリとディスク容量を確保することをお勧めします
・カメラ、分光器、モーター等を接続する場合は、対応するドライバーや通信環境が整っているか事前にご確認ください。

### ネットワークと権限

- 最初のインストール後にアップデートの確認、プラグイン マーケットへのアクセス、またはリモート サービスへの接続が必要な場合は、ネットワークが利用可能であることを確認してください。
・導入環境に厳しい制限がある場合は、プログラムディレクトリ、ログディレクトリ、ユーザードキュメントディレクトリに書き込み権限があることを事前に確認してください。

## インストールパッケージから実行する際に注意すべき点は何ですか?

- インストーラーは、独自のプロセスに従って必要なコンポーネントをチェックして展開します。前提条件が不足していることを示すメッセージが表示された場合は、インストーラーのプロンプトに従って前提条件を完了してください。
- 一部のサービス管理、デバイス ドライバー構成、またはシステム レベルの書き込み操作には、管理者権限が必要な場合があります。
- 環境内に古いバージョンがある場合は、新規インストールを行う前にアップグレードまたはアンインストールを完了することをお勧めします。

## ソースコードからビルドする場合に準備するもの

現在のウェアハウスは主に Windows WPF と x64 用です。以下の環境を準備することをお勧めします。

- .NET 8.0 SDK または .NET 10.0 SDK
- Visual Studio 2022 以降 (推奨)
- Git および PowerShell (リポジトリの取得とスクリプトの実行用)

メイン プログラムをビルドするには、x64 プラットフォームを使用することをお勧めします。


```powershell
dotnet restore
dotnet build .\ColorVision\ColorVision.csproj -p:Platform=x64
```


## 実行時の依存関係の説明 (ソース コード シナリオ)

インストール パッケージを介して展開するのではなく、ソース コードから直接実行している場合は、実行出力に次の依存関係が含まれている必要があることにも注意する必要があります。

- OpenCvSharp Windows ランタイム
- `DLL/CVCommCore.dll`
- `DLL/MQTTMessageLib.dll`
- `log4net.config` およびメイン プログラムに必要な関連リソース ファイル

これらのコンテンツは通常、プロジェクトの参照ルールとコピー ルールによって処理されます。プログラムは構築できても起動に失敗する場合は、まず出力ディレクトリに上記の依存関係がないかどうかを確認してください。
