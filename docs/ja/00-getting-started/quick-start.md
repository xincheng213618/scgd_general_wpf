# すぐに始めましょう

このページは、初めて ColorVision に触れる読者を対象としています。目標は、「プログラムの開始 -> インターフェイスの特定 -> 基本的な操作の実行 -> 次にどこを見るべきかを知る」という閉ループを最短パスで完了することです。

## 始める前に

最初に次の準備を完了することをお勧めします。

1. [システム要件](./prerequisites.md)を確認しました
2. [インストールガイド](./installation.md)の完了
3. メインプログラムが正常に起動できるようになりました

## 10分間の体験ルート

### ステップ 1: 起動し、メイン インターフェイスが正常であることを確認します。

- ColorVisionを開始する
・メインウィンドウが正常に表示できることを確認する
- 最初の起動が遅い場合は、初期化とプラグインのスキャンが完了するまで待ちます

まずインターフェイスの構造をよく知りたい場合は、[メイン ウィンドウ ガイド](../01-user-guide/interface/main-window.md) を直接読んでください。

### ステップ 2: いくつかの主要な領域を理解する

最初に次の領域に焦点を当てることをお勧めします。

- メニュー バー: ファイル、デバイス、プロセス、ヘルプなどの主要な機能を入力します。
- 中央ワークスペース: 画像、プロセス、結果を表示
- プロパティ パネル: 現在のオブジェクト パラメータを表示および変更します
- ステータスバーとログ：プログラムがエラーを報告しているかどうか、現在のステータスが正常であるかどうかを確認します

### ステップ 3: 最小限の操作を実行する

プログラムが利用可能であることを確認したいだけの場合は、次の 2 つの最も簡単なパスのいずれかを選択します。

#### パス A: 画像を開きます

1. ファイルメニューからローカルイメージを開きます
2. ワークスペースに画像が表示できることを確認する
3. プロパティ パネルまたは関連ツールが正常に応答するかどうかを確認します。

#### パス B: シミュレートされたデバイスまたは既存のデバイスを追加する

1. デバイス関連メニューに入ります
2. シミュレートされたデバイスを追加するか、既存のデバイスを接続します
3. デバイスがデバイス領域に表示され、正常に応答できることを確認します。

デバイス関連のコンテンツについては、引き続き [デバイス サービスの概要](../01-user-guide/devices/overview.md) を参照してください。

### ステップ 4: 実際のビジネスに近いエントリーを試す

目標に基づいてフォローアップ読書を選択してください。

- 画像を表示して注釈を付けたい場合: [画像エディターの概要](../01-user-guide/image-editor/overview.md) に移動します。
- デバイスとコレクション リンクを設定する場合: [デバイス サービスの概要](../01-user-guide/devices/overview.md) に移動します。
- プロセスがどのように設計され、実行されるかを理解し始めるには: [ワークフローの概要](../01-user-guide/workflow/README.md) に移動します。
