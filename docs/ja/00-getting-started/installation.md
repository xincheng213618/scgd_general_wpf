# インストールガイド

このページには、ユーザーが本当に必要とするインストール手順とインストール後の検証方法のみが記載されています。インストーラーの内部構造、パッケージ化スクリプト、および高度なインストーラー構成は、開発および展開のドキュメントに移動されました。

## インストール前に確認してください

始める前に、[システム要件](./prerequisites.md) を読み、次のことを確認してください。

- 現在の環境は Windows 10 1903 以降または Windows 11 です
- x64環境を使用する
- インストール ディレクトリへの書き込み権限を持っていること。サービス構成が関係する場合は、管理者権限を使用することをお勧めします
- オンラインアップデートやリモートサービスへのアクセスが必要な場合は、ネットワーク環境が利用可能です

## 方法 1: インストール パッケージを使用してインストールする

これはエンド ユーザーに推奨されるアプローチです。

### ステップ 1: インストール パッケージを入手する

- チーム配布チャネル、リリース ページ、または社内配布パッケージから ColorVision インストーラーを入手します。
- メインインストールパッケージと増分アップデートパッケージの両方が提供されている場合、初回インストールでは完全インストールパッケージを優先して使用してください。

### ステップ 2: インストーラーを起動する

- インストーラーをダブルクリックしてインストール ウィザードを開始します。
- システムに許可の確認がポップアップ表示された場合は、必要に応じてインストールを続行します。
- インストーラーが前提条件が欠落していることを示すプロンプトを表示した場合は、まずプロンプトに従ってインストールを完了してください。

### ステップ 3: インストール場所とコンポーネントの選択

- インストール ウィザードに従ってターゲット ディレクトリを選択します
- 必要に応じて、デスクトップ ショートカット、スタート メニュー エントリなどを作成するかどうかを決定します。
- 配信パッケージにプラグインまたはプロジェクト リソースが含まれている場合、多くの場合、デフォルトのインストール オプションを維持する方が安全です。

### ステップ 4: インストールを完了する

- ファイルのコピーと初期化が完了するまで待ちます
- 完了後、メインプログラムの起動を直接確認できます
- 環境内に古いバージョンが存在する場合、インストーラーは、続行する前に古いバージョンをアップグレード、上書き、またはアンインストールするように求めるプロンプトを表示する場合があります。

## 方法 2: ソース コードからビルドして実行する

これは、開発者とデバッグ シナリオのためのパスです。

### 最小ステップ


```powershell
dotnet restore
dotnet build .\ColorVision\ColorVision.csproj -p:Platform=x64
dotnet run --project .\ColorVision\ColorVision.csproj -p:Platform=x64
```


### 説明

- 現在のウェアハウスは主に Windows WPF と x64 用です。メイン プログラムをクロスプラットフォーム プロジェクトとして扱うことはお勧めできません。
- ビルドが成功したかどうかを確認したいだけの場合は、最初に `dotnet build` を実行し、次に実行とデバッグを開始します。
- 出力ディレクトリが実行中のプログラムによって占有されている場合は、最初に古いプロセスを閉じるか、一時出力ディレクトリを使用してビルドを確認します。

## インストール完了後の最小限の検証

インストール パッケージをインストールする場合でも、ソース コードを実行する場合でも、少なくとも次のチェックを行うことをお勧めします。

1. メインプログラムが正常に起動し、メインウィンドウが表示される
2. ログ ディレクトリまたはログ ウィンドウに明らかな起動レベルのエラーがない。
3. プラグインのスキャンが完了すると、ヘルプまたはプラグイン関連のメニューを通常どおり開くことができます。
4. 画像を開いて、メイン インターフェイスの基本的な操作が正常であることを確認できます。

これらの手順が正常であれば、基本的にインストール リンクが利用可能であり、[初回実行ガイド](./first-steps.md) と [クイック スタート](./quick-start.md) を読み続けることができることを意味します。

## 一般的なインストールの問題

### インストーラーが起動できません

- まず、インストール パッケージが破損していないことを確認し、必要に応じてインストール パッケージを再取得します。
- 現在のシステムバージョンが[システム要件](./prerequisites.md)を満たしているか確認します。
- システム ポリシーによってブロックされている場合は、十分な権限を持つアカウントを使用して再実行してください。

### インストールは完了しましたが、プログラムの起動に失敗しました

- 最初にログまたはエラーメッセージを確認してください
- インストール ディレクトリとユーザー ドキュメント ディレクトリに読み取りおよび書き込み権限があることを確認します。
- ソース コードから実行する場合は、まず出力ディレクトリに実行時依存関係ファイルがないかどうかを確認します。

### アップグレード後の異常な動作

- 設定が残っているか、古いバージョンが占有しているファイルがないかを優先して確認します
- 必要に応じて、メイン プログラムおよび関連するバックグラウンド サービスを閉じ、再インストールまたは再起動します。

## さらに深く知りたいときはどこに行くべきか

- 初回起動後の基本操作を確認するには：[初回実行ガイド](./first-steps.md)を参照してください。
- 最小限の閉ループ エクスペリエンスをすばやく完了したい場合: [クイック スタート](./quick-start.md) に移動します。
- インストーラー、アップデーター、およびデリバリー チェーンの実装を確認するには: [展開の概要](../02-developer-guide/deployment/overview.md) に移動します。
