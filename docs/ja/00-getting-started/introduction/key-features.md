## ここに直接の能力リストがないのはなぜですか?

- 古い機能リストには、現在のリポジトリから直接確認することが困難な記述が混在していました
- 一部の機能は、特定のプロジェクト、プラグイン、または配信環境でのみ確立されており、一般的なエントリーコミットメントとしては適していません。
- 現在の構成では、「インストールと最初の使用」、「日常使用」、「設計とアーキテクチャ」、および「API リファレンス」が分離されています。読者はこれらの入り口をたどって入ってください。

## 現在の判断原理

現在のトピック ページ、ソース コード、または現在のプラグイン ディレクトリで特定の機能が検証されていない場合は、現在のドキュメントとソース コードが古い機能リストよりも優先される必要があります。
