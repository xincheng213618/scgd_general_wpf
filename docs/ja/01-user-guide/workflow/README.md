# ワークフロー

この章は日常的に使用するものです。 FlowEngine アーキテクチャ自体については説明しませんが、プロセスをいつ設計するか、いつ実行およびデバッグするかを決定するのに役立ちます。

## ワークフローはどのような問題を解決するのに適していますか?

ColorVision では、通常、ワークフローを使用して、デバイスのアクション、テンプレートの処理、および結果の出力を反復可能なビジネス リンクに結び付けます。

一般的なシナリオには次のようなものがあります。

- 複数のデバイスまたはサービスを一定の手順で呼び出します
- 一連の検出または処理シーケンスを再利用する
- 実行が失敗したときにどのノードがスタックしているかを素早く特定します
- 手動操作を可能な限り保存可能で反復可能なプロセスに統合します。

## この章の読み方

### 手続きをしたいとき

- [プロセス設計](./design.md)

適切なシナリオ:

- 新しいプロセスを作成する
- ノードの順序、接続関係、パラメータを調整します
- 段差の位置を確認したい

### プロセスの実行またはトラブルシューティングを行う場合

- [プロセスの実行とデバッグ](./execution.md)

適切なシナリオ:

- 既存のプロセスを手動で実行する
- 実行状況と結果を観察する
- 最初に障害が発生したノードを特定する

## 推奨される使用順序

1. まず、機器、テンプレート、またはデータ処理ステップをまとめるかどうかを明確にします。
2. [Process Design](./design.md) と入力して、実行可能な最小バージョンをビルドします。
3. [プロセスの実行とデバッグ](./execution.md) に移動して、各ステップが実際に期待どおりに実行されるかどうかを確認します。
4. プロセスにデバイス サービスが含まれる場合は、プロセス ウィンドウで再試行するのではなく、戻って対応するデバイス ページを確認します。

## よくある質問

### プロセスを開くことはできますが、実行することはできません。

- 最初は多くのノードを同時に変更しないでください。
- 最初に障害が発生したノードを最初に特定します
- 次に、ノードが依存するデバイス、テンプレート、または入力データを確認します。

### プロセスの実行結果が期待と一致しません

- まず、正しいプロセス バージョンが現在実行されているかどうかを確認します。
- ノードパラメータと入力オブジェクトが現在のプロジェクトに対応しているか再確認します
- 結果がライブラリにドロップされるか、異常にエクスポートされる場合は、[データ管理](../data-management/README.md) にリンクします。

### 問題がプロセスにあるのか装置にあるのかわからない

- まずは[プロセスの実行とデバッグ](./execution.md)を見てください。
- 関連するデバイスのページに戻り、サービスのステータスと構成を確認します。
- [ログビューア](../interface/log-viewer.md)と組み合わせて範囲を絞ることもできます
