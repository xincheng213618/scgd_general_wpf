# プロセス設計

プロセス設計ページの焦点は、「ビジュアル エディターとは何か」を抽象的に紹介することではなく、プロセスを構築し、保存し、変更後にすぐに実行可能な状態に戻すのに役立つことです。

## 現在のデザイナーで直接何ができるのでしょうか?

現在の実装では、プロセス設計者は少なくとも次の一般的なアクションをサポートしています。

- 新しいプロセスを作成する
- 既存のプロセス テンプレートを開く
- 保存処理
- 削除処理
- 輸入プロセス
- 輸出プロセス
- インポートモジュール
- 自動レイアウト
- 自動的にキャンバスに適応します
- 元に戻すとやり直し
- プロパティ パネルを使用してノード関連のコンテンツを編集します

## 一般的な設計順序

1. まずプロセスを作成するか、開きます。
2. 必要なノードをキャンバスに配置します。
3. ノードの関係を接続し、プロパティ パネルで主要なパラメータを入力します。
4. 自動レイアウトや位置の微調整を行う前に、一度保存してください。
5. 大幅な変更を行った後、再度保存し、確認のために[プロセスの実行とデバッグ](./execution.md)を入力します。

## 設計で最も一般的に使用されるアクションのいくつか

### 新規作成、開いて保存

プロセス設計における最も一般的な問題は、「ノードをドラッグできない」ことではなく、正しいテンプレートを長期間変更した後に保存できないことです。設計する際は、現在どの工程を編集しているのかを確認し、保存してください。

### 自動レイアウトと自動適応

ノードが増えてキャンバスがますます乱雑になった場合は、まず自動レイアウトと自動適応を使用して構造を読みやすい状態に戻してから、パラメータの調整を続けます。通常、これは手動でドラッグするよりもはるかに安定しています。

### 元に戻すとやり直す

現在の実装には、明示的な元に戻すスタックとやり直しスタックがあります。大きな変更を加える場合は、恐れずに最初に試してみてください。しかし、試してみて方向が間違っているとわかった場合は、間違った状態のまま補い続けるのではなく、できるだけ早く撤退する必要があります。

### インポート、エクスポート、モジュールの再利用

プロセスを最初から構築していない場合は、まず、再利用可能なプロセスまたはモジュールがあるかどうかを確認します。既製のコンテンツをインポートして小さな変更を加えるほうが、通常、完全に手動で再構築するよりも速く、パラメータを見逃す可能性が低くなります。

## 設計時に特に注意すべきことは何ですか?

### キャンバス上のコンテンツは保存されたコンテンツと等しくありません

デザイナーに表示されるコンテンツは、保存されていない場合、以降の実行で実際に実行されるバージョンであるとは限りません。実行する前に必ず保存してください。

### 選択したプロセステンプレートは編集対象と一致している必要があります

現在の実装では、現在選択されているプロセス テンプレートが維持されます。選択したオブジェクトが自分の考えと一致しない場合、その後の保存、削除、エクスポートでは間違ったオブジェクトが作成されてしまいます。

### パラメータはまずプロパティパネルで確認します。

ノードが接続された後は、パラメーターが正しいと直接想定しないでください。多くの問題は、プロセス構造が間違っていることではなく、ノード パラメーターが依然として古いテンプレート値を使用していることです。

## よくある質問

### プロセスは開くことができますが、キャンバスは空です

- まず、現在選択されているテンプレートが正しいかどうかを確認してください
- テンプレートに実際にプロセスデータが存在するかを再度確認する
- テンプレート自体にコンテンツがない場合は、続行する前に対応するプロセスを作成します。

### 処理は変更されましたが、実行結果は変わらないようです。

- まず保存されているかどうかを確認してください
・実行ページで同じプロセステンプレートが選択されているか再確認
・ モジュールやプロセスをインポートした場合は、間違ったオブジェクトが変更されていないか確認してください。

### キャンバスがごちゃごちゃしていて、関係性がよくわかりません。

- 最初に自動レイアウトまたは自動適応を使用します
- キーノードの名前変更または再編成
- 構造を明確に理解せずに、いきなりトラブルシューティングを始めないでください。

### 問題が設計にあるのか実行にあるのかわからない

- ノードの関係とパラメータをまだ確認していない場合は、設計ページに留まってそれらを整理します。
- 構造が安定している場合は、[プロセスの実行とデバッグ](./execution.md) に移動して実行時の問題を特定します。
