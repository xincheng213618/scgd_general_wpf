# よくある質問

このページはコード分析ではなく、トラブルシューティング ポータルです。問題が発生した場合は、まず問題がどのカテゴリに属する​​かを判断し、該当するページにジャンプして範囲を絞り込みます。

## まずはこの順番で確認してみましょう。

1. まず[ログビューア](../interface/log-viewer.md)を見て、起動、デバイス、プロセス、プラグインのいずれの段階でエラーが発生しているかを確認します。
2. 現在のプロジェクト、デバイス インスタンス、テンプレート、またはウィンドウが、想定しているオブジェクトであるかどうかを再度確認します。
3. パス、ライセンス、データベース接続、サービスステータスなどの外部条件を確認します。
4. 最後に、ウィンドウを再度開くか、構成を再保存するか、関連サービスを再起動する必要があるかを検討します。

## 一般的なシナリオ

### メインウィンドウに関数が見つかりません

- まず、[メイン ウィンドウ ナビゲーション](../interface/main-window.md) に戻ります。
- 検索ボックスを使用して入り口を見つけることを優先します
・プラグイン機能の場合、プラグインが正しく読み込まれているか確認してください。

### デバイスは存在しますが、動作しません

- まず [デバイス サービスの概要](../devices/overview.md) を確認します。
- カメラ シーンの監視を続けます [カメラ サービス](../devices/camera.md)
- バインディングまたはハードウェア アクセスの問題の場合は、[物理カメラ管理](../devices/camera-management.md) を参照してください。

### パラメーターは変更されていますが、エフェクトは変わりません。

- まず、現在使用しているオブジェクトを変更していることを確認してください。
- 保存され、該当するウィンドウまたはサービスがリロードされているかを再度確認してください
- プロパティとテンプレートに関連するシナリオの場合は、まず [プロパティ エディタ](../interface/property-editor.md) または対応するデバイス パラメータ ページに戻って確認してください。

### プロセスの実行に失敗しました

- まず、[プロセスの実行とデバッグ](../workflow/execution.md) を確認します。
- ログ内のノード名、テンプレート名、障害キーワードの検索を優先します。
- 最初からプロセス全体を疑わず、最初に障害が発生したノードまで縮小します。

### JSON またはテンプレートのコンテンツの検証に失敗しました

- まずエディターのチェックボタンを使用して形式に問題があるかどうかを確認します
- コメント テキスト、説明テキスト、または古いテンプレートのコンテンツが誤って JSON に貼り付けられていないか、もう一度確認してください。
- 別の編集モードに切り替える場合は、現在保存されている JSON に戻ってから判断してください。

### ログが多すぎて要点が見えません。

- まずログ レベルを `Warn` または `Error` に切り替えます。
- キーワードを使用して範囲を絞り込む
- 検索ボックスが赤くなったら、まず通常のキーワードをあきらめて、一般的なキーワードに戻ります

## それでも見つからない場合

- 問題が発生した時刻を記録します
- 現在のプロジェクト、デバイス名、テンプレート名、またはプラグイン名をメモします。
- 対応するログフラグメントを添付してトラブルシューティングを続行すると、効率が大幅に向上します。
