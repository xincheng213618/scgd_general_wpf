# ログビューア

ログ ビューアは、起動例外、デバイス通信の問題、プロセス実行エラー、プラグインの読み込みエラーを見つけるために使用されます。 「インターフェイスが応答しない、機能が有効にならない、デバイスに接続できない」などの問題が発生した場合、通常は、やみくもに推測するよりも、まずログを確認した方が早いです。

## いつオンにする必要がありますか?

- メインウィンドウまたは特定の機能が開かない
- デバイスの接続状態の異常
- プロセスの実行中にエラーが報告されるが、インターフェイスのプロンプトが十分に明確ではない
- プラグイン、サービス、またはテンプレートの異常な読み込み動作

## よく使う関数

- 新しいログをリアルタイムで表示
- 検索キーワード
- ログレベルによるフィルタリング
- 一定範囲の履歴ログを表示します
- 自動スクロールと手動読み取りを切り替えます

## 基本的な使い方

1. ヘルプ メニューまたは対応するエントリからログ ウィンドウを開きます。
2. まず最新のログを読み込み、エラーが発生したおおよその時刻を確認します。
3. キーワードを使用して、モジュール名、デバイス名、例外キーワード、またはファイル名を検索します。
4. ログが多すぎる場合は、レベルを `Warn` または `Error` に上げて範囲を狭めます。

## 検索をより効率的に行う方法

- デバイス名、テンプレート名、`error`、`timeout` などの簡単なキーワードを最初に使用します。
- 検索結果が多すぎる場合は、複数のキーワードを組み合わせてください
- 正規表現が必要であることが確実な場合にのみ使用してください

入力後に検索ボックスが赤くなった場合は、通常、正規表現が正しく書かれていないことを意味します。この場合、最初に共通キーワード検索に戻る方が安定します。

## よくある質問

### ログウィンドウを開いた後はほとんど内容がありません

- まず、自動更新がオンになっているかどうかを確認します。
- 現在のログレベルが高すぎないかを再度確認してください
- 新しいログのみを表示したい場合は、現在の読み込み戦略が空でないことを確認してください。

### ログが大量にあり、インターフェイスが読みにくいです。

- まずレベルを `Warn` または `Error` に切り替えます。
- キーワードを使用して範囲を絞り込む
- 単なる一時的なトラブルシューティングの場合は、まだすべての履歴ログを保持することを追求しないでください。

### 間違った検索結果

- まず、複数のキーワードを同時に一致させる必要があるかどうかを確認します。
・ 正規表現を使用する場合は、特殊文字をエスケープする必要があるかどうかを確認してください。
- 検索が見つからない場合は、フィルターなしのビューに戻り、時点を再度検索します。

## トラブルシューティングの提案

### 機器の問題

- デバイス名、カメラコード、サービス名を最初に検索します
- 次に、[デバイス サービスの概要](../devices/overview.md) を参照するためのリンクを作成します。

### プロセスの問題

- テンプレート名、ノード名、または実行失敗キーワードを最初に検索します
- 次に、リンクして [プロセスの実行とデバッグ](../workflow/execution.md) を確認します。

### プラグインまたは起動の問題

- まず、アプリケーションの起動フェーズで最新のログを確認します。
- DLL、バージョン、パス、または読み込みエラーのプロンプトがあるかどうかに注意してください。
