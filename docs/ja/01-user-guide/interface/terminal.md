# 端子

ターミナル パネルは、スクリプトの実行とコマンド ライン出力の直接観察の 2 種類の用途に適しています。これは、独立した開発ドキュメントの入り口というよりも、メイン プログラム内の作業補助領域に似ています。

## いつオンにする必要がありますか?

- PowerShell または CMD コマンドを直接実行する必要があります
- 現在のプロジェクト コンテキストでスクリプトを実行したい
- スクリプト出力、エラー報告、または対話型プロンプトを観察する必要がある
- メインプログラムから切り替えずにコマンドラインを簡単にチェックしたい

## 通常はどのように使用しますか?

### シェルを選択

端末は通常、少なくとも 2 つの共通シェルをサポートします。

- パワーシェル
-CMD

現在のコマンドが PowerShell 構文に依存している場合は、CMD で直接再試行しないでください。実行する前にシェルの種類を確認してください。

### スクリプトを実行する

一般的な方法には次のようなものがあります。

- コマンドを直接入力してスクリプトを実行します
- リソース管理または関連ポータルから「ターミナルで実行」をトリガーします
- トラブルシューティングを行う場合は、最初に最小限のコマンドを実行し、その後徐々にパラメータを追加してください。

### 出力とエラーレポートを見てください。

- まず最初のエラーを確認してください
- 最後の行だけに注目しないでください
- スクリプトが出力を続ける場合は、停止していると考えるのではなく、まずスクリプトがまだ実行中であることを確認してください。

## 推奨される使用順序

1. まず、PowerShell を使用するか CMD を使用するかを確認します。
2. 現在の作業ディレクトリが予期したプロジェクトの場所であるかどうかを確認します。
3. まずは最低限のコマンドを実行し、環境、パス、権限が正常であることを確認します。
4. コマンドが失敗した場合は、まず[ログビューア](./log-viewer.md)とターミナル出力で判断します。

## よくある質問

### コマンドは実行できますが、結果は期待と一致しません。

- まず現在のシェルの種類を確認します。
- カレントディレクトリ、環境変数、スクリプトパラメータを再度確認してください
- スクリプトが外部プログラムに依存している場合は、まずそれらのプログラムが実際に利用可能かどうかを確認してください。

### 端末に出力がないか、応答していないようです。

- まず、コマンドが入力待ち状態であるかどうかを確認します。
- 間違ったシェルに切り替えていないか再度確認してください
- 長いタスクの場合は、結果の最初のバッチを出力するまでにしばらく時間を与えます。コマンドをすぐに再実行依頼しないでください。

### スクリプトの実行時にパスまたは権限エラーが発生する

- まずパスが正しいかどうかを確認してください
- スクリプトファイルおよび依存ファイルが存在するか再度確認してください
- PowerShell スクリプトを使用する場合は、最初に実行戦略と呼び出しメソッドが正しいかどうかを確認してください

### 対話型コマンドが異常に動作する

- 新しい端末セッションを作成して再試行してください
- 対話型コマンド自体に問題があるかどうかを確認するために、より単純なコマンドを再度使用してみてください。
- 結果だけを確認したい場合は、端末内ですべての操作を完了する必要はありません
