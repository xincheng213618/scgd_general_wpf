# メインウィンドウのナビゲーション

メイン ウィンドウは ColorVision のワークベンチです。機器、プロセス、画像表示、ログ、検索、ステータス監視など、日常業務のほとんどがここに入力されます。

## メインウィンドウに入った後、最初に見る場所

### トップメニューと検索エリア

- メニューはメイン機能の入り口です
- コマンド、ページ、オブジェクトをすばやく見つけるための検索ボックス
- 機能がどこに隠されているかわからない場合は、まず検索を使用してください

### 左側のナビゲーション領域

- プロジェクト、リソース ツリー、または関連するビュー タブを表示するためによく使用されます。
- 現在のページに多くのコンテンツがある場合は、通常、最初に左側の構造から始める方が速いです。

### 中間メインワークスペース

- 現在操作中のコアコンテンツを表示
- 画像ビュー、プロセス ウィンドウ、テンプレート ウィンドウ、またはその他のビジネス インターフェイスの場合があります
- 現在の作業モードに応じて、レイアウトが単一ビューまたは複数ビューに切り替わる場合があります。

### 下部ステータスバー

- システムステータス、プロンプト情報、およびいくつかのショートカットエントリを観察するために使用されます
- デバイス、サービス、またはプラグインのステータスが異常な場合、まずここでプロンプトが表示されるかどうかに注意してください。

## メイン ウィンドウで通常何をしますか?

- デバイス関連のウィンドウを開く
- プロセス設計と実行ページに入る
- 画像または結果ウィンドウの表示
- ログ、ターミナル、設定インターフェイスを開く
- 検索機能の入り口またはオブジェクトの場所

## 推奨される習熟順序

1. まず、メイン メニュー、検索エリア、ステータス バーについて理解します。
2. 左側のナビゲーションとメイン ワークスペースがどのように連携するかを理解します。
3. 次に、[プロパティ エディター](./property-editor.md) と [イメージ エディターの概要](../image-editor/overview.md) を入力します。
4. ハードウェアに関しては、[デバイス サービスの概要](../devices/overview.md) を引き続き参照してください。
5. 自動化が必要な場合は、[ワークフロー](../workflow/README.md)を入力します。

## よくある質問

### 特定の関数の入り口が見つかりません

- 最初に検索ボックスを使用してキーワードを検索します
- 該当する章のホームページに戻り、再度入力してください
- プラグイン機能の場合、プラグインがロードされているかどうかを確認します

### 開いているウィンドウが多すぎて、何を見ているのかわかりません。

- 左側のナビゲーションに戻り、現在のオブジェクトを再確認します
- メインのワークスペースのタイトルとステータス バーの情報を観察します。
- 複数のウィンドウをやみくもに切り替えるのではなく、業務に応じて該当するページに再入力します

### 予期したプロンプトがステータス バーに表示されません

- まず、関連するサービス、デバイス、またはプラグインが実際に開始されているかどうかを確認します。
- より具体的な情報については、[ログビューア](./log-viewer.md)を再度確認してください。
