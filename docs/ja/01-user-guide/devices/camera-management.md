#物理カメラの管理

このページでは、物理カメラ層の役割について説明します。これは、カメラ サービスを直接置き換えるのではなく、実際のカメラの検出、承認、保守を担当する「ハードウェア リソース台帳」に似ています。

## 最初にいつここに来るべきですか?

- 新しいカメラを初めて接続する場合
- ライセンスのインポートまたは更新
- 作成可能なカメラIDを更新
- 校正リソース、グループ化、物理構成を管理します
- 論理カメラ サービスがバインド可能なオブジェクトを見つけることができません

## 物理カメラ層は何を担当しますか?

- 物理的なカメラオブジェクトの作成
- `.lic` または `.zip` ライセンスをインポートします
- 物理カメラ構成とリソースツリーを維持する
- バインド可能なカメラ コードとカメラ サービスの基本パラメータを提供します

つまり、「ハードウェアが実際にシステムに認識されているかどうか」や「ライセンスが有効かどうか」などの問題は、通常は最初にこのレベルでチェックされる必要があります。

## 共通の操作シーケンス

1. デバイスを接続し、システムがアイドル状態のカメラ ID を認識できることを確認します。
2. 物理カメラ管理でカメラを作成するか、ライセンスをインポートします。
3. カメラ名、コード、モード、基本構成を確認します。
4. 必要に応じて、キャリブレーション、グループ化、および関連リソースを準備します。
5. [カメラ サービス](./camera.md) に戻り、論理サービスをこの物理カメラにバインドします。

## 設定中に何を確認する必要がありますか?

- ライセンスは有効ですか?
- 作成されていないカメラ ID は本当に存在しますか?
- 物理的なカメラリソースが完了しているかどうか
- カメラモードは実際のハードウェアと一致していますか?
- 変更後、論理カメラ サービスに戻って再保存または更新する必要がありますか?

## よくある質問

### 未作成のカメラが見つかりません

- ハードウェアが接続され、システムによって認識されていることを確認します。
- まずデバイス ID を更新してから、再試行してください
- デバイスを抜き差ししたばかりの場合は、ウィンドウを再度開いて再度確認してください。

### ライセンスをインポートした後も利用できない

- インポートされたファイル形式が正しいことを確認します
- ライセンスに対応するカメラ コードが現在のハードウェアと一致していることを確認します。
- データベースまたはローカルリソースが同期されているかどうかを確認します

### 物理的なカメラ構成が変更され、既存のカメラ サービスが異常に動作します。

- 論理サービスを再保存または再バインドする必要があるかどうかを確認します。
- 構成変更後に関連サービスを再起動する必要があるかどうかに注意してください
- パスとリソースの調整が関係する場合は、関連するファイルが新しい場所に移動されていることを確認してください。
