# モーターサービス

モーター サービスは、モーター機器を ColorVision の統合デバイス システムに組み込むために使用され、開閉、移動、原点復帰、現在位置の読み取りなどの共通の制御アクションを提供します。手動デバッグと自動プロセスの両方で位置制御が必要なシナリオでの使用に適しています。

## このページはいつ読むべきですか?

- モーターサービスを追加または構成したい場合
- モーターを指定された位置に手動で移動する必要があります
- 現在地が更新されているか確認する必要があります
- 「モーターは見えるが動かない」または「位置が更新されない」問題を解決する必要がある

## 一般的な使用シーケンス

1. まず、モーター サービスを作成し、デバイス リストに保存します。
2. 通信構成とタイムアウトパラメータが適切かどうかを確認します。
3. 最初にオープン動作を実行して、デバイスが動作可能な状態であることを確認します。
4. その後、移動、原点復帰、位置読み取りなどの動作を行います。
5. 完了後、業務上デバイスのシャットダウンが必要な場合は、再度シャットダウン操作を実行します。

## 本サービスでの共通操作

### 開閉

後続のアクションの多くは、デバイスがすでに動作していることを前提としています。 「コマンドを送信したが応答がない」場合は、移動コマンドを直接疑うのではなく、まずコマンドが正常にオープンされたかどうかを確認してください。

### 指定した場所に移動します

以下のパラメータに注意するのが一般的です。

- ターゲットの場所
- 絶対的または相対的な移動
- タイムアウト期間

デバッグ段階の場合は、最初は小さな可動範囲で方向や動作を確認し、その後、大きなステップ サイズで調整します。

### 原点回帰

現在位置が信頼できない場合、または統一された出発点を再確立する必要がある場合は、原点に戻ることを優先してその後のアクションを続行します。

### 現在の場所を取得する

現在位置は推測できないので、率先して読み返して確認する必要があります。特に自動化されたプロセスや複数の手動移動の後では、最初に位置を読み取ってから次のステップを実行する方が安定することがよくあります。

## よくある質問

### モーター サービスは存在しますが、開くことができません。

- 最初に通信設定が正しいかどうかを確認してください
- デバイスがオンラインであるかどうかを再度確認します
- オープンアクションが失敗した場合、通常、その後の動きと位置の読み取りは信頼できません。

### 移動コマンドが送信されましたが、位置は変更されませんでした。

- まずデバイスが正常に開かれたかどうかを確認します
- 目標位置、絶対/相対モード、タイムアウトを再確認します。
- インターフェイスが更新されたかどうかを確認するだけでなく、操作後に現在位置を積極的に読み取ります

### 現在地が正しく表示されない

- もう一度実行して、最初に場所を取得します
- 最後の移動が失敗したが、インターフェースが古い値を保持しているかどうかを再度確認します。
・原点復帰を伴う場合は、実際に原点復帰が完了しているか確認してください。

### プロセス内でモーター アクションの呼び出しに失敗しました

- まず、モーターが開いて正常に動くかどうかを手動で確認します。
- プロセス内のパラメータが手動テストと一致しているかどうかを再度確認します。
- 必要に応じて、[プロセスの実行とデバッグ](../workflow/execution.md) および [ログ ビューア](../interface/log-viewer.md) を表示するためのリンクを作成します。
