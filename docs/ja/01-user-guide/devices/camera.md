#カメラサービス

このページでは、日常使用における「カメラ サービス」レイヤーの役割と、物理カメラ、パラメーター構成、およびキャリブレーション テンプレートとの関係について説明します。

## カメラ サービスと物理カメラは同じものではありません

- 物理カメラ: 実際のハードウェア、ライセンス、リソース、および基本機能
- カメラサービス: プロジェクト内のデバイスリスト、ビューウィンドウ、プロセスによって実際に呼び出される論理デバイス
- 通常、画像キャプチャ、ビデオ、キャリブレーション、テンプレート機能を安定して使用するには、カメラ サービスを物理カメラにバインドする必要があります。

## いつもここで何をしますか?

- カメラサービス構成の編集
- カメラビューまたはライブフィードを開く
- 物理カメラのバインドまたは切り替え
- 自動露出、自動フォーカス、およびカメラパラメータテンプレートを入力します
- キャリブレーションパラメータ編集ウィンドウを開く
- ローカル デバッグ ポータルとカメラ ログの表示

## 推奨される使用順序

1. まず、[物理カメラ管理](./camera-management.md) でハードウェア、ライセンス、リソースが利用可能であることを確認します。
2. カメラ サービスに戻り、現在のサービス インスタンスを正しい物理カメラにバインドします。
3. [カメラパラメータ設定](./camera-configuration.md)で露出、ゲイン、チャンネル、モードを調整します。
4. 必要に応じて、自動露出、自動フォーカス、またはキャリブレーション テンプレートを入力します。
5. リアルタイム画面、ローカルウィンドウ、またはプロセスを使用して検証結果を実行します。

## このページに対応する能力範囲

現在のコードでは、カメラ サービス レイヤーにすでにいくつかの共通の入り口が含まれています。

- カメラ設定の編集
- デバイス ID と物理カメラのバインドを更新します
- 自動露出テンプレート
- オートフォーカステンプレート
- カメラパラメータテンプレート
- キャリブレーションテンプレート
- カメラログとローカルウィンドウ

したがって、「カメラがシステムで正常に使用できるかどうか」という問題に遭遇した場合、通常は、基盤となるカメラ リソースを直接調べるのではなく、カメラ サービスを最初に調べます。

## よくある質問

### カメラ サービスは存在しますが、開くことができないか、画像をキャプチャできません。

- まず物理カメラがバインドされているかどうかを確認します。
- ライセンス、接続ステータス、カメラログを再確認します
- 構成を変更したばかりの場合は、関連するウィンドウを再度開くか、サービスを再起動して再試行してください。

### 物理カメラ構成を変更した後、カメラ サービスが同期されません。

- 保存後にカメラサービス設定を再入力します
- 現在のサービスが予想される物理カメラにバインドされているかどうかを確認します
- 必要に応じてデバイス ID を更新するか、再バインドします。

### ビデオまたはローカル モードの例外

- まず現在のビデオまたはリアルタイム モードを閉じてから、再接続または再起動します。
- 接続の問題かパラメータの問題かを優先します
- 必要に応じてカメラログをチェックしてバックグラウンドサービスのステータスを確認します
