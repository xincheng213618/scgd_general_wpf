# カメラパラメータの設定

このページでは、「カメラ サービス設定ページ」で通常変更する必要がある内容について説明します。ここで調整されるのは論理カメラ サービス パラメータであり、デフォルト値の一部はバインドされた物理カメラから取得されます。

## 共通の設定内容

### 結合情報

- 現在のサービスインスタンス名
-バインドされた物理カメラまたは CameraCode
- 現在のプロジェクトで実際に呼び出されるカメラ サービス

### コレクションパラメータ

- チャンネルとビット深度
- 撮影モード
- 露出、ゲイン、彩度
- 自動露出スイッチと関連パラメータ

### 拡張パラメータ

- ビデオ関連の設定
- ファイルのキャッシュと保存形式
- モーター、CFW、オートフォーカス、その他のスレーブ構成

## 順序を調整することをお勧めします

1. まず、正しい物理カメラをバインドします。
2. サービスに一連の基本パラメータを同期させます。
3. 次に、露出、ゲイン、チャンネル、モードを調整します。
4. 保存後、リアルタイムのイメージまたはプロセスを使用して検証を実行します。
5. 高度な調整が必要な場合は、テンプレートまたは調整ページに入ります。

## 相互に影響を与える可能性が最も高いパラメータはどれですか?

- カメラモードとチャンネル
- ビット深度と取得モード
- 露光時間とゲイン
- 自動露出および手動露出パラメータ
- 物理カメラのデフォルトと現在のサービス表示構成

特定の変更後に画像が明らかに異常である場合は、単一のパラメータだけに焦点を当てるのではなく、これらのグループが一緒に変更されているかどうかを確認することを優先してください。

## よくある質問

### パラメータを変更しても効果はありません

- 変更するカメラサービスが現在使用しているカメラサービスであることを確認してください。
- 保存後にビューを再度開くか、サービスを再起動します。
- 物理カメラの同期設定によって上書きされているかどうかを確認します

### 不当な露出またはチャンネル設定

- まず、現在のカメラモードでサポートされているチャンネルを確認します。
- 3 チャンネルパラメータとシングルチャンネルパラメータを混合しないでください。
- 白とびやノイズが発生する場合は、露出とゲインの調整を優先してください

### 保存後も処理結果が異常のままです。

- リアルタイム画面を使用して、まず画像の取得が正常であるかどうかを確認します。
- プロセス内のテンプレートパラメータが古い値のままであるかどうかを再度確認します
- 必要に応じてキャリブレーションと自動露出テンプレートのリンクされたチェック
