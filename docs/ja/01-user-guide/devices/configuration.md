# デバイスの追加と構成

このページでは、デバイス サービスの追加方法、保存方法、パラメーターを変更するときに最初に確認する場所など、ユーザーの操作パスについてのみ説明します。

## 一般的なシナリオ

- 新しいデバイスを接続するには、まずシステム内に対応するサービスを作成する必要があります。
- デバイスはすでに存在しますが、名前、通信パラメータ、またはバインディング オブジェクトを変更する必要があります。
- プロジェクトを切り替えた後は、装備の順序を調整したり、古い装備を無効にしたり、新しい装備を追加したりする必要があります。

## 基本的なプロセス

1. デバイス管理ウィンドウを開きます。
2. デバイス設定または編集インターフェイスに入ります。
3. 対応するタイプの新しいデバイス サービスを追加するか、既存のデバイスを選択して変更します。
4. 必要なパラメータを入力して保存します。
5. デバイスリストに戻り、デバイスが利用可能なリストに表示されていることを確認します。

後続のプロセス、カメラ ウィンドウ、または結果インターフェイスにデバイスがまだ表示されない場合は、まずデバイス リストに戻って、この手順が実際に有効かどうかを確認します。

## 通常設定時に確認が必要な内容

### 基本情報

- デバイス名は簡単に区別できますか?
- デバイスの種類は正しく選択されていますか?
- 現在のプロジェクトが正しいデバイス インスタンスを参照しているかどうか

### コミュニケーションまたは拘束力のある情報

・ ネットワークアドレス、ポート、サブジェクト、通信識別子が正しいか
- 対応する物理デバイスまたはリソース オブジェクトがバインドされているかどうか
- ネイティブドライバー、サーバー、またはサードパーティプログラムの準備ができているかどうか

### 保存後の可視性

- デバイスがデバイスリストにすでに表示されているかどうか
・装置の状態は正常ですか？
- 該当するウィンドウまたはプロセスノードでデバイスを選択できるかどうか

## よくある質問

### デバイスを追加してもリストにデバイスがありません

- 保存が本当に成功したかどうかを確認します
- 間違ったプロジェクトまたは間違った構成コンテキストで操作していないか確認してください
- デバイスウィンドウを再度開いて、リストだけが更新されていないかどうかを確認します。

### パラメータは変更されましたが、デバイスの動作は変わりません。

- 再接続、コントロールの再生成、または関連するウィンドウを再度開く必要があるかどうかを確認します。
- 変更が現在使用されているデバイス インスタンスに対するものであるかどうかを確認します

### 設定ページで何を入力すればよいのかわかりません。

- まず、対応する機器の特別ページに移動してパラメータの説明を見つけます
- デバイスが物理デバイス オブジェクトに依存している場合は、最初に物理デバイスの管理または調整の手順を完了します。
