# ファイルサービス

ファイル サービスは、リモート ファイル リスト、ファイルのダウンロード、ファイルのアップロードを統合デバイス システムに統合するために使用されます。 「結果ファイルがサーバー側にある」「デバイス側から画像を取得する必要がある」「リモートエンドにファイルを送信する必要がある」などのシナリオでの使用に適しています。

## このページはいつ読むべきですか?

- ファイルサービスを追加または設定したい場合
- リモートエンドで利用可能なファイルのリストを表示したい場合
- 現在のシステムにソース ファイルをダウンロードしたい場合
- リモートサービスにファイルをアップロードしたい
・「ファイルは見えるのに開けない」「アップロードしても反応がない」などのトラブルシューティングを行っていますか？

## 通常のサービスの利用方法

### 最初に構成を完了します

まず、デバイス コード、送信トピック、サブスクライブ トピックの基本構成が正しいことを確認してください。ファイル サービスに関する問題の多くは、「ファイル操作が失敗しました」のように見えますが、実際には、接続またはテーマの構成の早い段階で停止しています。

### ファイルリストをもう一度見てください

現在リモート エンドにどのようなファイルがあるかわからない場合は、まずファイル リストを取得してから、ダウンロードするかアップロードするかを決定します。ファイル名を直接推測しないでください。

### コンテンツが必要なときに開くかダウンロードします

現在の実装における一般的なファイル アクションには次のものがあります。

- すべてのファイルのリストを取得する
- 指定したファイルを開くかダウンロードします
- 指定したファイルをアップロードする

## 一般的な使用シーケンス

1. まずファイルサービスを作成し、デバイスリストに保存します。
2. 接続関連の設定が正しいことを確認します。
3. まずファイル リストを取得して、リモート エンドにターゲット ファイルが実際に存在することを確認します。
4. ダウンロードまたは開くアクションを再度実行します。
5. ファイルをアップロードし直す必要がある場合は、アップロードを再度実行します。

## 使用する際の注意点は何ですか?

### 最初にファイル名を確認する必要があります

ファイル サービスは、ファイル名に基づいて操作を開始します。ファイル名が間違っていると、後で表示ウィンドウを見たときにあまり意味がありません。

### リストとコンテンツは 2 つのステップです

リストが表示されても、特定のファイルを正常に開けるとは限りません。同様に、アップロードが成功したとしても、それが正しいディレクトリまたは正しいタイプで表示されたことを意味するわけではありません。

### 最初にテーマとデバイスのコードの問題をトラブルシューティングします

現在の実装では、デバイス コードとサブスクリプション トピックの間に明示的な一致があります。応答がない場合は、まずこれらの基本的な状況を確認してください。

## よくある質問

### ファイルリストを元に戻すことはできません

- まず送信トピックと購読トピックが正しいか確認してください
- 現在のデバイスコードが返されたメッセージと一致するかどうかを再度確認します
- サーバー上にハートビートはあるが、ファイル イベントがない場合は、最初に特定のファイル操作構成を確認してください。

### ファイルは一覧表示できますが、開くことはできません。

- まずはファイル名が正しいか確認してください
- 要求されたファイル形式が目的のファイル形式であるかどうかを再確認します
- 問題が解決しない場合は、[ログ ビューア](../interface/log-viewer.md) を使用して、返されるメッセージに明確なエラー メッセージがあるかどうかを確認してください。

### アップロード後にファイルが表示されない

- まず、アップロードアクションが実際に送信され、完了したかどうかを確認します
- 古いリストだけに依存するのではなく、ファイル リストを再度取得します。
- リモート エンドにディレクトリまたはタイプの区別がある場合は、正しい範囲を表示していることを確認してください。

### ファイル サービスがプロセス中に不安定です

- まず、ファイルのリスト、ダウンロード、アップロードが成功したかどうかを手動で確認します。
- [プロセスの実行とデバッグ](../workflow/execution.md) に戻り、プロセスで同じファイル サービスが使用されているかどうかを確認します。
- 必要に応じて、デバイス コードとテーマ設定の両方をチェックして、手動テストと一致しているかどうかを確認します。
