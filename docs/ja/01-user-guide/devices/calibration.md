# 校正サービス

キャリブレーション サービスは、キャリブレーション プロセスを実際のカメラにバインドし、キャリブレーション テンプレートを管理し、キャリブレーション データ生成をトリガーし、必要に応じて RAW ファイルとキャッシュを処理するために使用されます。これは単純な「パラメータ ページ」ではなく、物理カメラ、テンプレート編集、キャリブレーション結果を接続するサービス層です。

## このページはいつ読むべきですか?

- 校正サービスを作成または構成したい場合
- キャリブレーション サービスを物理カメラにバインドする必要があります
- キャリブレーションテンプレートを編集したい
- キャリブレーション関連のデータ取得をトリガーしたい場合
- 「キャリブレーションを開始できないのはなぜですか?」などの問題のトラブルシューティングを行っていますか? 「なぜテンプレートを開けないのですか？」

## 使用する前に 2 つのことを確認してください

### 最初に物理カメラをバインドします

現在の実装では、キャリブレーション サービスは物理的なカメラ コードに直接依存しています。対応する物理カメラが最初に設定されていない場合、多くのキャリブレーション関連の操作はまったく続行されません。

### データベースが再び利用可能であることを確認します

現在の環境でMySQLが有効でもデータベースが接続されていない場合、キャリブレーションテンプレートの編集が正常に入りません。この場合は、まずデータベース接続を修正し、テンプレート ウィンドウを繰り返しクリックしないでください。

## 一般的な使用シーケンス

1. まず、デバイスリストでキャリブレーションサービスを作成または確認します。
2. 正しい物理カメラにバインドします。
3. キャリブレーション テンプレート編集ポータルを開き、テンプレートの内容が現在のカメラに対応していることを確認します。
4. 校正データを取得する必要がある場合は、関連する校正アクションを実行します。
5. 物理的なカメラの構成が変更された場合は、必ず戻って、調整サービスがまだ正しくバインドされているかどうかを確認してください。

## このサービスで最も一般的な操作

### 物理カメラを管理する

現在のキャリブレーション サービスに対応するカメラの準備がまだ整っていない場合は、トラブルシューティングのためにキャリブレーション ウィンドウに留まるのではなく、物理カメラ管理ページに移動して最初に処理してください。

続きを読むことができます:

- [カメラ管理](./camera-management.md)
- [カメラパラメータ設定](./camera-configuration.md)

### キャリブレーションテンプレートの編集

キャリブレーション テンプレートの編集は、現在バインドされている物理カメラに依存します。テンプレート ウィンドウを正常に開くことができるかどうかは、それ自体が貴重な健全性チェックです。

- 開けない場合は、カメラのバインドとデータベース接続が最初にチェックされます。
・開けるが内容が間違っている場合は、現在のカメラオブジェクトとテンプレートオブジェクトが正しいかどうかの確認を優先します。

### 元のファイルとキャッシュを処理する

キャリブレーション サービスには、生ファイルのリスト、ファイルのダウンロード、キャッシュのクリーニングも含まれます。 「古いデータが現在のキャリブレーションを妨げる」などの問題が発生した場合は、サービス全体が利用できないことを直接疑うのではなく、キャッシュと元のファイル範囲から始めてください。

## よくある質問

### [キャリブレーション テンプレートの編集] をクリックしましたが、エディターには入りませんでした。

- まず、対応する物理カメラがバインドされているかどうかを確認します
- MySQL が有効になっている場合は、データベース接続が正常であるかどうかを確認します。
- 正しいキャリブレーション サービス インスタンスが現在開かれていることを再確認します。

### カメラは交換されましたが、キャリブレーション結果はまだ古いようです。

- まず、キャリブレーション サービスにバインドされている `CameraCode` が更新されているかどうかを確認します。
- 物理カメラを切り替えた後、現在の設定が再保存されているかどうかを再確認します
- 必要に応じて、テンプレート エディターを再度起動して、現在のテンプレート オブジェクトを確認します。

### キャリブレーションプロセス中に取得された元のファイルが正しくありません。

- まず、現在のデバイスコードとデバイスタイプが対象デバイスであるかどうかを確認します。
- 古いキャッシュではなく、元のファイル リストが要求されていることを再確認します。
- キャッシュの干渉が疑われる場合は、まずキャッシュをクリーンアップしてから、再度確認してください。

### 調整サービスは存在しますが、プロセスでは有効になりません。

- まず、バインドされたカメラとテンプレートが有効かどうかを確認します
- 検査の再リンク [ワークフロー概要](../workflow/README.md) と [プロセスの実行とデバッグ](../workflow/execution.md)
- 同時に、[ログビューア](../interface/log-viewer.md)と組み合わせて、特定の障害段階を確認します
