#デバイスサービスの概要

このページはデバイスの章への入り口として機能します。 「どのようなデバイスページが利用可能か、通常はどのように設定すればよいか、問題が発生した場合はどこを最初に確認すればよいか」を優先的に回答します。

## デバイスサービスとは何ですか?

ColorVision では、通常、デバイスは「サービス」として管理されます。メイン プログラムはデバイス サービスのリストを維持し、ユーザーはデバイス ウィンドウでこれらのサービスを表示、構成、有効化、および操作できます。

デバイス関連の実装は主に次の場所にあります。

- `Engine/ColorVision.Engine/Services/`
- `Engine/ColorVision.Engine/Services/Devices/`

現在のコード ディレクトリで確認できる一般的なデバイス分類は次のとおりです。

-カメラ
-校正
- モーター
-ファイルサーバー
-フローデバイス
-センサー
- SMU
- スペクトル

## 一般的な使用シーケンス

1. まず、[デバイスの追加と構成](./configuration.md) を読んで、デバイスの追加と保存の基本プロセスを理解します。
2. 特定のデバイスのページに入り、デバイスのパラメータと動作を確認します。
3. カメラが関係する場合は、引き続き [カメラ管理](./camera-management.md) および [カメラ パラメーター設定](./camera-configuration.md) を参照してください。
4. 自動化プロセスにデバイスを関与させる必要がある場合は、[ワークフローの概要](../workflow/README.md) を参照してください。

## 使用時によく発生すること

- デバイス サービスは、実際のハードウェアにバインドされている場合もあれば、特定の種類の通信またはファイル サービスである場合もあります。
- デバイス リストの順序、有効化ステータス、構成内容は、多くの場合、後続のウィンドウやプロセスでの表示に影響します。
- 基本構成に加えて、一部のデバイスには、独立した物理デバイス管理、キャリブレーション、またはパラメータ構成ページもあります。

## トラブルシューティングの提案

### デバイスがリストに表示されません

- まず、デバイス設定ウィンドウで作成および保存されているかどうかを確認します。
・物理ハードウェア、ドライバ、通信環境など、対応するデバイスの依存関係が満たされているかを再度確認してください。

### デバイスは表示されますが、動作しません

- デバイスの特別ページでパラメータの説明を確認することを優先します。
・ログや接続状況を再度確認してください。
・ プロセスの呼び出しが失敗した場合は、[プロセスの実行とデバッグ](../workflow/execution.md)を併せて確認してください。
