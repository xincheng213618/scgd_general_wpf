# データベース操作

現在、最も明確で安定したデータベース エントリは「データベース ブラウザ」です。テーブルレベルの参照、検索、ページング表示、および条件が満たされた場合の追加、変更、削除に適しています。

## このページはいつ読むべきですか?

- 特定のデータのバッチが実際にデータベースに保存されているかどうかを確認したい
- 実際のテーブルデータをMySQLやSQLiteで確認したい
- キーワードでレコードを検索する必要があります
- 少量のテーブルデータのメンテナンスを行う必要がある
- 「データは書き込まれましたか?」などの問題のトラブルシューティングを行っていますか? 「なぜリストに表示されないのですか？」

## 現在の実際の入り口

現在、データベース ブラウザには明示的なメニュー エントリがあります。

- ツール → データベースブラウザ

現在の実装では、これは SQL エディターではなく、テーブルの参照と基本的なメンテナンスを目的としたウィンドウです。

## データベースブラウザでは何ができるのでしょうか?

### データ ソース、ライブラリ、テーブルを参照する

左側のツリーには最初にデータ ソースが表示され、次にライブラリとテーブルが表示されます。デフォルトで登録および表示できるソースには、現在少なくとも次のものが含まれます。

-MySQL
- SQLite ロギング ライブラリ

### データをテーブルごとに表示する

テーブルを選択すると、右側に次が表示されます。

- 表のタイトルとソース
- コラム情報
- データグリッド
- 検索ボックス
- ページネーション制御
- ステータスバー

### 検索とページング

現在のウィンドウは、キーワードによる検索と、大量のテーブル データがある場合のページ内の参照をサポートしています。 「明らかにデータがあるのに見当たらない」という場合は、書かれていないと直接判断するのではなく、まず検索条件やページ番号を確認してください。

### 基本的な書き込み操作

現在のデータ ソースとテーブルの両方で書き込みが許可されており、テーブルに主キーがある場合、インターフェイスは次のアクションを提供します。

- 新しい
- 保存
- キャンセル
- 削除
- リフレッシュ

テーブルに主キーがない場合、または現在のデータ ソースで書き込みが許可されていない場合は、すべての編集ボタンが使用可能であることを期待しないでください。

## 一般的な使用シーケンス

1. まず、[ツール] メニューからデータベース ブラウザを開きます。
2. 左側で適切なデータ ソース、ライブラリ、およびテーブルを展開します。
3. まずテーブル名、ソース、レコード数をチェックして、間違ったデータベースに入力されていないことを確認します。
4. データが多すぎる場合は、まず検索を使用してからページをめくってください。
5. メンテナンスが必要な場合は、現在のテーブルが書き込み可能かどうかを確認してください。

## 使用する際に特に注意すべきことは何ですか?

### 現在のウィンドウはテーブル ブラウザであり、ユニバーサル クエリ プラットフォームではありません

これはテーブルの表示、レコードのフィルタリング、および簡単なメンテナンスの実行に適していますが、現在のドキュメントでは統合 SQL コンソールとして説明されていません。モジュールに特別なクエリ ウィンドウがある場合は、そのモジュール自体の入り口を最初に使用する必要があります。

### 主キーのないテーブルは、簡単な変更には適していません。

現在の実装では、変更と削除の信頼性を判断するために主キーに依存しています。主キーのないテーブルは、読み取り専用の参照や慎重な新規追加に適しています。

### 書き込み機能はデータ ソースとテーブルに依存します

データベース ブラウザを開いたとしても、すべてのテーブルに書き込みできるわけではありません。ボタンが使用できないことがわかった場合は、まず、現在のテーブルが書き込み可能でないのか、データ ソース自体が読み取り専用であるかを判断します。

## よくある質問

### 見たいデータテーブルが見つかりません

- まず、現在展開されているデータ ソースとライブラリが正しいことを確認します。
- データソースツリーを再度更新します
・MySQLの場合、接続構成とデータベース名が正しいか確認してください

### テーブルは開くことができますが、データは空です

- まず、現在の検索条件が絞り込まれているかどうかを確認します
- ページ番号とページサイズを再度確認してください
- まだ空の場合は、[プロセスの実行とデバッグ](../workflow/execution.md) または [ログ ビューア](../interface/log-viewer.md) に戻って、書き込みが本当に成功したかどうかを確認します。

### ボタンはグレーで表示され、編集できません。

- まず、現在のテーブルが書き込み可能かどうかを確認します。
- テーブルに主キーがあるかどうかを再確認します
・ 主キーがない場合、「編集不可」をデータベース例外と誤判断しないでください。

### 修正後の結果は正しくありません。

- まず保存が成功したかどうかを確認します
- 現在のテーブルを更新して再度表示します
- 重要なビジネス データについては、最初に読み取りパスが正しいかどうかを確認し、手動で変更します。
