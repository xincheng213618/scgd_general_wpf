# データ管理

この章では、データは通常どこにあるのか、いつデータベース ページに入る必要があるのか、いつインポートおよびエクスポートする必要があるのかという 3 つの質問に答えるだけです。

## この章を読むのに適切な時期はいつですか?

- テスト結果や業務データがどこに保存されているか確認したいですか?
- データベースの内容を表示、整理、またはクリーンアップする必要がある
- バッチインポートまたはバッチエクスポートを行う必要があります
- 「データが保存されない」、「エクスポートに失敗した」、「履歴レコードが表示されない」などの問題のトラブルシューティングを行っていますか?

## この章の読み方

### まずデータベースを見てみましょう

- [データベース操作](./database.md)

適切なシナリオ:

・実際にデータベースにデータが保存されているか確認したい
- 記録やタイム、条件ごとに結果を確認したい
- 接続、書き込み、またはクエリの問題をトラブルシューティングしたい

### インポートとエクスポートをもう一度見てみましょう

- [データのエクスポートとインポート](./export-import.md)

適切なシナリオ:

- 結果は外部システムに送信するか、手動で分析する必要があります
- 設定またはデータのバッチを移行する必要がある
・現在対応しているエクスポート形式と処理順序を確認したい

## 一般的な使用シーケンス

1. まず、現在のプロジェクト、設備、またはプロセスが正しく実行されていることを確認します。
2. データが保存されていないと思われる場合は、まず[データベース操作](./database.md)に入力してレコードを確認します。
3. データが存在することを確認した後、以降の処理のために[データのエクスポートとインポート](./export-import.md)を入力します。
4. エクスポートされたコンテンツが正しくない場合は、エクスポート ボタンを繰り返し試行するのではなく、元に戻ってソース データを確認してください。

## よくある質問

### 必要な履歴データが表示されない

- まず、現在接続されているデータベース インスタンスが正しいかどうかを確認します。
- フィルター条件、時間範囲、プロジェクトオブジェクトが正しく選択されているかを再確認します
・ 実行直後の場合は、まず[ログビューア](../interface/log-viewer.md)と組み合わせて、本当に書き込みが成功しているか確認してください。

### データは存在しますが、エクスポート結果が正しくありません。

- まず、エクスポートされたオブジェクトが現在表示されているデータのバッチであるかどうかを確認します。
- エクスポート形式がその後の使用シナリオを満たしているかどうかを再確認します
・ フィールドが欠落している場合は、直接エクスポート機能の異常と判断するのではなく、ソースデータのページに戻って確認してください。

### 問題がプロセスにあるのかデータ層にあるのかわからない

- まず、[プロセスの実行とデバッグ](../workflow/execution.md) を確認します。
- 次に、[ログビューア](../interface/log-viewer.md)を使用して、実行、書き込み、またはエクスポートのフェーズでエラーが発生したかを確認します。
