# コンポーネントの相互作用

このページには、システム全体をカバーする「モジュール相互作用マトリックス」が維持されなくなりましたが、歪みが生じやすくなります。現時点でより価値のあるアプローチは、コード内に実際に存在し、ほとんどの場合変更の方向に影響を与えるいくつかの対話型メイン チェーンのみを記述することです。

## まずこれらのメインチェーンを見てみましょう

### 1. メインプログラムのエントリーチェーン

メイン プログラム エントリは `ColorVision/` にあります。

まず次のことを担当します。

- 基本的な DLL のプリロード
- 設定、ログ、テーマ、言語の初期化
- ファイル処理コマンドライン分岐
- プラグインがロードされているかどうかの判断
- 起動ウィザードまたは起動ウィンドウ

このレイヤーは、特定のビジネスそのものではなく、「ホスト」と「エントリー スケジューラー」に似ています。

### 2. UI ワークベンチ チェーン

メイン ワークスペースに入ると、`UI/` の下のウィンドウ、メニュー、パネル、およびエディターがユーザー操作を受け入れ、アクションをエンジン層に転送する役割を果たします。

ここでの最も一般的なやり取りは次のとおりです。

- メイン ウィンドウとデスクトップ メニューにアクセスできる
- 属性エディター、画像エディター、データベース ブラウザー、およびその他のモジュールは、それぞれの対話型ビューを担当します。
- 実際のビジネス アクションは引き続き `Engine/` に送信されます

### 3. サービスツリーとデバイスチェーン

`ServiceManager` は、ランタイムで最も重要な調整オブジェクトの 1 つです。それは次のことを行います:

- データベースからリソースとサービスの定義を読み取ります
- ターミナルサービスおよびデバイスサービスオブジェクトの生成
- グループリソースを整理する
- プロセス表示領域とデバイス表示コントロールを表示マネージャーに統合

したがって、「なぜデバイスがツリーに表示されるのか」、「なぜ表示領域を開くことができるのか」、「なぜ特定のデバイスがランタイムに入らないのか」については、通常、単一のデバイス クラスを直接調べるのではなく、`ServiceManager` を最初に調べます。

### 4. テンプレートの登録と編集のチェーン

テンプレート システムの重要な相互作用はスタンドアロン テーブルではありませんが、次のとおりです。

- 関連するアセンブリがメイン プログラムまたはプラグインによってロードされる
- `TemplateControl` は、`IITemplateLoad` を実装する型をスキャンします。
- これらのタイプは、`Load()` のグローバル テンプレート ディクショナリにテンプレートを登録します。
- これらのテンプレートを使用するためのテンプレート エディター、プロセス ウィンドウ、および特定のビジネス機能

そのため、「テンプレートが表示されない理由」のような問題は、通常、編集ウィンドウ自体だけでなく、読み込みチェーン、データベース ステータス、またはアセンブリ読み込みチェーンに関する問題です。

### 5. プロセス実行チェーン

プロセス実行中のメイン チェーンは大まかに次のとおりです。

1. `DisplayFlow` 現在のプロセス テンプレートを選択して更新します。
2. `FlowControl` は現在のプロセスを開始または停止します。
3. `FlowEngineLib` 実行ノードのグラフ。
4. `MQTTRCService` は、サービス トークンとステータスの更新を提供します。
5. デバイス ノードまたはアルゴリズム ノードは結果を返します。
6. 実行ステータス、バッチ レコード、ログ テキスト、およびノー​​ド メッセージがインターフェイスに更新されます。

このリンクは、共通の事実を示しています。プロセスは分離されたモジュールではなく、当然のことながらテンプレート、デバイス サービス、およびレジストリの状態に依存します。

### 6. プラグイン拡張チェーン

プラグインが読み込まれると、もう 1 つの DLL ではなく、新しいアセンブリがランタイム全体に取り込まれます。

- 新しいメニューまたはウィンドウのエントリ
- 新しいサービスまたはプロバイダーの実装
- 新しいテンプレート、結果ビュー、または拡張ポイントの実装

したがって、プラグインとメイン プログラムの対話は、外部で並行して実行されるというよりは、「既存のランタイムに機能を組み込む」ことに似ています。

## コードを変更するときにこのページを通常使用する方法

### メインウィンドウまたはユーザー入口を変更する

次のことを優先します。

- メインプログラム入口
- UIメニュー/パネル
- 対応エンジンサービス

### デバイスの動作を変更する

次のことを優先します。

- `ServiceManager`
- 特定の `DeviceService`
- MQTT またはランタイム状態オブジェクトに対応

### テンプレートまたはプロセスを変更する

次のことを優先します。

- `TemplateControl`
- テンプレートエディター
- `DisplayFlow` / `FlowControl` / `FlowEngineLib`

### プラグイン拡張子の変更

次のことを優先します。

- `PluginLoader`
- プラグインセット
- プラグインによって実装されたメニュー、サービス、テンプレート、またはビュー インターフェイス

## このページはそれ以上何もしません

このページでは、次のような危険性の高いコンテンツは管理されなくなります。

- 現在の実装とは関係のない、一見完全なモジュール マトリックス テーブル
- 未検証のイベント名とインターフェースの大きなリスト
- 実際のコードディレクトリから分離された理論上の依存関係グラフ

トピックで詳細な設計分析が必要な場合は、ここで過度な概要を広めるのではなく、対応するトピック ページに戻る必要があります。
