## ディレクトリの説明

- `overview/` は、起動、実行時、コンポーネントの関係などのシステム レベルの観点に焦点を当てています。
- `components/engine/` プロセス エンジンと実行モデルに焦点を当てます。
- `components/templates/` テンプレート システムの設計と現状分析に重点を置きます。
- `security/` 権限モデルとセキュリティ境界に焦点を当てます。

## 推奨される読み方

- 初めてシステムに触れるときは、「システムの概要→ランタイム→コンポーネントの相互作用」の順に読んでください。
- プロセスまたはテンプレートを変更する必要がある場合は、`components/` の下のトピック ページに入ります。
- インターフェースとタイプの詳細が必要な場合は、[API リファレンス](../04-api-reference/README.md) を参照してください。
