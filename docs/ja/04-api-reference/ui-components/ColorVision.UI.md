#ColorVision.UI

このページには、現在の最も重要なインフラストラクチャと `UI/ColorVision.UI/` のエントリ タイプのみが保持され、古いドキュメントの「バージョン リスト + 完全な疑似 API + 更新ログ」の記述方法は維持されなくなります。

## モジュールの配置

`ColorVision.UI` は個別のコントロールではなく、デスクトップ アプリケーションの大量の UI インフラストラクチャが存在する場所です。現在の役割は「UI シェルとパブリック サービス コレクション」に近く、主にメイン プログラム、エンジン、その他の UI サブプロジェクトによって再利用されます。

ディレクトリ構造から判断すると、対象となるコンテンツには少なくとも次のものが含まれます。

- 読み取り、書き込み、および環境パスを構成する
- プラグインのロードとプラグイン パッケージの処理
- メニューシステム
- プロパティエディター
- ショートカットキーシステム
- 多言語リソース
- ログ関連の UI 設定
- シェル、検索、ステータスバー、ページ、および共通コントロール

したがって、これは単一の「コントロール ライブラリ」ではなく、安定したパブリック API サーフェスを備えた標準 SDK として作成するのにも適していません。

## 現在最も重要なディレクトリ

認知度をすぐに高めたいだけの場合は、まず次のディレクトリを参照することをお勧めします。

- `Plugins/`: プラグインの検出、メタデータ、依存関係のチェック、解凍および更新
- `PropertyEditor/`: 属性編集ウィンドウ、ツリーノード、エディタタイプシステム
- `Menus/`: メニューの登録と動的メニューの更新
- `HotKey/`: グローバルおよびウィンドウレベルのショートカット キー
- `Languages/`: 言語とリソースの切り替え
- `LogImp/`: ログ関連の設定とウィンドウのステータス
- `ConfigSetting/` およびルート ディレクトリ `ConfigHandler.cs`: システム エントリを構成します
- `Shell/`、`Serach/`、`StatusBar/`: デスクトップの対話型支援機能

## キー入力タイプ

### `ConfigHandler`

`ConfigHandler` は、このプロジェクトのコア インフラストラクチャの 1 つです。多くの `IConfig` 構成オブジェクトは、最終的にそのオブジェクトまたは関連する構成サービスで読み取られ、キャッシュされ、保存されます。

「設定が保存されない」、「構成がロードされない」、「デフォルト値が異常」という問題が発生した場合は、通常、最初にこのリンクを確認してください。

### `PluginLoader`

`PluginLoader` は現在、プラグインのランタイム読み込みを担当しています。これは「DLL のスキャン」だけでなく、次のことも行います。

- `Plugins/` ディレクトリをスキャンします
- `manifest.json` を読む
- オプションの `.deps.json` を解析します
- `ColorVision.*` 依存関係のバージョンを確認する
- プラグイン アセンブリの最終ロード

これが、プラグイン関連のドキュメントが「リフレクション スキャン プラグイン タイプ」とだけ書かれていると通常歪んでしまう理由です。

### `MenuManager`

`MenuManager` はメニュー システムの中心的なオブジェクトです。多くの動的メニュー、最近のファイル更新、およびプラグイン メニュー エントリは、最終的にその登録または更新チェーンに分類されます。

したがって、この部分は、静的な XAML メニュー定義のセットというよりは、アプリケーション シェルのメニュー コーディネーターに似ています。

### `PropertyEditor`

`PropertyEditor/` 属性編集エクスペリエンスを担当する現在のメイン チェーン:

- `PropertyEditorWindow`
- `PropertyTreeNode`
- エディターの種類と補助クラス

このシステムは、倉庫内で `Category`、`DisplayName`、`Description`、`PropertyEditorType` などの特性を持つ多数のオブジェクトと組み合わせて使用されます。これは、現在の動的属性編集エクスペリエンスの基礎です。

### `HotKey`

ショートカット キー システムは現在単一のポイントとして実装されていませんが、次のように分割されています。

- `GlobalHotKey/`
- `WindowHotKey/`
- `HotKeys` とその構成および設定ウィンドウ

したがって、ショートカット キーを変更する場合は、通常、システム レベルのホットキーを変更するのか、ウィンドウ全体のホットキーを変更するのかを区別する必要があります。

### `Languages`

多言語リソースと UI カルチャ切り替え関連機能はここで一元管理されます。メイン プログラムの起動段階で UI カルチャーを設定すると、多くのインターフェイス リソースの読み込みが影響を受けます。

### `LogImp`

ログ関連の UI 構成とローカル ログ ウィンドウのステータスもこのプロジェクトに配置されます。これは完全なログ バックエンド自体ではなく、「ログの表示と構成の一致」に関するものです。

## このプロジェクトは現在、間違って記述されている可能性が最も高くなります。

### 単一のコントロール ライブラリではありません

古いドキュメントでは、`ColorVision.UI` を「コア UI コントロール パッケージ」と記述することが好まれています。現在のコードはこれよりもはるかに複雑です。また、プラグイン、構成、メニュー、ショートカット キー、プロパティ編集、多言語などの横断的な機能も担っています。

### プラグイン システムは拡張ポイント定義自体と等しくありません

`PluginLoader` はここにありますが、プラグインが実際に拡張する機能は、各プラグイン アセンブリと実装されたメニュー、テンプレート、サービス、および結果ビュー インターフェイスによって異なります。

### このページでは権限を「グローバル RBAC センター」に一般化しないでください。

現在、グローバルの粗いアクセス許可は `Authorization.Instance.PermissionMode` から提供されますが、より詳細なローカル RBAC サブシステムは主に `UI/ColorVision.Solution/Rbac/` にあります。 `ColorVision.UI` は認証インフラストラクチャとパブリック依存関係を提供するため、ここでは完全な権限プラットフォームとして記述するべきではありません。

## 現在このプロジェクトをどのように読むのがより適切ですか?

### 構成およびグローバル サービスを確認したい

まずはご覧ください:

- `ConfigHandler.cs`
- `Environments.cs`
- `FileProcessorFactory.cs`

### プラグインの実行時間を確認したい

まずはご覧ください:

- `Plugins/PluginLoader.cs`
- `Plugins/PluginManifest.cs`
- `Plugins/PluginInfo.cs`

### 属性編集システムを知りたい

まずはご覧ください:

- `PropertyEditor/PropertyEditorWindow.xaml(.cs)`
- `PropertyEditor/PropertyTreeNode.cs`
- `PropertyEditor/PropertyEditors.cs`

### メニューやショートカットキーを知りたい

まずはご覧ください:

- `Menus/MenuManager.cs`
- `HotKey/HotKeys.cs`
- `HotKey/GlobalHotKey/`
- `HotKey/WindowHotKey/`

## このページはそれ以上何もしません

このページでは、次のような危険性の高いコンテンツは管理されなくなります。

- 古いバージョン番号とターゲット フレームワーク リスト
- クラスメンバー用の未検証の疑似コードの大部分
- `ColorVision.UI` を安定したパブリック SDK として説明します
- 権限、プラグイン、ログ、その他の横断的な機能を独自の完全なプラットフォームに変換します

将来、特定のサブシステムを補足する必要がある場合は、ここで「大きくて包括的な」手順を積み続けるのではなく、対応するトピック ページに直接移動する必要があります。
