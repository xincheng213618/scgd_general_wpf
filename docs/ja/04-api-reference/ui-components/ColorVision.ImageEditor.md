#ColorVision.ImageEditor

このページでは、現在実装されている UI/ColorVision.ImageEditor のメイン コントロール チェーンと拡張ポイントについてのみ説明します。旧ドキュメントの「完全な機能 + チュートリアルの例 + 性能数値の約束」という記述方法は維持されなくなります。

## モジュールの配置

ColorVision.ImageEditor は現時点では単なる画像表示コントロールではなく、「イメージホスト + スケーラブル キャンバス + 描画ツール + オープナー + ランタイム ツール オーバーレイ + 設定システム」を組み合わせたモジュールのセットです。

その主なラインは次のようなものに近いです。

- `ImageView` をホストとして使用
- `EditorContext` を現在のビュー ランタイム コンテナーとして使用
- 実際の描画キャンバスとしての `DrawCanvas`
- `IEditorToolFactory` は、ツール、メニュー、オープナーの検出と組み立てを担当します。
- `IImageOpen` は、さまざまなファイル タイプのオープン チェーンを担当します

## 現在最も重要なディレクトリ

プロジェクト ディレクトリの中で、最初に読む価値のあるものは次のとおりです。

- `ImageView.xaml(.cs)`: メイン コントロールとランタイム オーケストレーションの入り口
- `EditorContext.cs`: 各ビュー インスタンスのランタイム コンテナー
- `DrawCanvas.cs`: ビジュアル ツリーと描画キャンバス
- `EditorToolFactory.cs`: ツール、メニュー、オープナーの検出とアセンブリ
- `Abstractions/`: エディターの拡張ポイントの境界
- `Draw/`: グラフィック要素、ツール、選択ボックス、および注釈のインポートおよびエクスポート
- `EditorTools/`: ズーム、疑似カラー、3D、アルゴリズム、フルスクリーンなどの非原始的なツール
- `Video/`: ビデオ オープナー
- `Layers/`: レイヤー/チャネル切り替えセマンティクス
- `Realtime/`、`Settings/`: ライブ画像と設定関連のサポート

## キー入力タイプ

### 画像ビュー

`ImageView` は、現在のエディター モジュールのメイン エントリです。それは次のことを担当します。

- `EditorContext` の初期化
- バインディング `DrawCanvas`、`Zoombox`、コンテキスト メニューおよびステータス バー
- `IImageComponent` を 1 回限りの初期化を実行します
- 標準ファイルコマンドの管理
- 配線構成の変更、スケーリングの変更、オーバーレイの更新
- 画像を開く、クリーニング、保存、注釈のインポートとエクスポートなどのプロセスを処理します。

このモジュールを理解したい場合は、最初のエントリは `ImageView.xaml.cs` です。

### エディタコンテキスト

`EditorContext` は、各 `ImageView` インスタンスのランタイム コンテナーであり、現在は一元的に保存されています。

- 現在の`ImageView`
- `DrawCanvas`
- `Zoombox`
- `ImageViewConfig`
- `DrawEditorManager`
- `IEditorToolFactory`
- 現在のオープナー、コンテキスト メニュー、プリミティブ リスト
- 軽量サービス レジストリのセット

これは実行時状態コンテナーとローカル サービス ロケーターの両方であり、現在のモジュールの重要な実装境界でもあります。

### DrawCanvas

`DrawCanvas` は、実際の図面を含むレイヤーです。コントロールを表示するだけでなく、次のことも行います。

- ビジュアルオブジェクトのコレクションを維持する
- ヒットテストを実行する
- グラフィック要素の追加と削除の処理
- メンテナンスの取り消し/やり直し
- 多数の描画ツールにマウス イベントをフックするターゲットとして機能します。

### IEditorToolFactory

名前は `IEditorToolFactory` ですが、これは現時点ではインターフェイスではなく具象クラスです。構築時に反射的にスキャンされ、組み立てられます。

- `IDVContextMenu`
- `IIEditorToolContextMenu`
- `IEditorTool`
- `IImageComponent`
- `IImageOpen`

同時に、「グローバル ツール + 現在のオープナー ランタイム ツール」の有効なビューも維持され、`GuidId` に従ってツールバーが上書きおよび再構築されます。

これは、現在の ImageEditor の初期化コストと拡張機能が最も集中している場所でもあります。

### IImageOpen とその拡張インターフェイス

現在、オープニング チェーンは統合ファイル マネージャーではなく、個別の `IImageOpen` 実装によって処理されます。

さらに、オプションで `IImageOpen` を実装できます。

- `IImageOpenEditorToolProvider`
- `IImageOpenEditorToolLifecycle`

このようにして、特定の特殊なファイル タイプは、すべてのブランチをグローバル ツールにスタックするのではなく、開いた後にツールバーの機能を一時的に引き継いだり、オーバーライドしたりできます。

### VideoOpen/Window3D/ModelViewer3DControl

ビデオと 3D 機能は現在、エディター モジュールの実際のサブ機能ですが、これらは追加のオープナーまたはツールであり、モジュール全体の唯一のメイン スレッドではありません。

- `Video/VideoOpen.cs`: ビデオ オープナー
- `EditorTools/ThreeD/Window3D.xaml.cs`: 画像を 3D サーフェス ウィンドウに変換
- `EditorTools/ThreeD/ModelViewer3DControl.xaml.cs`: OBJ/STL ビュー コントロール

古いドキュメントではこれらの機能について大幅に拡張されていますが、より信頼性の高いドキュメントの読み方は、まず `ImageView` とツール ファクトリのメイン チェーンを理解することです。

## 現在のランタイムのメインチェーン

既存の制御チェーンは大まかに次のとおりです。

1. `ImageView` を作成します。
2. `EditorContext`、`SelectionVisual`、`CompactInspectorPresenter` を初期化します。
3. `IEditorToolFactory` を作成し、グローバル ツール、コンテキスト メニュー、イメージ コンポーネント、およびオープナーを反射的にアセンブルします。
4. すべての `IImageComponent.Execute(ImageView)` を実行します。
5. ユーザーがファイルを開いた後、拡張子に基づいて `IImageOpen` を選択します。
6. オープナーは `SetImageSource(...)` を呼び出し、オプションで独自のランタイム ツールを提供します。
7. __​​IC_54__、オーバーレイ、ステータス バー、レイヤー切り替え、擬似カラーなどは、現在の画像コンテキストに合わせて引き続き動作します。

## 現在の実装の境界は何ですか?

### ImageView は純粋な表示コントロールではありません

`SetImageSource(...)` は現在、`ImageShow.Source` を設定するだけでなく、疑似カラー構成、キャリブレーション サービス、およびその他のエディター ランタイムの副作用もトリガーする可能性があります。

シーンを表示するだけの場合は、`EnableEditorImageServices` などのスイッチに注意する必要があります。デフォルトでは、ImageView 全体を副作用のないピクチャ フレームとして扱うことはできません。

### ツールの検出はリフレクション主導型です

`IEditorToolFactory` 現在、各ビュー インスタンスでは複数回のスキャンと作成が行われます。これは実際の重要な制御チェーンです。古いドキュメントの「静的ツール アーキテクチャ図」では、この問題は隠蔽されています。

### EditorContext は、状態コンテナーとサービス ロケーター属性の両方です。

現在の設計では、「構成」、「ツールのステータス」、および「ランタイム サービス」が完全に分離されていませんが、`EditorContext`、`ImageViewConfig`、および少数のサービスに部分的に集中しています。これは、読書とその後の再構成の際に認識しなければならない現実です。

### インポートおよびエクスポートに関するコメントには、実際の配置ポイントが含まれています。

グラフ要素の永続性は概念レベルにとどまりませんが、現在実際には `Draw/Annotations/` および `ImageView` のインポートおよびエクスポート エントリに影響します。注釈機能を読むときは、「あらゆる描画ツールを自動的に永続化できる」と一般化するのではなく、このチェーンを直接見る必要があります。
