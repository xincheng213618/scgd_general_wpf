# 拡張ポイントの概要

この章では、コードと直接一致する拡張ポイントのトピックのみを保持し、「一目でわかるすべての拡張メカニズム」の古い一般的なリストは保持しません。

## 現在の適用範囲

現在、このブランチには 1 種類の安定したトピックのみが含まれています。

- [FlowEngineLib ノード拡張機能](./flow-node.md)

これは、現時点では `extensions/` が完全な拡張百科事典ではなく、非常に限定された「組織化された拡張ポイント エントリ」であることを意味します。

## まず境界を区別しましょう

- プラグインの検出、ロード、およびデプロイメントはここには属しません。[プラグインとステータス ページ](../plugins/README.md) および [プラグイン開発の概要](../../02-developer-guide/plugin-development/overview.md) を参照してください。
- アルゴリズム テンプレートとプロセス テンプレートはここには属しません。[アルゴリズムとテンプレートの概要](../algorithms/README.md) にアクセスする必要があります。
- ランタイム モジュール間の依存関係はここでは展開されないため、[アーキテクチャ設計](../../03-architecture/README.md) に戻す必要があります。

## この章の使用方法

1. まず、「フローノード」を展開するか「プラグイン/テンプレート/サービス」を展開するかを確認します。
2. フローノードの場合は、[FlowEngineLib ノード拡張子](./flow-node.md)を入力します。
3. 問題がランタイム実行チェーンに関するものである場合は、[FlowEngineLib アーキテクチャ](../../03-architecture/components/engine/flow-engine.md) と一緒に読んでください。

## なぜここには 1 ページしかないのですか?

- 現在のウェアハウスには、ドキュメントによって安定したトピックに実際にバンドルされている拡張ポイントはあまりありません。
- 完全に見えても実際にはすぐに期限切れになる拡張ディレクトリを維持し続けるよりも、コードで直接検証できるエントリのみを保持する方が良いでしょう。
