# ScreenRecorder ステータスの説明

対応するソース コード プロジェクトが現在の `scgd_general_wpf` ワークスペースで見つからなくなったため、このページでは現在のウェアハウスの標準プラグイン実装として ScreenRecorder を記述しなくなりました。

## 現在のワークスペースの実際のステータス

現在の倉庫構造に従って確認してください:

- `Plugins/` ディレクトリの下に `ScreenRecorder/` ソース コード ディレクトリはありません。
- ワークスペースに対応するプラグインプロジェクトファイルがありません。
- 現在のプラグイン インデックス ページ [Plugins/README.md](../../../../Plugins/README.md) には、既存のプラグイン ディレクトリとしてリストされていません。
- 対応するプラグイン実装が現在のウェアハウスに存在することを示すためではなく、履歴ステータスの説明ページに引き続きアクセスできるようにするためだけに、現在のドキュメントのサイドバーに保持されます。

したがって、このページでは旧バージョンの「高機能画面録画プラグイン API マニュアル」を継続して保持することはできません。そうしないと、過去の記述が現在の実装と誤って記載されてしまいます。

## このページには現在どのような情報が保持されていますか?

現時点で残っている結論は 1 つだけです。

ScreenRecorder 関連のドキュメントは、現在のソース コードに基づいて検証できる API リファレンス ページではなく、このワークスペース内の履歴ページです。

このプラグインが後で再導入される場合は、少なくとも次の実際のアンカーに基づいて新しいドキュメントを書き直す必要があります。

- プラグインディレクトリとプロジェクトファイル
- `manifest.json`
- メニューまたはプロバイダーのアクセスポイント
- 録音ウィンドウと録音ソース管理の実装
- 構成と出力ドロップポイント

エンコード形式、録音ソースの種類、オーバーレイ API などの説明は、これらのコードが再登場するまで追加しないでください。

## 古い原稿を維持し続けてみませんか?

古いページでは、ScreenRecorder を現在の実際の画面録画プラグインとして説明し、録画ソース、エンコーダ、オーバーレイ、および高度な機能のリストを示しています。しかし、現在のソース コード ツリーには、これらの内容の検証可能な実装はありません。

古いドラフトを磨き続けると、ドキュメントはより完全に見えるだけですが、現在の倉庫からはどんどん切り離されていきます。
