# ImageProjector ステータスの説明

対応するソース コード プロジェクトが現在の `scgd_general_wpf` ワークスペースで見つからなくなったため、このページでは ImageProjector を現在のウェアハウスの標準プラグイン実装として記述しなくなりました。

## 現在のワークスペースの実際のステータス

現在の倉庫構造に従って確認してください:

- `Plugins/` ディレクトリの下に `ImageProjector/` ソース コード ディレクトリはありません。
- ワークスペースに対応するプラグインプロジェクトファイルがありません。
- 現在のプラグイン インデックス ページ [Plugins/README.md](../../../../Plugins/README.md) には、既存のプラグイン ディレクトリとしてリストされていません。
- 対応するプラグイン実装が現在のウェアハウスに存在することを示すためではなく、履歴ステータスの説明ページに引き続きアクセスできるようにするためだけに、現在のドキュメントのサイドバーに保持されます。

したがって、このページでは「マルチモニター投影ツール完全マニュアル」の旧バージョンを保持し続けることはできません。そうでない場合、過去の機能は現在のソース コードの事実として記述されます。

## このページには現在どのような情報が保持されていますか?

現時点で残っている結論は 1 つだけです。

ImageProjector 関連のドキュメントは、現在のソース コードに基づいて検証できる API リファレンス ページではなく、このワークスペース内の履歴ページです。

このプラグインが後で再導入される場合は、少なくとも次の実際のアンカーに基づいて新しいドキュメントを書き直す必要があります。

- プラグインディレクトリとプロジェクトファイル
- `manifest.json`
- メニューまたはプロバイダーのアクセスポイント
- メインウィンドウまたは投影ウィンドウの実装
- 構成の配置

このコードが再表示されるまでは、機能の説明、構成テーブル、または API リストに追加を加えないでください。

## 古い原稿を維持し続けてみませんか?

古いページでは、ImageProjector を現在存在するプラグインとして説明し、完全な機能リスト、表示モードの説明、およびコンポーネント構造を示します。しかし、現在のソース コード ツリーでは、これらの記述には検証可能な実装ベアラーがありません。

このまま文書を書き続けると、「過去に存在した可能性のある機能」が「現在の倉庫の事実」に偽装されてしまいます。今回のクリーンアップが避けようとしているのはまさにそれだ。
