## 現在の組織原則

- トップレベルのホームページには、要約された概要ページのみが表示され、すべての単一ページが最初の画面の入り口とは見なされなくなりました。
- セグメント化されたトピック ページはそれぞれのディレクトリに保持されますが、デフォルトではソース コードに対して読み取る必要があります。
- ドキュメントが実装と一致しない場合は、ソース コード、XML コメント、および実際の実行動作が優先されます。

## 現在の章の境界

- `ui-components/` は主に WPF UI 側モジュールとデスクトップ シェルをカバーします。
- `engine-components/` では、完全なアルゴリズム百科事典ではなく、主に Engine ディレクトリ内のランタイム モジュールについて説明します。
- `algorithms/` では、基盤となるすべてのイメージ オペレーター ディレクトリではなく、主にテンプレート システムとアルゴリズムのアクセス チェーンをカバーします。
- `plugins/` では、ソース コードとまだ一致する現在のワークスペースの標準プラグインと、少数の過去に残っているプラ​​グインの現在のステータス説明ページを主に取り上げます。
- `extensions/` は現在、主にフロー ノード拡張機能のトピックを予約しています。これは、実際のコードに直接一致する拡張ポイントです。

## 補足的な読み方

- `plugins/` の下のページには、プラグインのトピック「現在のソース コードの照合が可能」および「履歴ステータスの説明」ページも含まれています。単一ページに入る前に、章の概要を読む必要があります。
- `extensions/` 現在の範囲は非常に狭く、コード アンカーと直接一致するフロー ノードの拡張などのトピックに主に焦点を当てています。これは完全な拡張メカニズムの百科事典ではありません。
- どちらのタイプのページも、ユーザー ガイドや開発ガイドに代わる一般的なエントリ ポイントとしてではなく、オンデマンド アクセス ポイントとして適しています。

## 推奨される読む順序

1. まず、[UI コンポーネントの概要](./ui-components/README.md) を読んで、クライアント シェルと UI インフラストラクチャを理解します。
2. [エンジン コンポーネントの概要](./engine-components/README.md) をもう一度見て、サービス、テンプレート、プロセス ランタイムを理解します。
3. 最後に、[アルゴリズムとテンプレートの概要](./algorithms/README.md) と [アルゴリズム システムの概要](./algorithms/overview.md) を参照して、テンプレートとアルゴリズムのアクセス リンクを接続します。
4. プラグインまたは拡張ポイントのステータスを確認する必要がある場合は、[プラグインとステータス ページ] (./plugins/README.md) および [拡張ポイントの概要] (./extensions/README.md) に入ります。
