# プラグイン開発の概要

この章は、ColorVision の機能を拡張する必要がある開発者を対象としており、現在有効なプラグイン開発パスを優先します。

## ウェアハウス内のプラグインの場所

- ランタイム プラグインのソース コードは `Plugins/` にあります。
- プラグインは実行時にメイン プログラムによって検出され、ロードされます。
- プラグインに UI がある場合、通常は WPF を有効にし、メイン アプリケーションのインターフェイス規約に従う必要があります。

## プラグイン開発への最短パス

1. まず [拡張性の概要](../core-concepts/extensibility.md) を確認します。
2. [プラグイン開発入門](./getting-started.md) をもう一度見てください。
3. 実行フェーズを理解する必要がある場合は、[プラグインのライフサイクル](./lifecycle.md)を参照してください。

## 現在推奨されている規則

- ターゲット フレームは、メイン リポジトリと同じ Windows デスクトップの方向を維持します。
- インターフェースが必要な場合に WPF を有効にする
- 構築後、プロダクトをメインプログラム出力ディレクトリの `Plugins/<Name>/` にコピーします。
- 新しい規則セットを作成するのではなく、既存の標準プラグインの編成を優先します。
