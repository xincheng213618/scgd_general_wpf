# プラグイン開発の概要

このページでは、現在のウェアハウスで実際に利用可能なプラグイン開発モデルのみを説明し、古い一般化されたインターフェイスの例を引き続き使用することは避けます。

## 現在のプラグイン モデル

ColorVision のプラグインは、メイン プログラム実行ディレクトリの下の `Plugins/` 内の独立したディレクトリにデプロイされます。メイン プログラムが起動すると、各プラグイン ディレクトリをスキャンし、`manifest.json` を読み取り、マニフェストで指定された DLL に従ってアセンブリを読み込みます。

現在のコードで直接確認できるいくつかの重要なポイント:

- プラグインの基本インターフェイスは `UI/ColorVision.Common/Interfaces/IPlugin.cs` にあります。
- プラグイン マニフェストは `UI/ColorVision.UI/Plugins/PluginManifest.cs` にあります
- プラグイン読み込みロジックは `UI/ColorVision.UI/Plugins/PluginLoader.cs` にあります

## 主要なコンポーネント

### 1. プラグインインターフェース

現在リポジトリに表示されている基本インターフェイスは非常に軽量です。


```csharp
public interface IPlugin
{
    string Header { get; }
    string Description { get; }
    void Execute();
}
```


単純なプラグイン エントリを作成するだけの場合は、通常、`IPluginBase` から始める方が便利です。

### 2.マニフェスト.json

通常、プラグイン ディレクトリには `manifest.json` が必要です。現在のマニフェスト オブジェクトには、少なくとも次のフィールドが含まれています。

- `id`
- `manifest_version`
- `name`
- `version`
- `requires`
- `description`
- `dllpath`
- `author`
- `url`
- `entry_point`
- `icon`

最も中心的なものは、プラグインの識別、説明、および DLL パスです。 `entry_point` は、エントリ タイプを明示的に指定する必要がある場合に使用されます。

### 3. 読み込みプロセス

メイン プログラムが開始されると、`PluginLoader` は次のことを行います。

1. `Plugins/` の下のプラグイン ディレクトリをスキャンします。
2. 各ディレクトリの `manifest.json` を読み取ります。
3. マニフェストに基づいて DLL パスを計算します。
4. 依存関係とバージョンを確認します。
5. アセンブリをロードし、プラグイン情報を内部キャッシュに書き込みます。

プラグイン ディレクトリにマニフェストがない場合でも、プラットフォームは「ディレクトリ名と同じ名前の DLL」によってプラグインをロードしようとしますが、これは推奨される形式ではなくなりました。

## 推奨されるディレクトリ構造


```text
Plugins/
└── MyPlugin/
    ├── manifest.json
    ├── MyPlugin.csproj
    ├── MyPlugin.dll
    ├── README.md
    ├── CHANGELOG.md
    ├── Assets/
    └── Sources/ 或 *.cs/*.xaml
```


## 開発に関する提案

### プラットフォーム内での開発

- ウェアハウスに新しい `Plugins/<PluginId>/` プロジェクトを作成します。
- ビルド後、出力をメイン プログラム出力ディレクトリの `Plugins/<PluginId>/` にコピーします。
- 既存の標準プラグインのディレクトリとパッケージ化方法を優先します。

### 外部の自主開発

- プラグインの最終成果物は、直接コピーできる完全なディレクトリとして保存する必要があります。
- プラットフォームのメイン プログラムに付属する `ColorVision.*.dll` を再パッケージ化しないでください。
- サードパーティのランタイム依存関係は、プラグイン製品と一緒にリリースされる必要があります。

## 推奨される読む順序

1. まずは[プラグイン開発の概要](./README.md)をご覧ください。
2. [プラグイン開発入門](./getting-started.md) をもう一度見てください。
3. 読み込みフェーズと実行フェーズを理解する必要がある場合は、[プラグイン ライフ サイクル](./lifecycle.md) を参照してください。
4. 既製のプラグインを参照したい場合は、[パターンプラグイン](../../04-api-reference/plugins/standard-plugins/pattern.md)を直接参照してください。

## 説明

- 古いバージョンのドキュメントに記載されている `IPluginContext`、非同期ライフサイクル インターフェイス、および独立したプラグイン ホスト モデルは、現在のウェアハウスのメイン パスで直接表示される基本インターフェイスではありません。
- ドキュメントとコードの間に矛盾がある場合は、`UI/ColorVision.Common/Interfaces/IPlugin.cs`、`UI/ColorVision.UI/Plugins/PluginLoader.cs`、および既存のプラグイン プロジェクトが優先されます。
