# 導入の概要

このページには、現在のリポジトリでまだ使用されているデプロイメント エントリのみが保持され、Windows デスクトップ アプリケーション、インストーラー、および更新メカニズムの説明に重点が置かれています。

## 現在のデプロイメントオブジェクト

- `ColorVision/`: プログラム本体
- `ColorVisionSetup/`: 関連プログラムのインストールと更新
- `Scripts/`: 補助スクリプトのビルド、パッケージ化、公開
- `Plugins/`: 実行時にロードされるプラグイン ディレクトリ

## 現在の推奨パス

### 開発環境またはテスト環境

メイン プログラムをソースから直接ビルドして実行します。


```powershell
dotnet restore
dotnet build -p:Platform=x64
dotnet run --project ColorVision/ColorVision.csproj
```


### 配信環境

- インストーラーを使用して完全なデスクトップ プログラムを配布します
- 必要に応じてプラグイン ディレクトリとランタイムの依存関係を保持します
- オンライン更新が関係する場合は、[自動更新システム](./auto-update.md) を読み続けてください。

## 導入前の確認事項

- 対象環境はWindowsです
- メインアプリはx64用に構築されています
- ランタイムの依存関係とローカル DLL がパッケージとともに正しくエクスポートされる
- 必要な設定ファイルが出力ディレクトリにコピーされている

## 説明

- 古い Docker、クラウド デプロイメント、実稼働クラスターなどの手順は、デフォルトのデプロイメント パスではなくなりました。
- プロジェクトに特別な配信方法がある場合は、一般的な展開ページに積み上げ続けるのではなく、対応するプロジェクト ディレクトリまたはプロジェクト ドキュメントに個別に保持する必要があります。
