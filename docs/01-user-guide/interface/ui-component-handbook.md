# UI 组件使用手册

本页从操作者、测试工程师和现场交付人员的视角说明 ColorVision 里常见 UI 组件。它回答“什么时候打开、从哪里进入、能做什么、做完怎么算成功、失败先查什么”。

如果你要发布 DLL 或改源码，请转到 [UI DLL 组件手册](../../04-api-reference/ui-components/component-handbook.md) 和 [UI 组件目录](../../04-api-reference/ui-components/control-catalog.md)。本页不展开类库实现细节。

## 什么时候先看本页

- 看到界面组件名称，但不知道它在软件里负责哪类操作。
- 要给现场人员讲解主窗口、设置、日志、数据库、Socket、调度、图像编辑器等入口。
- 要判断一个问题应该归到 UI 操作、设备、流程、插件、项目包还是底层发布。
- 要确认某个窗口打开后应该看到哪些结果。

## 组件总览

| UI 组件 | 何时使用 | 常见入口 | 主要操作 | 完成标准 | 失败先查 |
| --- | --- | --- | --- | --- | --- |
| 主窗口工作台 | 进入系统后的日常操作起点 | 程序启动后自动进入 | 识别菜单、搜索、工作区、状态栏 | 能找到设备、流程、图像、日志、插件入口 | 插件是否加载、权限、语言、布局 |
| 顶部菜单 | 打开全局工具、设备、插件、帮助 | 主窗口顶部菜单 | 按业务选择工具或插件功能 | 对应窗口或功能正常打开 | 菜单项权限、插件状态、热键冲突 |
| 搜索框 | 不知道功能在哪时快速定位 | 主窗口搜索区域 | 输入关键词，打开命令或页面 | 搜到目标入口并可打开 | 关键词、插件加载、搜索索引 |
| 状态栏 | 查看服务、插件、Socket、调度状态 | 主窗口底部 | 查看提示或点击状态图标 | 状态与现场实际一致 | Provider 是否加载、服务配置、日志 |
| 设置窗口 | 修改全局配置、模块配置 | 设置/选项类菜单入口 | 查找设置项、修改、保存、重启验证 | 修改值重启后仍生效 | 配置路径、权限、字段类型、只读项 |
| 属性编辑器 | 编辑设备、模板、流程节点、配置对象参数 | 属性面板或配置弹窗 | 按分类查看参数，修改并保存 | 参数更新后对象行为符合预期 | 元数据、校验、只读字段、保存路径 |
| 日志查看器 | 排查启动、设备、流程、插件异常 | 帮助 -> 日志，默认 `Ctrl+L` | 按时间、级别、关键词过滤 | 找到第一条有意义错误 | 日志级别、时间点、模块名 |
| 终端 | 现场执行命令、脚本或查看文件 | 终端入口或工作区组件 | 运行命令、查看输出 | 命令能执行并返回清晰结果 | 当前目录、权限、环境变量 |
| 图像编辑器 | 查看图像、结果 overlay、ROI、视频、3D、伪彩 | 图像结果、文件打开、工作区编辑器 | 缩放、标注、测量、导入导出注释 | 图像和叠加层对应正确 | 文件路径、图像仍在写入、坐标映射 |
| 数据库浏览器 | 查看 MySQL/SQLite 表数据 | 工具 -> 数据库浏览器 | 选择数据源、表、搜索、分页、少量维护 | 找到目标记录或确认未落库 | 连接配置、搜索条件、页码、主键 |
| Socket 管理窗口 | 查看本机 TCP server 状态和消息 | 状态栏 Socket 图标 | 检查启用、端口、连接、消息历史 | 外部系统能连接并收发消息 | 端口占用、协议模式、Handler 加载 |
| 调度任务窗口 | 查看和维护 Quartz 定时任务 | 工具 -> 任务管理 | 新建、暂停、恢复、立即执行、看历史 | 下次执行时间和执行记录正确 | Cron、任务程序集、执行异常 |
| 工作区/文件树 | 管理 `.cvsln`、项目文件和编辑器 | Solution 工作区窗口 | 新建、打开、编辑文件、切换布局 | 文件能打开并用正确编辑器处理 | 文件类型、编辑器注册、布局缓存 |
| 插件市场 | 安装、查看和更新插件/DLL | 帮助 -> 插件市场 | 查看插件、版本、下载或更新 | 插件包下载后能被宿主加载 | 管理员权限、网络、manifest、版本 |
| 下载器 | 下载插件、更新包或外部资源 | 插件市场或下载窗口 | 新增任务、查看进度、重试 | 文件完整落到目标目录 | aria2c、路径权限、网络 |
| 第三方应用窗口 | 快速打开外部工具 | 工具或桌面辅助入口 | 扫描系统应用、添加自定义工具 | 外部程序能启动 | 路径、快捷方式、权限 |
| 向导窗口 | 执行分步骤初始化或配置 | 向导入口 | 按步骤填写、下一步、完成 | 每一步校验通过并生成配置 | 必填项、设备连接、输出路径 |
| 主题和通用弹窗 | 调整外观、查看提示、确认操作 | 设置、主题菜单、系统提示 | 切换主题、确认消息、查看加载状态 | 主题生效且提示不遮挡关键操作 | 资源字典、主题配置、窗口焦点 |

## 基础界面组件

### 主窗口工作台

主窗口是所有日常操作的入口。第一次打开系统时，先不要急着进入具体业务，先确认四件事：

| 检查项 | 你应该看到什么 | 异常时先查 |
| --- | --- | --- |
| 菜单 | 文件、工具、帮助、插件或项目相关菜单 | 插件是否加载、权限是否足够 |
| 搜索 | 可以输入关键词并返回命令或页面 | 搜索配置、模块是否注册 |
| 工作区 | 中间区域能打开窗口、图像、流程或编辑器 | 布局缓存、工作区状态 |
| 状态栏 | 服务、Socket、调度、日志等状态提示 | 状态栏 Provider、服务配置 |

主窗口只负责把入口组织起来，不直接代表某个业务流程。看到“功能没出现”时，先判断它是菜单未注册、插件未加载、权限不足，还是对应项目包没有启用。

### 菜单、搜索和状态栏

| 组件 | 使用建议 | 交接提醒 |
| --- | --- | --- |
| 菜单 | 对固定入口和管理员工具优先走菜单 | 菜单项可能来自宿主、UI 模块、插件或项目包 |
| 搜索 | 对不熟悉入口的新用户优先用搜索 | 搜不到不一定是功能不存在，可能是插件未加载 |
| 状态栏 | 对服务和后台任务优先看状态栏 | Socket、调度等状态图标可直接打开管理窗口 |

如果某个按钮没有响应，先打开 [日志查看器](./log-viewer.md)，按点击时间点查最近的 `Error` 或 `Warn`。

## 配置和参数编辑组件

### 设置窗口

设置窗口面向全局配置和模块配置。现场修改配置时建议遵循：

1. 先确认要改的是全局配置、设备配置、流程配置还是项目配置。
2. 改动前记录原值，尤其是端口、路径、数据库、Socket、文件服务器参数。
3. 修改后保存，并按需要重启应用或刷新对应服务。
4. 用状态栏、日志、设备页或项目页验证配置是否生效。

不要把设置窗口当成数据库维护工具。表数据、历史记录和业务结果应走 [数据库操作](../data-management/database.md)。

### 属性编辑器

属性编辑器用于编辑对象参数，常见对象包括设备、模板、流程节点、图元、插件配置和项目配置。

| 操作 | 正确做法 | 失败表现 |
| --- | --- | --- |
| 查参数含义 | 先看分类、显示名和描述 | 只看到字段名，说明元数据不够完整 |
| 修改路径 | 使用文件/文件夹选择控件 | 手写路径错误、权限不足 |
| 修改枚举 | 使用下拉选项 | 运行后模式不对 |
| 修改列表/字典 | 使用集合编辑窗口 | JSON 格式错误或保存失败 |
| 修改后验证 | 回到业务窗口执行一次最小动作 | 值未持久化或对象未刷新 |

如果属性编辑器没有出现预期控件，开发侧应检查对应属性是否有 `Category`、`DisplayName`、`Description`、自定义编辑器类型和可见性标记。

## 诊断组件

### 日志查看器

日志查看器是排障第一入口。现场最常用的查法：

| 问题类型 | 关键词建议 | 下一页 |
| --- | --- | --- |
| 启动失败 | `Error`、`DllNotFoundException`、插件名、配置文件名 | [常见问题](../troubleshooting/common-issues.md) |
| 设备连不上 | 设备名、端口、IP、`timeout`、服务名 | [设备服务概览](../devices/overview.md) |
| 流程失败 | 流程名、节点名、模板名、`failed` | [流程执行与调试](../workflow/execution.md) |
| 插件未出现 | 插件目录名、`manifest`、`deps.json`、DLL 名 | [现有插件能力说明](../../04-api-reference/plugins/README.md) |
| 数据没写入 | 表名、批次号、SN、导出路径 | [数据管理概览](../data-management/README.md) |

日志很多时，先按时间点缩小，再按级别过滤。不要一开始就只搜最终报错，因为真正原因经常在前几行。

### 终端

终端适合现场临时验证环境，例如查看目录、测试网络、运行脚本或启动辅助程序。使用时注意：

- 先确认当前目录和命令环境。
- 对会修改文件的命令，先确认目标路径。
- 运行脚本前优先看脚本文档或参数说明。
- 命令失败时，把输出和日志查看器里的同一时间段一起看。

终端不是普通操作者的必需入口；更适合交付工程师和开发支持。

## 数据、通信和调度组件

### 数据库浏览器

数据库浏览器适合确认结果是否落库、记录是否可查、表结构是否符合当前版本。

| 场景 | 操作顺序 | 完成标准 |
| --- | --- | --- |
| 查某个 SN 或批次 | 选数据源 -> 选库表 -> 搜索 SN/批次 | 能看到对应记录和时间 |
| 确认流程是否写库 | 先运行最小流程 -> 刷新表 -> 按时间查 | 新记录出现且字段完整 |
| 做少量维护 | 确认表有主键和可写权限 -> 编辑 -> 保存 -> 刷新 | 刷新后值仍正确 |

按钮灰掉时，先看数据源是否只读、表是否有主键、当前连接是否有效。

### Socket 管理窗口

Socket 管理窗口用于本机 TCP server 的状态查看和消息排查。通常从状态栏 Socket 图标进入。

| 检查项 | 正常表现 | 异常方向 |
| --- | --- | --- |
| 启用状态 | Socket server 已启用 | 配置未启用 |
| 监听端口 | IP/端口符合外部系统配置 | 端口冲突、防火墙、IP 错误 |
| 连接状态 | 外部系统连接后状态变化 | 网络、协议模式、客户端地址 |
| 消息历史 | 请求和响应能落到历史库 | 消息库、Handler、日志 |

项目包如果通过 Socket 触发，还要继续看 [项目说明](../../00-projects/README.md) 和对应项目页，确认事件名、字段和返回格式。

### 调度任务窗口

调度任务窗口负责 Quartz 定时任务，不负责流程编排本身。

| 操作 | 说明 |
| --- | --- |
| 新建任务 | 选择任务类型、填写名称、组名、Cron、优先级、超时和自定义配置 |
| 暂停/恢复 | 临时停用或恢复已有任务 |
| 立即执行 | 不等下一次 Cron，手动触发一次 |
| 执行历史 | 查看成功、失败、耗时和错误详情 |

如果任务看不到，先确认任务所在程序集是否被加载；如果任务不执行，先查 Cron、调度器状态和执行历史。

## 图像和结果查看组件

### 图像编辑器

图像编辑器不是只看图片，它承载结果查看、ROI/POI、标注、overlay、视频、伪彩、直方图、3D 和 CIE 等操作。

| 场景 | 推荐操作 | 完成标准 |
| --- | --- | --- |
| 查看检测结果 | 打开结果图 -> 对照 overlay -> 查属性 | 原图、结果层和坐标一致 |
| 标注 ROI/POI | 选择绘图工具 -> 画图元 -> 保存或导出注释 | 注释可重新导入或被结果页识别 |
| 调整显示 | 缩放、平移、伪彩、直方图 | 能看清关键区域，不改变原始数据 |
| 查看视频 | 播放/暂停、拖动进度、调整预览缩放 | 视频流畅且画面对应 |
| 查看 3D/CIE | 切换对应工具 | 数据适合可视化且窗口响应正常 |

图像显示异常时，先确认文件是否完整写入，再查坐标映射和 overlay 来源。

## 工作区和辅助组件

### 工作区、文件树和编辑器

工作区负责 `.cvsln`、文件树、编辑器标签页、终端和多图浏览。它适合组织项目文件，不负责执行 Engine 流程。

| 组件 | 用途 | 异常时先查 |
| --- | --- | --- |
| 文件树 | 浏览项目目录、新建文件或项目 | 路径、权限、模板注册 |
| 编辑器系统 | 按扩展名打开文本、图像、Markdown、十六进制等文件 | 编辑器注册、文件后缀 |
| 布局管理 | 保存和恢复停靠面板 | 布局缓存、窗口状态 |
| 多图浏览 | 批量查看图片和缩略图 | 文件类型、缩略图缓存 |
| 本地 RBAC | 管理用户、角色、权限和会话 | 登录状态、数据库、权限模式 |

如果目标是运行客户项目流程，不要停留在工作区文件树里，应转到对应项目窗口或 [项目说明](../../00-projects/README.md)。

### 插件市场、下载器、第三方应用和向导

| 组件 | 适合做什么 | 失败先查 |
| --- | --- | --- |
| 插件市场 | 查看插件、下载插件包、查看 DLL 版本和更新计划 | 管理员权限、市场地址、manifest、版本兼容 |
| 下载器 | 下载插件、更新包或外部资源 | 网络、aria2c、目标路径权限 |
| 第三方应用窗口 | 管理外部工具快捷入口 | 快捷方式、程序路径、启动权限 |
| 向导窗口 | 分步骤完成初始化或配置 | 必填项、设备状态、输出路径 |

插件市场只能说明包是否下载和安装到位，不能证明插件业务流程一定可用。业务能力仍要看 [现有插件能力说明](../../04-api-reference/plugins/README.md)。

## 组件边界

| 你看到的问题 | 先归到哪里 |
| --- | --- |
| 窗口打不开、菜单没有、按钮无响应 | UI 操作和日志 |
| 设备状态异常、相机无法采集、电机不动 | 设备服务 |
| 流程节点失败、结果没有生成 | 工作流程和 Engine |
| Socket 能连但字段不对 | 项目包协议或 Socket handler |
| 插件没有出现或版本不兼容 | 插件装载和插件市场 |
| UI DLL 缺失、native DLL 缺失 | UI DLL 发布和安装包 |
| 数据库查不到结果 | 数据管理、流程写库、项目导出 |

## 交接检查清单

交接 UI 操作时，至少现场演示一次：

- 从主窗口打开日志、设置、数据库浏览器、调度任务和插件市场。
- 修改一个安全配置项，确认保存和重启后仍生效。
- 打开一张图像，完成缩放、标注、属性查看和注释导出。
- 打开数据库浏览器，按时间或 SN 查一条结果记录。
- 如果启用 Socket，打开状态栏 Socket 管理窗口，确认端口、连接和消息历史。
- 如果启用调度，打开任务窗口，查看任务、立即执行一次并查看历史。
- 如果交付插件或项目包，确认对应菜单、状态栏或项目窗口出现，并能打开。
