# UI 组件使用手册

这页面向操作者、测试工程师和现场交付人员。它只回答：这个窗口什么时候打开、做完怎么算成功、失败先看哪里。

改源码或发布 DLL 时不要从这里开始，转到 [UI 组件目录](../../04-api-reference/ui-components/control-catalog.md)、[UI 运行时组件](../../04-api-reference/ui-components/ui-runtime-handoff.md) 和 [UI DLL 发布](../../04-api-reference/ui-components/publishing.md)。

## 先看哪类问题

| 你遇到的情况 | 先打开 |
| --- | --- |
| 不知道功能入口在哪里 | 主窗口搜索框、顶部菜单 |
| 某个按钮或窗口报错 | [日志查看器](./log-viewer.md) |
| 配置、端口、路径、语言要修改 | 设置窗口或 [属性编辑器](./property-editor.md) |
| 要确认结果有没有落库 | 数据库浏览器 |
| 外部软件连接或发指令异常 | Socket 管理窗口 |
| 定时任务没有执行 | 调度任务窗口 |
| 图片、ROI、overlay、标注不对 | 图像编辑器 |
| 插件要安装、升级或确认版本 | 插件市场 |
| 要查看项目文件、脚本或终端 | Solution 工作区、[终端](./terminal.md) |

## 组件地图

| 组件 | 什么时候用 | 成功标准 | 失败第一步 |
| --- | --- | --- | --- |
| 主窗口工作台 | 日常操作起点 | 能看到菜单、搜索、工作区和状态栏 | 插件是否加载、权限、布局 |
| 顶部菜单 | 打开固定工具、插件、帮助 | 目标窗口能打开 | 菜单权限、插件状态 |
| 搜索框 | 不熟悉入口时定位功能 | 搜到并能打开目标入口 | 关键词、插件加载 |
| 状态栏 | 看后台服务、Socket、调度、日志状态 | 状态和现场实际一致 | Provider、服务配置 |
| 设置窗口 | 修改全局或模块配置 | 保存后重启仍生效 | 配置路径、权限 |
| 属性编辑器 | 改设备、模板、流程节点参数 | 对象行为符合新参数 | 只读项、校验、保存 |
| 日志查看器 | 排查启动、设备、流程、插件异常 | 找到第一条有意义错误 | 时间点、级别、模块名 |
| 数据库浏览器 | 查 MySQL/SQLite 记录 | 找到目标记录或确认未写入 | 连接、条件、页码、主键 |
| Socket 管理窗口 | 查本机 TCP 服务和消息 | 外部系统能连接并收发 | 端口、协议模式、handler |
| 调度任务窗口 | 查 Quartz 定时任务 | 下次执行时间和历史正确 | Cron、任务类型、异常 |
| 图像编辑器 | 看图像、ROI、overlay、标注 | 原图、结果层和坐标一致 | 文件完整性、坐标映射 |
| 插件市场 | 安装、更新、查看插件包 | 包下载后宿主能加载 | 网络、manifest、版本 |
| 下载器 | 下载插件或更新包 | 文件完整落到目标目录 | `aria2c`、路径权限、网络 |
| 工作区/文件树 | 管理 `.cvsln` 和项目文件 | 文件能用正确编辑器打开 | 文件类型、权限、编辑器注册 |
| 终端 | 现场执行命令或脚本 | 命令返回清晰结果 | 当前目录、权限、环境变量 |
| 向导窗口 | 分步骤初始化或配置 | 每一步校验通过并生成配置 | 必填项、设备、输出路径 |

## 最常用流程

| 场景 | 做法 |
| --- | --- |
| 功能找不到 | 先搜功能名、插件名或设备名；再看顶部菜单、状态栏和启动日志；只有某台机器缺失时对比插件目录、配置和权限 |
| 配置修改 | 先判断是全局、设备、流程节点还是项目参数；修改前记录端口、数据库、Socket、路径和设备参数；保存后用最小流程验证 |
| 现场排障 | 先按时间点看日志；设备看设备页，流程看节点和执行历史，数据看数据库和导出路径，外部系统看 Socket 连接和响应码 |
| 图像和结果确认 | 先确认原图完整写入，再检查缩放、坐标、ROI/POI、overlay 和绑定结果；结果存在但不显示时再查 Engine 结果链路 |

## 组件边界

| 问题 | 归属 |
| --- | --- |
| 窗口打不开、菜单没有、按钮无响应 | UI 操作、插件加载、权限、日志 |
| 相机、光谱仪、电机、串口异常 | 设备服务 |
| 流程节点失败、模板结果为空 | 工作流程和 Engine |
| Socket 能连但字段或业务不对 | 项目包协议或 Socket handler |
| 插件没有出现或版本不兼容 | 插件装载和插件市场 |
| DLL/native 依赖缺失 | 安装包或 UI DLL 发布 |
| 数据库查不到结果 | 数据管理、流程写库、项目导出 |

## 交付检查

现场交付时至少演示一次：

| 检查项 | 通过标准 |
| --- | --- |
| 打开设置、日志、数据库、Socket、调度、插件市场 | 窗口能打开且没有启动级错误 |
| 修改一个安全配置项 | 保存、重启后仍生效 |
| 打开一张图像 | 能缩放、查看属性、看到预期 overlay |
| 查询一条结果记录 | 能按时间、SN 或批次找到记录 |
| 启用 Socket 时 | 能看到端口、连接和消息历史 |
| 启用调度时 | 能看到任务、立即执行一次并查看历史 |
