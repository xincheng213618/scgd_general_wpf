# 终端

终端面板适合做两类事情：运行脚本，和直接观察命令行输出。它更像主程序里的工作辅助区，而不是单独的开发文档入口。

## 什么时候应该打开它

- 需要直接运行 PowerShell 或 CMD 命令
- 想在当前工程上下文里执行脚本
- 需要观察脚本输出、报错或交互提示
- 想快速做一次命令行检查，而不切出主程序

## 你通常会怎么用

### 选择 Shell

终端通常支持至少两种常见 Shell：

- PowerShell
- CMD

如果当前命令依赖 PowerShell 语法，就不要在 CMD 里直接重试；先确认 Shell 类型再执行。

### 运行脚本

常见做法包括：

- 直接输入命令运行脚本
- 从资源管理或相关入口触发“在终端中运行”
- 在排查时先跑最小命令，再逐步追加参数

### 看输出和报错

- 优先先看第一条错误
- 不要只盯最后一行
- 如果脚本还会继续输出，先确认它是不是仍在运行，而不是假设已经卡死

## 建议的使用顺序

1. 先确认当前要用的是 PowerShell 还是 CMD。
2. 再确认当前工作目录是不是你预期的工程位置。
3. 先执行最小命令，确认环境、路径和权限都正常。
4. 命令失败时，优先结合 [日志查看器](./log-viewer.md) 和终端输出一起判断。

## 常见问题

### 命令能运行，但结果和预期不一致

- 先确认当前 Shell 类型
- 再确认当前目录、环境变量和脚本参数
- 如果脚本依赖外部程序，先确认这些程序是否真的可用

### 终端没有输出，或看起来没反应

- 先确认命令是不是还在等待输入
- 再确认是否切到了错误的 Shell
- 如果是长任务，先给它一点时间输出第一批结果，不要马上重复提交命令

### 运行脚本时报路径或权限错误

- 先检查路径是否正确
- 再确认脚本文件和依赖文件是否存在
- 涉及 PowerShell 脚本时，优先确认执行策略和调用方式是否正确

### 交互式命令表现异常

- 先新建一个新的终端会话重试
- 再尝试改用更简单的命令确认是否是交互命令本身的问题
- 如果只是想看结果，不一定要在终端里完成全部交互
