# 主窗口导览

主窗口是 ColorVision 的工作台。大多数日常操作都会从这里进入，包括设备、流程、图像查看、日志、搜索和状态监控。

## 进入主窗口后先看哪里

### 顶部菜单和搜索区

- 菜单是主要功能入口
- 搜索框用于快速定位命令、页面或对象
- 当你不知道功能藏在哪时，优先先用搜索

### 左侧导航区

- 常用于查看工程、资源树或相关视图页签
- 如果当前页面内容很多，先从左侧结构切入通常更快

### 中间主工作区

- 显示当前正在操作的核心内容
- 可能是图像视图、流程窗口、模板窗口或其他业务界面
- 布局可能根据当前工作模式切换为单视图或多视图

### 底部状态栏

- 用来观察系统状态、提示信息和部分快捷入口
- 当设备、服务或插件状态异常时，先注意这里有没有提示

## 你通常会在主窗口做什么

- 打开设备相关窗口
- 进入流程设计和执行页面
- 查看图像或结果窗口
- 打开日志、终端和配置界面
- 搜索功能入口或对象位置

## 建议的熟悉顺序

1. 先认识主菜单、搜索区和状态栏。
2. 再理解左侧导航和主工作区是如何配合的。
3. 接着进入 [属性编辑器](./property-editor.md) 和 [图像编辑器概览](../image-editor/overview.md)。
4. 涉及硬件时，继续看 [设备服务概览](../devices/overview.md)。
5. 需要自动化时，进入 [工作流程](../workflow/README.md)。

## 常见问题

### 找不到某个功能入口

- 先用搜索框搜关键词
- 再回到对应章节首页重新进入
- 如果是插件功能，检查插件是否已经被加载

### 打开的窗口太多，不知道当前在看什么

- 回到左侧导航重新确认当前对象
- 观察主工作区标题和状态栏信息
- 按业务重新进入一次对应页面，而不是在多个窗口间盲切

### 状态栏没有出现预期提示

- 先确认相关服务、设备或插件是否已经真正启动
- 再查看 [日志查看器](./log-viewer.md) 获取更明确的信息
