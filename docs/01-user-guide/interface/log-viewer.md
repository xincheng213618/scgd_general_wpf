# 日志查看器

日志查看器用于定位启动异常、设备通信问题、流程执行错误和插件装载失败。遇到“界面没反应、功能没生效、设备连不上”这类问题时，先看日志通常比盲猜更快。

## 什么时候应该打开它

- 主窗口或某个功能打开失败
- 设备连接状态异常
- 流程执行报错但界面提示不够明确
- 插件、服务或模板加载行为异常

## 你通常会用到的功能

- 实时查看新日志
- 搜索关键词
- 按日志级别过滤
- 查看一定范围的历史日志
- 在自动滚动和手动阅读之间切换

## 基本使用方式

1. 从帮助菜单或对应入口打开日志窗口。
2. 先看最近一段日志，确认错误发生的大致时间点。
3. 用关键词搜索模块名、设备名、异常关键字或文件名。
4. 如果日志太多，再把级别提高到 `Warn` 或 `Error` 缩小范围。

## 搜索时怎么更高效

- 先用简单关键词，例如设备名、模板名、`error`、`timeout`
- 结果过多时，再组合多个关键词
- 只有确定需要时再用正则表达式

如果输入后搜索框变红，通常说明正则写错了；此时先退回普通关键词搜索更稳。

## 常见问题

### 打开日志窗口后几乎没有内容

- 先检查自动刷新是否开启
- 再检查当前日志级别是否过高
- 如果只想看新日志，确认当前加载策略不是空

### 日志很多，界面不容易看

- 先把级别切到 `Warn` 或 `Error`
- 再用关键词缩小范围
- 若只是临时排障，先不要追求保留全部历史日志

### 搜索结果不对

- 先确认是否把多个关键词都写成了必须同时匹配
- 如果用了正则，先确认特殊字符是否需要转义
- 搜索不到时，回到未过滤视图重新找时间点

## 排查建议

### 设备问题

- 先搜设备名、相机代码、服务名
- 再联动看 [设备服务概览](../devices/overview.md)

### 流程问题

- 先搜模板名、节点名或执行失败关键字
- 再联动看 [流程执行与调试](../workflow/execution.md)

### 插件或启动问题

- 先看应用启动阶段的最近日志
- 再关注是否有 DLL、版本、路径或加载失败提示
