# 流程执行与调试

本页只保留当前代码里能证实的执行入口和排查路径。流程跑不起来时，先确认模板、起始节点、注册中心、服务令牌和预处理，不要按旧文档里的断点/单步说明排查。

## 执行前先确认什么

| 检查项 | 失败表现 |
| --- | --- |
| 已选中有效流程模板 | 打开了执行页，但没有真正开始 |
| 流程存在起始节点 | 直接失败，不进入执行阶段 |
| 注册中心已连接 | 提示刷新服务或服务列表为空 |
| 服务令牌可用 | 无法派发到设备/算法服务 |
| 预处理通过 | 一开始就取消，不继续往下跑 |

## 常见执行顺序

1. 先在 [流程设计](./design.md) 中确认当前流程内容和起始节点。
2. 在执行窗口选中正确的流程模板。
3. 确认相关设备服务已经在线。
4. 点击运行入口开始执行。
5. 执行过程中观察日志区、当前运行节点和进度变化。
6. 需要中止时使用停止入口。

## 执行时你能看到什么

| 信息 | 用法 |
| --- | --- |
| 当前运行节点 | 卡住时先看停在哪个节点 |
| 进度和耗时 | 用来判断大致阶段，不当作严格业务完成度 |
| 结果和状态 | 完成后记录批次状态、耗时和摘要；停止会记为取消 |
| 日志区 | 优先看最后一个节点、预处理失败、取消执行或状态消息 |

## 流程运行要记录什么

流程运行记录不能只保存一份流程文件。维护人员需要知道它依赖哪些设备、模板、输入、结果和外部系统。

| 记录项 | 要写清楚什么 | 用途 |
| --- | --- | --- |
| 流程模板 | 模板名称、版本、导入来源、最后修改人 | 防止跑错旧流程 |
| 起始条件 | 起始节点、SN/批次输入、项目窗口或外部触发方式 | 判断流程为什么没有开始 |
| 设备依赖 | 相机、电机、SMU、文件服务等节点绑定 | 排查设备层失败 |
| 模板依赖 | 图像处理模板、校准模板、判定阈值 | 排查结果偏差 |
| 数据去向 | 数据库表、导出文件、图片目录、Socket/MES 返回 | 确认流程完成后结果在哪里 |
| 失败证据 | 第一个失败节点、日志时间点、错误信息 | 让后续人员能复现而不是重猜 |

## 最小复测脚本

现场复测或升级后，不要一开始就跑完整生产链路。先用下面顺序确认流程能稳定进入和退出。

1. 打开流程设计页，确认流程模板和起始节点存在。
2. 打开执行页，选择同一条流程模板。
3. 确认相关设备服务在线，必要时先做一次设备最小动作。
4. 准备一组不会影响正式产线的 SN、图片或测试输入。
5. 点击运行入口执行，记录开始时间、当前节点、最终状态和耗时。
6. 查询同一轮结果：日志、图像、数据库、导出文件或外部响应至少要能对应上。
7. 如需停止，点击停止入口后确认状态被记录为取消，而不是停在未知状态。

## 失败分叉

| 失败位置 | 典型表现 | 优先排查 |
| --- | --- | --- |
| 执行前 | 按运行没有反应、提示刷新服务、起始节点缺失 | 注册中心、服务列表、流程模板、起始节点 |
| 预处理 | 一开始就取消或提示预处理失败 | 输入参数、模板合法性、项目窗口上下文 |
| 设备节点 | 等待超时、设备无响应、返回码异常 | 对应设备页、硬件连接、设备 Code、MQTT/串口/IP |
| 模板节点 | 执行完成但结果不对 | 模板版本、阈值、图片来源、校准数据 |
| 数据节点 | 流程完成但看不到结果 | 数据库写入、批次/SN、导出对象、权限 |
| 外部系统 | ColorVision 完成但 MES/Socket 没收到 | 协议配置、端口、项目 handler、响应字段 |

## 排查顺序

1. 先分清是执行前被拦住，还是已经开始后中途失败。
2. 已经开始的流程，先找第一个失败或没有继续推进的节点。
3. 设备节点回设备页查连接、Code、MQTT/串口/IP。
4. 模板节点回模板版本、阈值、图片来源和校准数据。
5. 数据或外部系统问题，再查数据库、导出对象、Socket/MES handler。
