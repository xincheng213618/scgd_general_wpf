# 流程执行与调试

介绍如何执行和调试工作流程。

## 执行流程

### 开始执行

1. 打开流程文件
2. 点击工具栏"执行"按钮或按 `F5`
3. 查看执行进度和结果

### 执行模式

- **正常模式**: 连续执行所有节点
- **单步模式**: 逐节点执行，便于调试
- **断点模式**: 在断点处暂停

## 调试功能

### 设置断点

1. 在节点上右键
2. 选择"设置断点"
3. 节点标记为红色

### 单步执行

1. 按 `F10` 或点击"单步执行"
2. 流程在当前节点暂停
3. 查看节点输入输出数据

### 查看数据

- **节点输出**: 在属性面板查看
- **中间结果**: 双击节点查看详细数据
- **日志**: 查看执行日志

## 监控执行

### 执行状态

节点状态指示：
- 🟢 绿色 - 执行成功
- 🔴 红色 - 执行失败
- 🟡 黄色 - 正在执行
- ⚪ 灰色 - 未执行

### 性能监控

- 查看每个节点的执行时间
- 识别性能瓶颈
- 优化慢速节点

## 异常处理

### 错误提示

- 节点执行失败会显示错误信息
- 在日志中查看详细堆栈

### 异常恢复

- **跳过**: 跳过失败的节点继续执行
- **重试**: 重新执行失败的节点
- **停止**: 停止整个流程

## 执行结果

### 查看结果

- 在结果面板查看处理后的图像
- 查看数据统计和报告
- 导出结果到文件

### 结果保存

- 自动保存到数据库
- 导出为 Excel/CSV
- 生成 PDF 报告
