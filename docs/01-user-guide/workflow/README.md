# 工作流程

本章节面向日常使用，不解释 FlowEngine 架构本身，而是帮助你判断什么时候该设计流程，什么时候该去执行和调试。

## 工作流程适合解决什么问题

在 ColorVision 里，工作流程通常用来把设备动作、模板处理和结果输出串成一条可重复执行的业务链路。

常见场景包括：

- 按固定步骤调用多个设备或服务
- 复用一套检测或处理顺序
- 在执行失败时快速定位卡在哪个节点
- 把人工操作尽量收敛成可保存、可重复的流程

## 本章节怎么读

### 要搭流程时

- [流程设计](./design.md)

适合的场景：

- 新建一条流程
- 调整节点顺序、连接关系或参数
- 想确认某一步应该放在哪个位置

### 要跑流程或排错时

- [流程执行与调试](./execution.md)

适合的场景：

- 手动执行已有流程
- 观察执行状态和结果
- 定位第一个失败节点

## 建议的使用顺序

1. 先明确要串起来的是设备、模板还是数据处理步骤。
2. 进入 [流程设计](./design.md) 搭出最小可运行版本。
3. 再到 [流程执行与调试](./execution.md) 验证每一步是否真的按预期执行。
4. 如果流程涉及设备服务，回头核对对应设备页，而不是只在流程窗口里反复重试。

## 常见问题

### 流程能打开，但跑不通

- 先不要同时改很多节点
- 先定位第一个失败节点
- 再去核对该节点依赖的设备、模板或输入数据

### 流程执行结果和预期不一致

- 先确认当前运行的是不是正确的流程版本
- 再确认节点参数和输入对象是否对应当前项目
- 如果结果落库或导出异常，再联动看 [数据管理](../data-management/README.md)

### 不确定问题出在流程还是设备

- 先看 [流程执行与调试](./execution.md)
- 再回到相关设备专题页确认服务状态和配置
- 同时结合 [日志查看器](../interface/log-viewer.md) 缩小范围
