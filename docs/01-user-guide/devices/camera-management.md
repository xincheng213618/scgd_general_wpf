# 物理相机管理

本页说明物理相机这一层负责什么。它更像“硬件资源台账”，负责发现、授权和维护真实相机，而不是直接代替相机服务。

## 什么时候应该先来这里

- 首次接入新相机
- 导入或更新许可证
- 刷新可创建的相机 ID
- 管理校准资源、分组和物理配置
- 逻辑相机服务找不到可绑定对象

## 物理相机层负责什么

- 创建物理相机对象
- 导入 `.lic` 或 `.zip` 许可证
- 维护物理相机配置和资源树
- 为相机服务提供可绑定的相机代码和基础参数

这意味着“硬件有没有被系统真正识别”和“许可证是否有效”这类问题，通常都应该先在这一层排查。

## 常见操作顺序

1. 接好设备并确认系统能够看到空闲相机 ID。
2. 在物理相机管理中创建相机，或导入许可证。
3. 检查相机名称、代码、模式和基础配置。
4. 如有需要，准备校准、分组和相关资源。
5. 返回 [相机服务](./camera.md)，把逻辑服务绑定到这个物理相机。

## 配置时要特别确认什么

- 许可证是否有效
- 是否真的存在未创建的相机 ID
- 物理相机资源是否完整
- 相机模式是否和实际硬件一致
- 修改后是否需要回到逻辑相机服务重新保存或刷新

## 常见问题

### 找不到未创建的相机

- 确认硬件已经连接并被系统识别
- 先刷新设备 ID 再重试
- 如果刚插拔过设备，重新打开窗口后再看一次

### 导入许可证后仍不可用

- 确认导入文件格式正确
- 确认许可证对应的相机代码与当前硬件一致
- 检查数据库或本地资源是否已经同步

### 改了物理相机配置，现有相机服务行为异常

- 检查逻辑服务是否需要重新保存或重新绑定
- 关注配置变更后是否需要重启相关服务
- 若涉及路径和资源调整，确认相关文件已经落到新位置
