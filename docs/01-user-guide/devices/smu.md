# SMU 服务

SMU 服务用于源测量一体化操作，例如给定电压测电流、给定电流测电压，以及按模板执行扫描。它适合点测、IV 扫描，或把电学结果联动到后续流程。

## 什么时候看

- 新增或配置 SMU 设备服务。
- 在 A/B 通道之间切换测量。
- 做手动点测或执行扫描模板。
- 确认结果在哪里显示、如何被后续窗口使用。

## 核心概念

| 概念 | 说明 |
| --- | --- |
| 通道 | 常见为 Channel A / Channel B；操作和结果显示都要先确认当前通道 |
| 源类型 | 电压源测电流，或电流源测电压 |
| 手动点测 | 使用当前显示配置里的测量值和限值 |
| 扫描 | 使用 `SMUParam` 模板里的起点、终点、点数和限值 |

如果改了显示配置但扫描结果没有变化，先确认实际执行的是不是模板扫描。

## 使用顺序

1. 在设备列表里确认 SMU 服务已经创建并能连接。
2. 进入 SMU 窗口，确认当前通道和源类型。
3. 手动点测时，先设置测量值和限值，再执行一次读数。
4. 扫描时，先确认 `SMUParam` 模板，再执行扫描。
5. 结果出来后，先看当前窗口结果视图，再决定是否交给流程或其他服务。

## 操作注意

| 场景 | 先确认 |
| --- | --- |
| 手动点测 | 当前是电压源还是电流源，输入的是测量值还是限值，限值保护是否拦截，读数属于哪个通道 |
| 扫描 | 模板里的源类型、起始值、结束值、限值、点数、通道是否匹配 |
| 结果联动 | 后续窗口读取的是当前通道值、SMU 结果视图，还是显示配置 |

## 结果位置

- 单次测量结果回到 SMU 结果视图。
- 扫描结果也会在 SMU 视图中新增结果项。
- 当前选中通道的电压、电流读数会同步回显示配置。
- 部分电压、电流结果会联动给依赖这些值的其它窗口或服务。

## 常见问题

| 现象 | 先查 |
| --- | --- |
| 点测没有返回结果 | 设备是否打开、通道是否选对、源类型/测量值/限值是否合理 |
| 扫描结果和手动设置不一致 | 执行的是扫描模板还是手动点测，模板起止值/限值/点数是否正确 |
| 两个通道数据看起来串了 | 当前显示通道、最近一次操作通道，按通道理解结果 |
| 结果出来但后续流程没用到 | 后续流程或窗口读取哪个通道和值，必要时看 [流程执行与调试](../workflow/execution.md) |
