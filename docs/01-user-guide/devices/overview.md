# 设备服务概览

本页作为设备章节入口，回答“有哪些设备页可看、通常怎么配置、遇到问题先查哪里”。

## 设备服务是什么

在 ColorVision 里，设备通常以“服务”的形式被管理。主程序维护设备服务列表，用户在设备窗口中查看、配置、启用和操作这些服务。设备实现主要位于 `Engine/ColorVision.Engine/Services/` 和 `Services/Devices/`。

当前代码目录中可以看到 Camera、Calibration、Motor、FileServer、FlowDevice、Sensor、SMU、Spectrum 等典型设备分类。

## 使用顺序

1. 先看添加与配置设备。
2. 再进入具体设备页，确认参数和操作。
3. 相机相关继续看相机管理和相机参数配置。
4. 需要让设备参与自动化流程时，再看流程章节。

## 现场记录

现场交付设备时，不要只写“设备已配置”。至少记录：

| 记录项 | 要写清楚什么 | 用途 |
| --- | --- | --- |
| 设备身份 | 名称、Code、设备类型、真实硬件或服务 | 防止流程绑定错设备 |
| 通信配置 | IP、端口、串口、波特率、MQTT 主题、文件路径 | 复现现场连接条件 |
| 最小动作 | 相机拍图、电机回零、SMU 点测、文件服务列目录 | 证明设备不是只出现在列表里 |
| 流程绑定 | 被哪条流程、哪个节点或哪个项目窗口引用 | 排查“手动能用、流程失败” |
| 日志证据 | 成功连接时间点、失败错误、超时或权限信息 | 给远程排查留线索 |
| 回退方式 | 上一版配置、驱动、固件或项目包位置 | 升级异常时能退回 |

## 现场最小验收

| 步骤 | 操作 | 通过标准 |
| --- | --- | --- |
| 1 | 打开设备列表并刷新 | 目标设备可见，名称和 Code 与现场记录一致 |
| 2 | 打开设备详情或专用窗口 | 关键参数可见，保存状态明确 |
| 3 | 执行一个低风险最小动作 | 有返回、有日志、状态变化可解释 |
| 4 | 打开依赖它的流程或项目窗口 | 能选到同一台设备 |
| 5 | 跑最小流程或模拟流程 | 设备节点能执行到，并能定位结果或失败原因 |

第 3 步失败先查设备层；第 3 步通过但第 5 步失败，优先查流程绑定、模板参数和项目映射。

## 分层定位

| 现象 | 先查 | 再查 |
| --- | --- | --- |
| 列表里没有设备 | 是否创建并保存、类型是否正确 | 插件/项目包是否提供该设备入口 |
| 设备有但打不开 | 硬件在线、驱动、端口/IP、权限 | 日志里的连接异常和超时 |
| 手动动作成功但流程失败 | 流程节点引用的设备 Code | 节点参数、模板版本、项目窗口选择 |
| 结果有但图像或数据不对 | 设备参数和采集条件 | 后续模板、导出字段或数据库记录 |
| 升级后才异常 | 配置是否被覆盖 | 旧包、旧配置和新 DLL 是否混用 |
