# 文件服务

文件服务用于把远端文件列表、文件下载和文件上传纳入统一设备体系。它适合在“结果文件在服务端”“需要从设备端取图”“需要把文件送到远端”这类场景下使用。

## 什么时候应该看这一页

- 你要新增或配置一个文件服务
- 你要查看远端可用文件列表
- 你要下载某个源文件到当前系统
- 你要上传文件到远端服务
- 你在排查“文件看得到但打不开”“上传后没反应”这类问题

## 这个服务通常怎么用

### 先完成配置

先保证设备代码、发送主题和订阅主题这些基础配置是正确的。文件服务很多问题看起来像“文件操作失败”，实际更早就卡在连接或主题配置了。

### 再看文件列表

如果你都不知道远端现在有哪些文件，先取文件列表，再决定要下载还是上传，不要直接猜文件名。

### 需要内容时再打开或下载

当前实现里常见文件动作包括：

- 获取全部文件列表
- 打开或下载指定文件
- 上传指定文件

## 常见使用顺序

1. 先在设备列表里创建并保存文件服务。
2. 确认连接相关配置无误。
3. 先拉一次文件列表，确认远端确实有目标文件。
4. 再执行下载或打开动作。
5. 需要回传文件时，再执行上传。

## 使用时要注意什么

### 文件名要先确认

文件服务是按文件名发起操作的。文件名错了，后面再看显示窗口意义不大。

### 列表和内容是两步

能看到列表，不代表具体文件一定能正常打开；同样，上传成功也不代表你已经在正确目录或正确类型下查看到了它。

### 先排除主题和设备代码问题

当前实现对设备代码和订阅主题有明确匹配。遇到没有响应时，优先先查这些基础条件。

## 常见问题

### 文件列表拉不回来

- 先确认发送主题和订阅主题是否正确
- 再确认当前设备代码是否和返回消息匹配
- 如果服务端有心跳但没有文件事件，优先查具体文件操作配置

### 文件能列出来，但打开失败

- 先确认文件名是否正确
- 再确认请求的是不是目标文件类型
- 如果问题持续，结合 [日志查看器](../interface/log-viewer.md) 看返回消息里有没有更明确的错误信息

### 上传后看不到文件

- 先确认上传动作是否真的发出并完成
- 再重新拉一次文件列表，而不是只依赖旧列表
- 如果远端有目录或类型区分，确认你查看的是正确范围

### 文件服务在流程里不稳定

- 先用手动方式验证文件列表、下载或上传能否成功
- 再回到 [流程执行与调试](../workflow/execution.md) 看流程里用的是不是同一台文件服务
- 必要时同时检查设备代码和主题配置是否与手动测试一致
