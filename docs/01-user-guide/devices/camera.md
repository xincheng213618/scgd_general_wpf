# 相机服务

本页说明“相机服务”这一层在日常使用中的作用，以及它和物理相机、参数配置、校准模板之间的关系。

## 相机服务和物理相机不是一回事

- 物理相机：真实硬件、许可证、资源和基础能力
- 相机服务：项目里真正被设备列表、视图窗口和流程调用的逻辑设备
- 一台相机服务通常要先绑定一个物理相机，之后才能稳定使用采图、视频、校准和模板功能

## 你通常会在这里做什么

- 编辑相机服务配置
- 打开相机视图或实时画面
- 绑定或切换物理相机
- 进入自动曝光、自动对焦、相机参数模板
- 打开校准参数编辑窗口
- 查看本地调试入口和相机日志

## 推荐使用顺序

1. 先在 [物理相机管理](./camera-management.md) 中确认硬件、许可证和资源可用。
2. 回到相机服务，把当前服务实例绑定到正确的物理相机。
3. 在 [相机参数配置](./camera-configuration.md) 中调整曝光、增益、通道和模式。
4. 需要时再进入自动曝光、自动对焦或校准模板。
5. 用实时画面、本地窗口或流程执行验证结果。

## 这页对应的能力范围

当前代码里，相机服务这一层已经包含了几类常用入口：

- 编辑相机配置
- 刷新设备 ID 和物理相机绑定
- 自动曝光模板
- 自动对焦模板
- 相机参数模板
- 校准模板
- 相机日志和本地窗口

因此遇到“相机能不能被系统正常使用”的问题，通常先看相机服务，而不是直接去看底层相机资源。

## 常见问题

### 相机服务存在，但打不开或采不到图

- 先检查是否已经绑定物理相机
- 再检查许可证、连接状态和相机日志
- 若刚改过配置，重新打开相关窗口或重启服务后再试

### 改了物理相机配置后，相机服务没同步

- 保存后重新进入相机服务配置
- 检查当前服务是否绑定到了预期的物理相机
- 必要时刷新设备 ID 或重新绑定一次

### 视频或本地模式异常

- 先关闭当前视频或实时模式，再做重连或重启
- 优先排查是连接问题还是参数问题
- 需要时查看相机日志确认后台服务状态
