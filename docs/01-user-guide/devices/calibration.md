# 校准服务

校准服务用于把校准流程和实际相机绑定起来，管理校准模板、触发校准数据生成，并在需要时处理原始文件和缓存。它不是单纯的“参数页”，而是连接物理相机、模板编辑和校准结果的一层服务。

## 什么时候应该看这一页

- 你要新建或配置一个校准服务
- 你要把校准服务绑定到某个物理相机
- 你要编辑校准模板
- 你要触发一次校准相关的数据获取
- 你在排查“为什么不能开始校准”“为什么模板打不开”这类问题

## 使用前先确认两件事

### 先绑定物理相机

当前实现里，校准服务会直接依赖物理相机代码。如果没有先配好对应物理相机，很多校准相关操作根本不会继续。

### 再确认数据库可用

如果当前环境启用了 MySQL，但数据库没有连通，校准模板编辑不会正常进入。遇到这种情况，先修数据库连接，不要反复点模板窗口。

## 常见使用顺序

1. 先在设备列表里创建或确认校准服务。
2. 把它绑定到正确的物理相机。
3. 打开校准模板编辑入口，确认模板内容和当前相机对应。
4. 需要取校准数据时，再执行相关校准动作。
5. 如果物理相机配置变了，记得回头确认校准服务是否仍然绑定正确。

## 这个服务里最常见的操作

### 管理物理相机

如果当前校准服务对应的相机还没准备好，先去物理相机管理页处理，而不是只停留在校准窗口里排查。

相关入口：

- [相机管理](./camera-management.md)
- [相机参数配置](./camera-configuration.md)

### 编辑校准模板

校准模板编辑依赖当前绑定的物理相机。模板窗口能否正常打开，本身就是一个很有价值的健康检查：

- 打不开时，优先查相机绑定和数据库连接
- 能打开但内容不对时，优先查当前是不是正确的相机和模板对象

### 处理原始文件和缓存

校准服务还会涉及原始文件列表、文件下载和缓存清理。遇到“旧数据干扰当前校准”这类问题时，先从缓存和原始文件范围入手，而不是直接怀疑整个服务不可用。

## 常见问题

### 点了编辑校准模板，但没有进入编辑器

- 先确认是否已经绑定了对应物理相机
- 如果启用了 MySQL，再确认数据库连接是否正常
- 再确认当前打开的是正确的校准服务实例

### 相机已经换了，但校准结果看起来还是旧的

- 先确认校准服务绑定的 `CameraCode` 是否已经更新
- 再确认物理相机切换后有没有重新保存当前配置
- 必要时重新进入模板编辑器确认当前模板对象

### 校准过程中拿到的原始文件不对

- 先确认当前设备代码和设备类型是不是目标设备
- 再确认请求的是不是原始文件列表而不是旧缓存
- 如果怀疑缓存干扰，先做一次缓存清理再复查

### 校准服务存在，但流程里不生效

- 先确认绑定的相机和模板是否有效
- 再联动检查 [工作流程概览](../workflow/README.md) 和 [流程执行与调试](../workflow/execution.md)
- 同时结合 [日志查看器](../interface/log-viewer.md) 看具体失败阶段
