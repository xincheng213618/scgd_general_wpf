# 电机服务

电机服务用于把电机设备纳入 ColorVision 的统一设备体系，并提供打开、关闭、移动、回原点和读取当前位置等常见控制动作。它适合在手动调试和自动化流程里都需要位置控制的场景使用。

## 什么时候应该看这一页

- 你要新增或配置一个电机服务
- 你要手动移动电机到指定位置
- 你要确认当前位置是否已经更新
- 你要排查“电机能看到但不动”或“位置没有刷新”的问题

## 常见使用顺序

1. 先在设备列表里创建并保存电机服务。
2. 检查通信配置和超时参数是否合理。
3. 先执行一次打开动作，确认设备处于可操作状态。
4. 再做移动、回原点或读位置等动作。
5. 完成后如果业务要求关闭设备，再执行关闭动作。

## 这个服务里常见的操作

### 打开和关闭

很多后续动作都默认设备已经处于可操作状态。遇到“命令发了但没反应”时，先确认是否真的已经成功打开，而不是直接怀疑移动命令。

### 移动到指定位置

常见会关心这几个参数：

- 目标位置
- 绝对移动还是相对移动
- 超时时间

如果是调试阶段，先用较小范围移动确认方向和行为，再做大步长调整。

### 回原点

当当前位置不可信，或者需要重新建立统一起点时，优先先回原点再继续做后续动作。

### 获取当前位置

当前位置不是靠猜的，应该主动读回来确认。特别是在自动化流程或手动多次移动之后，先读位置再做下一步通常更稳。

## 常见问题

### 电机服务存在，但一直打不开

- 先检查通信配置是否正确
- 再确认设备是否在线
- 如果打开动作失败，后续移动和读位置通常也不会可靠

### 发送了移动命令，但位置没变化

- 先确认设备是否已经成功打开
- 再确认目标位置、绝对/相对模式和超时时间
- 操作后主动读一次当前位置，不要只看界面是否刷新

### 当前位置显示不对

- 先重新执行一次获取位置
- 再确认是不是上一次移动失败但界面仍保留旧值
- 如果涉及回原点，先确认回原点动作是否真正完成

### 在流程里调用电机动作失败

- 先用手动方式验证这台电机当前能否正常打开和移动
- 再检查流程里的参数是不是和手动测试一致
- 必要时联动看 [流程执行与调试](../workflow/execution.md) 和 [日志查看器](../interface/log-viewer.md)
