# 数据管理

本章节只回答三个问题：数据通常在哪里看，什么时候需要进数据库页，什么时候应该走导入导出。

## 这一章适合什么时候看

- 你想确认测试结果或业务数据存到了哪里
- 你需要查看、整理或清理数据库内容
- 你需要做批量导入或导出
- 你在排查“数据没保存”“导出失败”“看不到历史记录”这类问题

## 本章节怎么读

### 先看数据库

- [数据库操作](./database.md)

适合的场景：

- 想确认数据有没有真的落库
- 想按记录、时间或条件查结果
- 想排查连接、写入或查询问题

### 再看导入导出

- [数据导出与导入](./export-import.md)

适合的场景：

- 需要把结果交给外部系统或人工分析
- 需要迁移一批配置或数据
- 想确认当前支持的导出格式和处理顺序

## 常见使用顺序

1. 先确认当前项目、设备或流程已经正确执行完成。
2. 如果怀疑数据没保存，先进入 [数据库操作](./database.md) 查记录。
3. 只有在确认数据已经存在后，再进入 [数据导出与导入](./export-import.md) 做后续处理。
4. 如果导出内容不对，先回头核对源数据，而不是反复重试导出按钮。

## 常见问题

### 看不到想要的历史数据

- 先确认当前连接的数据库实例是不是正确的
- 再确认筛选条件、时间范围和项目对象是否选对
- 如果是刚执行完流程，先结合 [日志查看器](../interface/log-viewer.md) 看是否真的写入成功

### 数据存在，但导出结果不对

- 先确认导出的对象是不是当前正在查看的那批数据
- 再确认导出格式是否符合你的后续使用场景
- 如果字段缺失，优先回到源数据页确认，而不是直接判断导出功能异常

### 不确定问题出在流程还是数据层

- 先看 [流程执行与调试](../workflow/execution.md)
- 再用 [日志查看器](../interface/log-viewer.md) 确认失败发生在执行、写入还是导出阶段
