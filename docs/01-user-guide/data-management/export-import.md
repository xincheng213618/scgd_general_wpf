# 数据导出与导入

当前没有统一的“所有数据导入导出中心”。不同对象走不同入口：设置、流程模板、数据库记录、结果文件、项目报告各自归属对应窗口。

## 先判断对象

| 要处理什么 | 优先入口 |
| --- | --- |
| 软件设置 | 工具 -> 导入导出设置，`.cvsettings` |
| 流程模板 | 流程设计器导入/导出 |
| 数据库记录 | 数据库浏览器或业务结果页 |
| CSV/Excel | 对应业务窗口或插件导出 |
| 图片/overlay | 图像编辑器或结果窗口 |
| Socket/MES 响应 | 项目窗口、SocketProtocol 或外部联调工具 |

如果找不到入口，先确认对象属于哪一类，不要继续找“总导出菜单”。

## 主要入口

| 入口 | 当前用途 |
| --- | --- |
| 工具 -> 导入导出设置 | 导出/导入软件配置，主要用于配置迁移 |
| 流程设计器 | 导出当前流程、导入流程、导入模块 |
| 业务结果窗口 | 导出 CSV、Excel、结果图、报告或客户格式 |
| 数据库浏览器 | 查看和维护底层记录，辅助确认数据范围 |

设置导入导出实现主要位于 `UI/ColorVision.UI.Desktop/Settings/ExportAndImport/`。

## 交付前验收

| 步骤 | 通过标准 |
| --- | --- |
| 固定样例 | 用明确 SN、批次或时间跑一次最小流程 |
| 确认源数据 | 数据库或结果窗口能查到同一轮记录 |
| 执行导出 | 文件生成，路径和文件名可解释 |
| 打开核对 | 字段顺序、单位、判定、时间、SN 与客户要求一致 |
| 留样 | 保存样例文件、截图和外部响应样例 |

## 故障分流

| 现象 | 第一检查点 |
| --- | --- |
| 找不到导出按钮 | 对象是否属于设置、流程或某个业务窗口 |
| 文件生成但为空 | 源数据是否落库，批次/SN 是否选对 |
| 字段缺失或顺序不对 | 当前窗口导出的对象、项目 exporter、客户格式版本 |
| 图片/overlay 不对齐 | 原图和结果图是否同一轮，ROI/POI 坐标和缩放 |
| 外部系统收不到 | ColorVision 是否完成并生成结果，协议、端口和项目 handler |
| 迁移后行为变化 | 是否只导出了设置，流程模板和数据库是否同步迁移 |

## 不再默认承诺

以下能力只有在具体模块窗口已实现时才说明：

- 统一 Excel/JSON/XML/PDF 导出中心
- 通用列映射导入向导
- 通用批量文件夹导入向导
- 所有结果数据随 `.cvsettings` 一起迁移
