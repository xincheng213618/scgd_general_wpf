# 数据库操作

当前最明确、最稳定的数据库入口是“数据库浏览器”。它适合做表级浏览、搜索、分页查看，以及在满足条件时做新增、修改、删除。

## 入口和能力

| 项 | 说明 |
| --- | --- |
| 菜单入口 | 工具 -> 数据库浏览器 |
| 定位 | 表浏览器和基础维护窗口，不是统一 SQL 控制台 |
| 数据源 | 当前默认能注册和显示 MySQL、SQLite 日志库 |
| 浏览 | 左侧树显示数据源、库和表；右侧显示表标题、来源、列、数据网格、分页和状态 |
| 搜索和分页 | 支持关键字搜索和分页，数据没看到时先查搜索条件和页码 |
| 写操作 | 数据源和表允许写入且表有主键时，提供新增、保存、撤销、删除、刷新 |

如果表没有主键，或者当前数据源不允许写入，就不要期待所有编辑按钮都可用。

## 适用场景

- 确认某批数据有没有真的落库。
- 查看 MySQL 或 SQLite 里的实际表数据。
- 按关键字搜索记录。
- 做少量表数据维护。
- 排查“数据写入了没”“为什么列表里看不到”。

## 常见顺序

1. 从工具菜单打开数据库浏览器。
2. 在左侧展开正确的数据源、库和表。
3. 先看表名、来源和记录数，确认没进错库。
4. 数据太多时，先用搜索再翻页。
5. 需要维护时，再确认当前表是否允许写入和是否有主键。

## 注意事项

| 风险 | 说明 |
| --- | --- |
| 不是万能查询台 | 它适合看表、筛记录和少量维护；模块有专门查询窗口时优先用模块入口 |
| 没有主键不适合改 | 修改和删除依赖主键可靠定位记录 |
| 写入能力取决于数据源和表 | 按钮不可用时，先判断表不可写还是数据源只读 |

## 常见问题

| 现象 | 先查 |
| --- | --- |
| 找不到想看的数据表 | 数据源和库是否正确，刷新数据源树，MySQL 连接和数据库名是否正确 |
| 表能打开但数据为空 | 搜索条件、页码、分页大小，再看 [流程执行与调试](../workflow/execution.md) 或 [日志查看器](../interface/log-viewer.md) |
| 按钮是灰的，不能编辑 | 当前表是否允许写入，表是否存在主键 |
| 修改后结果不对 | 保存是否成功，刷新后是否一致，读路径是否正确 |
