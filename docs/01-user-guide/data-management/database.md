# 数据库操作

当前最明确、最稳定的数据库入口是“数据库浏览器”。它适合做表级浏览、搜索、分页查看，以及在满足条件时做新增、修改、删除。

## 什么时候应该看这一页

- 你想确认某批数据有没有真的落库
- 你要查看 MySQL 或 SQLite 里的实际表数据
- 你要按关键字搜索记录
- 你要做少量表数据维护
- 你在排查“数据写入了没”“为什么列表里看不到”这类问题

## 当前真实入口

数据库浏览器当前有明确菜单入口：

- 工具 → 数据库浏览器

从当前实现看，它不是 SQL 编辑器，而是一个面向表浏览和基础维护的窗口。

## 数据库浏览器里能做什么

### 浏览数据源、库和表

左侧树会先显示数据源，再进入库和表。当前默认能注册和显示的来源至少包括：

- MySQL
- SQLite 日志库

### 按表查看数据

选中某个表后，右侧会显示：

- 表标题和来源
- 列信息
- 数据网格
- 搜索框
- 分页控件
- 状态栏

### 搜索和分页

当前窗口支持按关键字搜索，并在表数据较多时分页浏览。遇到“明明有数据但没看到”时，先查搜索条件和页码，而不是直接判断没写入。

### 基础写操作

如果当前数据源和表都允许写入，并且表有主键，界面会提供这些动作：

- 新增
- 保存
- 撤销
- 删除
- 刷新

如果表没有主键，或者当前数据源不允许写入，就不要期待所有编辑按钮都可用。

## 常见使用顺序

1. 先从工具菜单打开数据库浏览器。
2. 在左侧展开正确的数据源、库和表。
3. 先看表名、来源和记录数，确认没进错库。
4. 数据太多时，先用搜索再翻页。
5. 需要维护时，再确认当前表是否允许写入。

## 使用时要特别注意什么

### 当前窗口是表浏览器，不是万能查询台

它适合看表、筛记录和做少量维护，但当前文档不再把它描述成统一 SQL 控制台。若某个模块有专门查询窗口，应优先使用那个模块自己的入口。

### 没有主键的表不适合随便改

当前实现会依赖主键决定修改和删除的可靠性。没有主键的表，更适合只读浏览或谨慎新增。

### 写入能力取决于数据源和表

就算你已经打开了数据库浏览器，也不代表所有表都能写。看到按钮不可用时，先判断是当前表不可写，还是数据源本身只读。

## 常见问题

### 找不到想看的数据表

- 先确认当前展开的是正确的数据源和库
- 再刷新数据源树
- 如果是 MySQL，确认连接配置和数据库名是否正确

### 表能打开，但数据为空

- 先确认当前是不是被搜索条件过滤了
- 再确认页码和分页大小
- 如果仍为空，回到 [流程执行与调试](../workflow/execution.md) 或 [日志查看器](../interface/log-viewer.md) 看是否真的写入成功

### 按钮是灰的，不能编辑

- 先确认当前表是否允许写入
- 再确认表是否存在主键
- 如果没有主键，不要把“不可编辑”误判为数据库异常

### 修改后结果不对

- 先确认保存是否成功
- 再刷新当前表重新查看
- 对关键业务数据，优先先查读路径是否正确，再做人工修改
