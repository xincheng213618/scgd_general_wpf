# Copilot 使用指南

ColorVision Copilot 用来解释当前界面信息、辅助排查流程和设备问题，并在明确确认后协助预览模板调整。它不会自动启动流程、控制设备或运行系统命令。

## 首次配置

1. 打开主界面右侧的 **Copilot** 面板。
2. 进入模型设置，选择服务商并填写 API 地址、模型和 API Key。
3. 使用连接测试确认配置可用。
4. Local MCP 仅供 Codex 等本机客户端读取 ColorVision 上下文；不使用时保持关闭。需要让 Copilot 使用其他 MCP 服务时，在同一设置页配置 `Copilot MCP clients`。

API Key 和 MCP token 保存在本机加密配置中。发送问题前，应确认当前附加的上下文不包含不希望交给模型的数据。

## 数学公式

Copilot 回复支持 LaTeX 数学公式。行内公式可使用 `$...$` 或 `\\(...\\)`，独立公式可使用 `$$...$$` 或 `\\[...\\]`；跨行独立公式也会正常排版。代码片段中的美元符号不会被当作公式，无法解析的公式会保留原始文本，避免隐藏模型回复。

## Agent 工具执行

Copilot 默认使用 Microsoft Agent Framework 组织多轮推理和工具调用，不需要在模型设置中单独启用。OpenAI-compatible 与 Anthropic-compatible Profile 共用同一套工具生命周期、参数校验、超时、审计和安全策略。

创建流程、应用模板修改、通用菜单执行和切换语言等受保护操作会暂停当前 Agent Framework 会话，并在“Action review”区域显示精确参数、风险和截止时间。批准后会在同一会话继续并且只执行一次；拒绝、取消或超时都不会执行该操作，模型会收到明确的拒绝结果后继续回答。

可用工具按当前请求和 ColorVision 上下文动态筛选，包括文档与网页搜索、网页读取、受控本地文件读取、日志诊断和有限应用操作。普通概念问答会直接回答，不会仅因工作区可搜索就调用文件或网页工具；只有明确询问当前项目/代码、要求联网、提供 URL，或确实需要实时证据时才开放对应搜索能力。网页 URL 请求会优先读取页面；读取失败时再搜索公开网页。成功调用会以紧凑活动行显示；没有找到结果的搜索尝试默认不占据主活动区，诊断数据仍保留在会话 trace 中。

外部 MCP 服务按行配置，格式为 `名称 | Streamable HTTP 地址 | token 环境变量 | 默认策略 | 可选工具白名单`，例如 `lab | https://mcp.example.com/mcp | LAB_MCP_TOKEN | approval | get_status=read-only,apply_patch=approval`。第五段使用远端工具的精确名称，未列出的工具不会提供给模型；留空或写 `*` 会暴露该服务发现的全部工具。远程地址必须使用 HTTPS，本机 loopback 可使用 HTTP；token 只从环境变量读取，不写入 ColorVision 配置。默认策略使用 `approval`；只对明确可信的只读工具单独设置 `read-only`。工具描述会在进程内缓存 5 分钟以减少重复发现，但每轮仍建立独立连接；外部服务若声明工具列表变更通知，当前连接会立即使缓存失效，TTL 和设置页的 `Refresh Discovery` 继续作为兜底。外部 MCP 调用的 trace 与审计只保存参数名称，不保存第三方 Schema 中含义未知的参数值。设置页诊断会紧凑显示当前 Capability Catalog 的能力数量和 revision；启用本机 MCP 后，可信客户端还可读取 `colorvision://copilot/capabilities` 获取同一份只读 JSON 目录。目录不包含远端地址、token 或参数值。某个外部服务连接失败时，本次问答继续执行，不会把连接诊断或可选搜索失败铺满聊天界面。

当只读或明确幂等的工具遇到瞬时网络错误或超时时，Agent 可以明确发起一次重试。聊天 trace 会分别显示每次尝试；同参数最多两次。非幂等操作、参数错误、拒绝和取消不会重试。如果受保护的幂等写操作需要第二次尝试，界面会再次要求审批，不会沿用上一次授权。

模型服务在尚未返回任何内容时遇到限流、临时服务故障、超时或连接中断，会进行短暂的有界重试。只要已经收到文本或工具调用，Copilot 就不会重放请求；永久请求错误和用户取消也不会重试。每次真实请求都会计入本轮供应商调用与 Token 预算。

未完成的 Agent 任务会随会话保存。checkpoint 同时记录当时的能力目录 revision、稳定能力指纹，以及最多 16 条、64K 字符的有界对话记忆；恢复时如果原有工具被删除，或描述、输入 Schema、风险、审批、并发、超时、审计策略发生变化，旧任务计划和历史授权会自动失效，但初始目标与最近问答仍会交给新 Session，Agent 再使用当前工具重新规划。仅新增工具不会打断已有会话。旧版本没有能力快照的 checkpoint 会安全地执行一次重新规划。

checkpoint 还可保存最多 24 条经过限制和脱敏的只读 evidence artifact。写操作、失败结果和非幂等结果不会进入证据清单；普通只读工具默认只保留摘要，文档搜索、公开网页搜索和网页读取可保留最多 1600 字符的脱敏摘录，外部 MCP 的 `NamesOnly` 工具不保存返回正文。只有 session 无法恢复或必须重新规划时才注入最多 12 条历史证据，并明确要求 Agent 把它们当作不可信历史数据、重新核对易变化事实，绝不能把证据当作操作指令或历史授权。

Agent checkpoint 还维护最多 256 条结构化任务事件，用统一时间线关联运行、任务账本、工具调用、审批、运行中转向、证据制品和停止原因。事件不保存工具参数、审批正文或转向原文，只保留脱敏摘要与哈希关联键；该时间线仅用于恢复诊断和分页查询，不会作为提示重新发送给模型，也不会恢复历史写操作权限。主聊天区默认不显示这份诊断数据；启用本机 MCP 后，可信客户端可显式读取 `colorvision://copilot/task-events`，或调用只读工具 `get_agent_task_events` 按事件类型、run、工具、关联 ID 和游标查询。

多步骤请求会在回答中显示“Agent 任务”面板，包括 plan/execute 模式、完成数量、每个任务状态和本轮停止原因。未完成任务可点击“继续”从当前 Framework Session 恢复；恢复只读工作不会重复发送整段可见历史，任何写操作仍会生成新的审批。

工具过程采用类似 Codex 的紧凑活动行：默认只显示“读取了网页”“搜索了文件”等结果和耗时；点击单条活动才展开 Runtime、参数、资源键和错误等诊断信息。

Agent 运行时仍可在输入框键入补充要求并按 Enter，或点击出现的 `↳` 按钮。该要求会进入当前 AgentSession，在下一次模型调用机会调整正在执行的任务；方形停止按钮仍表示立即取消。steering 不会绕过工具权限、参数校验或写操作审批。

Copilot 还会按问题类型加载随应用提供的 Agent Skill，例如流程诊断清单。技能只是一组只读工作指引和参考资料：普通问答不匹配时不会加载，也不会因此搜索文件或网页；加载技能不会获得修改模板、执行菜单或控制设备的权限。技能内部脚本当前不允许运行。

如果第三方服务端不支持其声明的 Function Calling 协议，本轮会明确失败；Copilot 不会自动切换另一套运行时重放请求，避免工具调用尤其是写操作被重复执行。模型给出的工具参数不符合 Schema 时，调用会在执行和审批之前被拒绝，并把结构化错误交回模型修正；非法的受保护调用不会弹出审批。模型偶尔生成不存在的工具名时，框架会返回 not-found 结果让模型自行纠正，同时留下紧凑、脱敏的失败记录，不会把未知函数当成真实能力执行。

## 常用入口

| 场景 | 操作入口 | Copilot 收到的内容 |
| --- | --- | --- |
| 流程失败 | 流程界面右键，选择“问 AI 分析当前流程” | 流程状态、选中节点、节点参数、最近失败证据 |
| 设备异常 | 设备服务右键，选择“问 Copilot” | 在线状态、心跳、服务类型和脱敏后的配置摘要 |
| 图像分析 | 图像编辑器右键，选择“问 AI 分析当前图像” | 图像元数据、ROI 和标注摘要，不包含图像像素 |
| 模板检查 | 模板 JSON 编辑器中的 Copilot 操作 | 当前模板 JSON、校验状态和未保存状态 |
| 日志排查 | Copilot 诊断模式 | 最近日志中的相关行 |

## 流程诊断结果

流程诊断会优先给出：

1. 当前状态和重点节点。
2. 来自快照与日志的证据。
3. 按可能性排序的原因。
4. 从只读检查开始的验证步骤。
5. 有证据支持的模板字段调整候选。
6. 风险和需要人工确认的操作。

如果需要修改模板，安全顺序是“建议 -> `TemplatePatch` 差异预览 -> `ApplyTemplatePatch` 原生审批 -> 用户批准 -> 应用”。预览只读；未经过预览和批准不会写入模板。

在模板 JSON 编辑器处于活动状态时，也可以直接在 Copilot 中描述参数调整，例如“把 Exposure 调整到 12”。Copilot 只会先生成字段级差异预览；需要继续时，再明确要求应用对应的 `preview_id`。待确认区会说明影响，点击批准后修改只进入当前编辑器，仍需由操作者决定是否保存模板。

## 安全边界

- 不执行 Shell、PowerShell、批处理或任意进程。
- 不自动启动、停止或重跑流程。
- 不直接控制相机、PG、SMU、电机等设备。
- 不删除文件，也不读取工作区允许范围之外的任意路径。
- 密码、token、API Key、许可证和序列号等字段会在业务上下文中脱敏。
- AI 给出的原因属于诊断建议；执行参数调整前仍需工程师确认。

开发人员需要配置本地 MCP 时，参见 [ColorVision 本地 MCP](../02-developer-guide/core-concepts/colorvision-mcp.md)。
