# 使用手册操作工作流矩阵

这页面向操作者、测试工程师和现场交付人员，用“我要完成什么操作”来定位文档入口。它不讲源码实现，也不替代具体设备页；它把日常工作中最常见的操作、完成标准和失败时第一检查点放在同一张表里。

## 什么时候先看本页

| 场景 | 本页能帮你做什么 |
| --- | --- |
| 第一次上机 | 确认安装、启动、主窗口、设备、流程、数据的阅读顺序 |
| 现场交付 | 按项目、设备、流程、导出、外部系统逐项验收 |
| 产线操作 | 找到日常运行、切换项目、查看结果、导出数据的入口 |
| 异常排查 | 先定位是界面、设备、流程、数据还是外部系统问题 |

如果已经确定是具体设备、流程节点或项目包源码问题，再进入对应设备页、工作流程页或模块参考。

## 按角色看日常流程

| 角色 | 日常动作 | 最常用文档 |
| --- | --- | --- |
| 操作员 | 打开主程序、选择项目/流程、输入 SN、运行、查看 PASS/FAIL、导出结果 | 本页、主窗口导览、项目说明、数据导出 |
| 测试工程师 | 配设备、调相机、调流程、确认结果字段、对比历史数据 | 设备服务、流程设计/执行、图像编辑器、数据管理 |
| 现场交付人员 | 安装部署、插件/项目包验收、Socket/MES 联调、培训操作员 | 安装指南、本页、项目说明、现有插件能力、常见问题 |
| 维护开发人员 | 判断问题属于 UI、Engine、插件还是项目包，再进入模块参考 | 本页、UI 组件使用手册、项目结构总览、Engine 矩阵、UI 组件目录 |

## 典型上机顺序

首次交付或复测按“安装/首次运行 -> 主窗口和菜单 -> 设备服务 -> 流程 -> 结果 -> 导出/上传 -> 日志排查”走。日常生产通常从“确认设备服务”和“执行流程/项目”开始。

## 数据交付检查

| 交付类型 | 使用者关心什么 | 先查哪里 |
| --- | --- | --- |
| CSV / Excel | 文件是否生成、字段顺序、单位、PASS/FAIL | 数据导出页、项目包 exporter、输出路径 |
| SQLite / MySQL | 能否按 SN、时间、批次查询 | 数据库操作、连接配置、批次号 |
| Socket / MES / Modbus | 外部系统是否收到正确状态码和数据 | 项目包总览、SocketProtocol、现场协议 |
| 图像 / overlay | 图像是否能打开，点位/框线是否对齐 | 图像编辑器、文件服务器、结果展示链路 |
| Summary / 文本 | 产量、良率、失败项是否准确 | 项目包配置、导出目录、字段映射 |

## 排查分流

| 现象 | 先判断 | 下一步 |
| --- | --- | --- |
| 菜单或窗口没有出现 | 插件/项目包是否加载 | 看现有插件能力、项目包总览、日志 |
| 设备不在线 | 设备服务还是物理硬件问题 | 看设备页、驱动、端口/IP、服务日志 |
| 流程启动但不结束 | 流程节点或设备命令卡住 | 看流程执行页和第一个未完成节点 |
| 结果有但导出为空 | 项目结果映射问题 | 看项目包 Process/Recipe/Fix/exporter |
| 图像能打开但没有 overlay | 显示结果处理问题 | 看图像编辑器和 Engine 结果展示链路 |
| 外部系统收不到结果 | 协议、端口或项目窗口状态问题 | 看项目包总览、SocketProtocol、日志 |
