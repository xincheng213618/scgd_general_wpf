# 常见问题

本页不是代码分析，而是一个排查入口。遇到问题时，先确定它属于哪一类，再跳到对应页面继续缩小范围。

## 先按这个顺序排查

1. 先看 [日志查看器](../interface/log-viewer.md)，确认错误发生在启动、设备、流程还是插件阶段。
2. 再确认当前项目、设备实例、模板或窗口是不是你以为的那个对象。
3. 再检查路径、许可证、数据库连接、服务状态这些外部条件。
4. 最后再考虑是否需要重开窗口、重新保存配置或重启相关服务。

## 常见场景

### 主窗口里找不到功能

- 先回到 [主窗口导览](../interface/main-window.md)
- 优先使用搜索框定位入口
- 如果是插件功能，确认插件是否已经正确加载

### 设备存在，但不能工作

- 先看 [设备服务概览](../devices/overview.md)
- 相机场景继续看 [相机服务](../devices/camera.md)
- 若是绑定或硬件接入问题，再看 [物理相机管理](../devices/camera-management.md)

### 参数改了，但效果没变化

- 先确认改的是当前正在使用的对象
- 再确认是否已经保存，并且相关窗口或服务已经重新加载
- 属性和模板相关场景，优先回到 [属性编辑器](../interface/property-editor.md) 或对应设备参数页复查

### 流程执行失败

- 先看 [流程执行与调试](../workflow/execution.md)
- 日志里优先搜节点名、模板名和失败关键字
- 不要一开始就怀疑整条流程，先缩到第一个失败节点

### JSON 或模板内容校验失败

- 先用编辑器里的校验按钮确认是不是格式问题
- 再看是否把注释文本、说明文本或旧模板内容误贴进了 JSON
- 如果切换了不同编辑模式，先回到当前实际保存的 JSON 再判断

### 日志太多，看不出重点

- 先把日志级别切到 `Warn` 或 `Error`
- 再用关键词缩小范围
- 如果搜索框变红，先放弃正则，退回普通关键词

## 如果还是没定位到

- 记录出问题的时间点
- 记下当前项目、设备名、模板名或插件名
- 附带对应日志片段再继续排查，效率会明显高很多

## 说明

- 本页只保留排查路径，不再继续维护泛化的项目结构分析稿。
