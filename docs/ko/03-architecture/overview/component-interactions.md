# 구성요소 상호작용

이 페이지는 전체 시스템을 포괄하지만 왜곡되기 쉬운 "모듈 상호 작용 매트릭스"를 더 이상 유지하지 않습니다. 현재 더 가치 있는 접근 방식은 코드에 실제로 존재하고 변경 방향에 가장 자주 영향을 미치는 몇 가지 대화형 메인 체인만 설명하는 것입니다.

## 먼저 메인체인을 살펴보겠습니다

### 1. 메인 프로그램 엔트리 체인

기본 프로그램 항목은 `ColorVision/`에 있습니다.

먼저 다음을 담당합니다.

- 기본 DLL 사전 로딩
- 구성, 로그, 테마, 언어 초기화
- 파일 처리 명령줄 분기
- 플러그인 로드 여부 결정
- 마법사 실행 또는 실행 창

이 레이어는 특정 비즈니스 자체가 아니라 "호스트" 및 "입력 스케줄러"에 가깝습니다.

### 2. UI 워크벤치 체인

기본 작업 공간에 들어간 후 `UI/` 아래의 창, 메뉴, 패널 및 편집기는 사용자 작업을 수락한 다음 해당 작업을 엔진 레이어로 전송하는 역할을 합니다.

여기서 가장 일반적인 상호 작용은 다음과 같습니다.

- 메인 창 및 데스크탑 메뉴에 대한 액세스 제공
- 속성 편집기, 이미지 편집기, 데이터베이스 브라우저 및 기타 모듈은 각각의 대화형 보기를 담당합니다.
- 실제 비즈니스 액션은 'Engine/'으로 계속 전송됩니다.

### 3. 서비스 트리 및 디바이스 체인

'ServiceManager'는 런타임에서 가장 중요한 조정 개체 중 하나입니다. 그것은:

- 데이터베이스에서 리소스 및 서비스 정의 읽기
- 단말 서비스 및 장치 서비스 객체 생성
- 그룹 리소스 구성
- 프로세스 표시 영역 및 장치 표시 제어를 디스플레이 관리자로 통합

따라서 "디바이스가 트리에 나타나는 이유", "디스플레이 영역이 열릴 수 있는 이유", "특정 디바이스가 런타임에 진입하지 않는 이유"에 대해 우리는 일반적으로 단일 디바이스 클래스를 직접 살펴보기보다는 `ServiceManager`를 먼저 살펴봅니다.

### 4. 템플릿 등록 및 편집 체인

템플릿 시스템의 주요 상호 작용은 독립형 테이블이 아니지만 다음과 같습니다.

- 관련 어셈블리가 메인 프로그램이나 플러그인에 의해 로드됩니다.
- `TemplateControl`은 `IITemplateLoad`를 구현하는 유형을 검색합니다.
- 이러한 유형은 `Load()`의 전역 템플릿 사전에 템플릿을 등록합니다.
- 이러한 템플릿을 사용하기 위한 템플릿 편집기, 프로세스 창 및 특정 비즈니스 기능

따라서 "템플릿이 표시되지 않는 이유"와 같은 문제는 일반적으로 편집 창 자체뿐만 아니라 로딩 체인, 데이터베이스 상태 또는 어셈블리 로딩 체인의 문제입니다.

### 5. 프로세스 실행 체인

프로세스 실행 중 메인 체인은 대략 다음과 같습니다.

1. 'DisplayFlow'는 현재 프로세스 템플릿을 선택하고 새로 고칩니다.
2. `FlowControl`은 현재 프로세스를 시작하거나 중지합니다.
3. `FlowEngineLib`은 노드 그래프를 실행합니다.
4. `MQTTRCService`는 서비스 토큰과 상태 업데이트를 제공합니다.
5. 장치 노드 또는 알고리즘 노드는 결과를 반환합니다.
6. 실행 상태, 배치 기록, 로그 텍스트 및 노드 메시지가 인터페이스에 다시 업데이트됩니다.

이 링크는 일반적인 사실을 보여줍니다. 프로세스는 격리된 모듈이 아니며 자연스럽게 템플릿, 장치 서비스 및 레지스트리 상태에 의존합니다.

### 6. 플러그인 확장 체인

플러그인이 로드된 후에는 하나 이상의 DLL 대신 새 어셈블리가 전체 런타임으로 가져옵니다.

- 새 메뉴 또는 창 항목
- 새로운 서비스 또는 공급자 구현
- 새로운 템플릿, 결과 보기 또는 확장 지점 구현

따라서 플러그인과 기본 프로그램의 상호 작용은 외부에서 병렬로 실행되기보다는 "기존 런타임에 기능 통합"에 더 가깝습니다.

## 코드 변경 시 일반적으로 이 페이지를 사용하는 방법

### 메인창이나 사용자 출입구 변경

다음의 우선순위를 정하세요:

- 메인 프로그램 입장
- UI 메뉴/패널
- 해당 엔진 서비스

### 장치 동작 변경

다음의 우선순위를 정하세요:

-`서비스 관리자`
- 특정 `DeviceService`
- MQTT 또는 런타임 상태 객체에 해당

### 템플릿 또는 프로세스 변경

다음의 우선순위를 정하세요:

- `템플릿 컨트롤`
- 템플릿 편집기
- `DisplayFlow` / `FlowControl` / `FlowEngineLib`

### 플러그인 확장자 변경

다음의 우선순위를 정하세요:

- `플러그인로더`
- 플러그인 세트
- 플러그인으로 구현된 메뉴, 서비스, 템플릿 또는 보기 인터페이스

## 이 페이지에서는 더 이상 아무것도 수행하지 않습니다.

이 페이지에서는 더 이상 다음과 같은 고위험 콘텐츠를 유지하지 않습니다.

- 현재 구현과 동떨어진 것처럼 보이는 완전한 모듈 매트릭스 테이블
- 확인되지 않은 이벤트 이름 및 인터페이스의 대규모 목록
- 실제 코드 디렉터리와 분리된 이론적 종속성 그래프

주제에 대한 심층적인 설계 분석이 필요한 경우 여기에 지나치게 예상되는 요약을 퍼뜨리는 대신 해당 주제 페이지로 반환되어야 합니다.
