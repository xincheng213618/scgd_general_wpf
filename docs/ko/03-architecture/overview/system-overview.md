# 시스템 아키텍처 개요

이 페이지에서는 전체 창고를 표준 교과서로 설명하기 위해 더 이상 추상 레이어 세트를 사용하려고 시도하지 않습니다. 대신, 현재 코드 웨어하우스의 메인 디렉터리에 따라 시스템이 어떻게 구성되어 있는지, 코드를 읽을 때 어디서부터 시작해야 하는지 직접적으로 설명합니다.

## 먼저 이 창고를 어떻게 이해해야 할까요?

현재 디렉토리 구조로 볼 때 ColorVision은 데스크톱 메인 프로그램을 핵심으로 삼고 엔진, UI, 플러그인, 프로젝트 패키지, 설치 및 업데이트 시스템을 중심으로 돌아가는 Windows WPF 플랫폼에 더 가깝습니다.

가장 중요한 최상위 영역은 다음과 같습니다.

- `ColorVision/`: 메인 애플리케이션 입구 및 메인 창
- `UI/`: WPF UI 프레임워크, 테마, 속성 편집기, 이미지 편집기, 데이터베이스 및 데스크탑 메뉴 등
- `Engine/`: 장치 서비스, 템플릿 시스템, 프로세스 실행, OpenCV 통합, 파일 처리
- `Plugins/`: 런타임 플러그인 확장
- `프로젝트/`: 클라이언트 프로젝트 및 맞춤형 비즈니스 포트폴리오
- `ColorVisionSetup/`: 관련 프로그램 설치 및 업데이트
- `Backend/marketplace/`: 플러그인 마켓 백엔드
- `Scripts/`: 스크립트 빌드, 패키지, 게시

## 시스템 역할별 구조 보기

### 메인 프로그램 레이어

'ColorVision/'은 메인 창, 애플리케이션 시작, 글로벌 구성, 업데이트 포털 및 전반적인 워크벤치 구성을 담당하는 데스크탑 애플리케이션 포털입니다.

"프로그램 시작 후 가장 먼저 무슨 일이 일어나는지"를 쫓고 있다면 일반적으로 여기에서 시작한 다음 `UI/`와 `Engine/`을 함께 살펴봅니다.

### UI 레이어

`UI/`는 단일 프로젝트가 아니라 인터페이스 관련 모듈의 집합입니다. 현재 주요 내용은 다음과 같습니다.

- `ColorVision.UI/`: 메뉴, 패널, 속성 편집기 등과 같은 범용 UI 프레임워크 및 기능
- `ColorVision.Themes/`: 테마 및 시각적 리소스
- `ColorVision.ImageEditor/`: 이미지 보기, 주석 및 결과 표시
- `ColorVision.Database/`: 데이터베이스 브라우저 등 데이터베이스 관련 UI 기능
- `ColorVision.UI.Desktop/`: 데스크탑 수준 메뉴 및 설정 항목

### 엔진 레이어

`Engine/`은 시스템의 비즈니스 핵심이지만 단일 프로젝트 네임스페이스는 아닙니다. 현재는 주로 여러 부분으로 구성됩니다.

- `ColorVision.Engine/`: 장치 서비스, 템플릿 시스템, 프로세스 창, MQTT 및 비즈니스 조정
- `FlowEngineLib/`: 프로세스 노드 편집 및 실행 기반
- `cvColorVision/`: 낮은 수준의 시각적 처리 및 OpenCV 관련 통합
- `ColorVision.FileIO/`: 파일 읽기 및 쓰기 처리
- `ColorVision.ShellExtension/`: 외부 통합 관련 확장

### 플러그인 및 프로젝트 레이어

- `Plugins/`는 Conscope, Spectrum, SystemMonitor 등과 같은 런타임 플러그인 확장을 제공합니다.
- 'Projects/'는 일반적으로 기존 엔진 및 UI 기능을 특정 솔루션으로 재결합하여 고객 프로젝트 또는 비즈니스 패키지 구현을 저장하는 데 사용됩니다.

### 전달 및 주변 레이어

- `ColorVisionSetup/`은 부가 프로그램 설치 및 업데이트를 담당합니다.
- `Backend/marketplace/`는 플러그인 마켓의 백엔드를 담당합니다.
- `Scripts/` 및 루트 배치 스크립트는 빌드, 패키징 및 게시를 담당합니다.

## 런타임 시 가장 일반적인 기본 링크

사용자 작업을 자세히 살펴보면 일반적으로 가장 일반적인 링크는 다음과 같습니다.

1. 사용자는 `ColorVision/`의 메인 창에서 기능을 입력합니다.
2. `UI/`의 해당 창이나 패널은 표시 및 상호 작용을 담당합니다.
3. `Engine/ColorVision.Engine/`의 장치 서비스, 템플릿 또는 프로세스 로직이 비즈니스 처리를 대신합니다.
4. 프로세스 실행이 필요한 경우 `Engine/FlowEngineLib/`를 추가로 호출합니다.
5. 이미지 또는 알고리즘 처리가 필요한 경우 `Engine/cvColorVision/`, `UI/ColorVision.ImageEditor/` 또는 특정 템플릿 구현을 계속 연결합니다.
6. 함수가 외부 확장에서 제공되는 경우 `Plugins/` 또는 `Projects/`에 구현을 입력합니다.

## 코드를 읽을 때 일반적인 진입점

### 메인 인터페이스와 입구에 대해 이해하고 싶습니다.

먼저 살펴보세요:

- `컬러비전/`
-`UI/ColorVision.UI/`
- `UI/ColorVision.UI.Desktop/`

### 장비, 템플릿 및 프로세스를 이해하고 싶습니다.

먼저 살펴보세요:

-`엔진/ColorVision.Engine/Services/`
- `엔진/ColorVision.Engine/템플릿/`
- `엔진/FlowEngineLib/`

### 이미지 결과 및 표시를 이해하고 싶습니다.

먼저 살펴보세요:

- `UI/ColorVision.ImageEditor/`
-`엔진/cvColorVision/`

### 확장 기능을 이해하고 싶습니다.

먼저 살펴보세요:

- `플러그인/`
-`프로젝트/`
- [플러그인 개발 개요](../../02-developer-guide/plugin-development/overview.md)

## 이 페이지에서는 더 이상 아무것도 수행하지 않습니다.

이 페이지는 다음과 같이 쉽게 왜곡되는 콘텐츠를 더 이상 유지하지 않습니다.

- 허구의 표준화된 6계층 아키텍처 명명
- 현재 디렉토리와 일치하지 않는 모듈 이름 목록
- "종속성 주입 컨테이너", "객체 풀", "보고서 템플릿" 등의 일반화된 교과서 스타일 요약

주제에 더 자세한 런타임 관계, 프로세스 실행 체인 또는 템플릿 구조가 필요한 경우 여기에서 모든 내용을 한꺼번에 다루는 대신 해당 주제 페이지로 이동하여 설명을 확인해야 합니다.
