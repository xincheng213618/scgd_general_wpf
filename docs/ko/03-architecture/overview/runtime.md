# 아키텍처 런타임

이 페이지에서는 현재 코드에서 볼 수 있는 기본 프로그램 런타임 링크만 설명하며 더 이상 영어 초안 메타데이터 및 일반 시작 시퀀스 다이어그램을 유지하지 않습니다.

## 먼저 런타임을 이해하는 방법

데스크톱 프로그램의 현재 런타임은 "모든 모듈을 한 번에 초기화한 다음 기본 인터페이스를 표시하는" 단일 모델이 아니라 여러 실제 분기로 나뉩니다.

- 명령줄은 파일 처리만 수행하고 직접 반환합니다.
- 일반적인 데스크탑 시작 및 마법사 또는 시작 창 입력
- 마지막 비정상 종료 후, 시작을 계속하기 전에 플러그인을 비활성화할지 여부를 묻습니다.

## 메인 프로그램이 링크를 시작합니다

현재 `ColorVision/App.xaml.cs` 구현을 보면 일반적인 시작 순서는 대략 다음과 같습니다.

1. 작업 디렉터리를 설정하고 필요에 따라 중요한 DLL을 미리 로드합니다.
2. 구성, 로그, 테마 및 언어를 초기화합니다.
3. 명령줄 매개변수를 구문 분석합니다.
4. `input`, `export` 등 파일 처리 브랜치인 경우 직접 처리하여 반환합니다.
5. 단일 인스턴스 검사를 수행하고 필요한 경우 명령줄 매개변수를 실행 중인 인스턴스에 전달합니다.
6. 디버깅하지 않는 상태의 좀비 프로세스를 정리합니다.
7. 마지막 시작 상태를 기반으로 플러그인을 비활성화할지 여부를 결정합니다.
8. WinForms 시각적 스타일을 초기화합니다.
9. 마법사 완료 상태에 따라 'WizardWindow' 또는 'StartWindow'를 표시합니다.

여기서 가장 중요한 것은 모든 단계를 기억하는 것이 아니라 실행이 항상 기본 창으로 직접 연결되는 것은 아니라는 점을 아는 것입니다.

## 플러그인은 언제 들어가나요?

현재 기본 프로그램은 마법사나 시작 창에 들어가기 전에 플러그인을 로드할지 여부를 결정합니다.

플러그인 로딩의 몇 가지 핵심 사항은 다음과 같습니다.

- `Plugins/` 디렉토리를 검색하세요.
- 각 플러그인 디렉터리에서 `manifest.json`을 읽습니다.
- 선택적으로 `.deps.json` 읽기
- `ColorVision.*` 종속성 버전을 확인하세요.
- 마지막으로 'Assembly.LoadFrom(...)'을 사용하여 플러그인 어셈블리를 로드합니다.

지난번에 비정상적으로 종료된 경우 먼저 시작 시 플러그인을 비활성화할지 여부를 묻습니다. 이는 플러그인이 이 실행에 참여하는지 여부가 명확한 분기이며 항상 무조건 로드되지는 않음을 의미합니다.

## 기본 작업 공간에 들어간 후 가장 중요한 런타임 개체는 무엇입니까?

### 서비스 트리

'ServiceManager'는 데이터베이스 연결이 가능해지면 서비스 트리를 로드하고 리소스 테이블의 데이터를 다음과 같이 구성합니다.

-`유형 서비스`
-`터미널 서비스`
-`장치 서비스`
-`그룹 리소스`

그런 다음 프로세스 표시 영역과 각 장치 표시 제어를 통합 디스플레이 관리자에 설치합니다.

### 등록 센터 및 서비스 토큰

`MQTTRCService`는 다음을 담당합니다.

- 등록센터와의 연결상태 유지
- 현재 사용 가능한 서비스 토큰 쿼리
- 서비스 상태 업데이트
- 서비스 상태를 장치 서비스 개체에 다시 동기화

따라서 "프로세스를 실행할 수 없습니다", "장치가 온라인이지만 상태가 업데이트되지 않습니다"와 같은 많은 문제는 본질적으로 UI 문제가 아니라 여기의 런타임 상태가 준비되지 않은 것입니다.

### 템플릿 등록

템플릿 시스템은 수동으로 하드 코딩된 등록이 아닙니다. 현재 `TemplateControl`은 데이터베이스 연결이 사용 가능해진 후 로드된 어셈블리에서 `IITemplateLoad` 구현을 검색하고 해당 `Load()`를 호출하여 템플릿 등록을 완료합니다.

즉, 템플릿이 표시되고 편집될 수 있는지 여부는 다음 두 가지 전제에 따라 달라집니다.

- 관련 어셈블리가 로드되었습니다.
- 데이터베이스 연결이 설정되었습니다.

## 런타임 중 프로세스 실행을 연결하는 방법

사용자가 프로세스 창에 들어가면 런타임 메인 체인은 계속해서 다음으로 확장됩니다.

- 'DisplayFlow'는 현재 프로세스 템플릿 새로 고침, 프로세스 시작 또는 중지를 담당합니다.
- `FlowControl`은 실행 중인 프로세스의 시작과 중지를 담당합니다.
- `FlowEngineLib`은 특정 노드 실행을 담당합니다.
- 'MQTTRCService'는 프로세스 실행에 필요한 서비스 토큰 및 상태 업데이트를 제공합니다.

실행 중에 런타임은 계속 업데이트됩니다.

- 현재 실행 중인 노드
- 실행 로그 텍스트
- 일괄 진행
- 노드 기록 및 메시지 기록

## 런타임 중 가장 일반적인 오류 지점

### 시작 단계

- 플러그인 종속성이 만족되지 않습니다.
- 마지막 비정상적인 종료로 인해 플러그인 비활성화 여부를 수동으로 결정해야 했습니다.
- 프로그램이 계속 시작되지 않는 것으로 잘못 생각하여 명령줄 분기가 일찍 반환됩니다.

### 서비스 준비 단계

- 데이터베이스가 연결되지 않아 서비스 트리나 템플릿이 제대로 로드되지 않습니다.
- 등록 센터 또는 MQTT 측이 준비되지 않아 서비스 토큰이 비어 있습니다.

### 실행 단계

- 프로세스 템플릿이 선택되지 않았거나 시작 노드가 누락되었습니다.
- 디바이스 서비스가 존재하지만 런타임 상태가 실행 가능 상태로 동기화되지 않았습니다.
