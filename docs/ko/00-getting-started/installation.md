# 설치 가이드

이 페이지에는 사용자에게 실제로 필요한 설치 단계와 설치 후 확인 방법만 포함되어 있습니다. 설치 프로그램 내부, 패키징 스크립트 및 고급 설치 프로그램 구성이 개발 및 배포 문서로 이동되었습니다.

## 설치 전 확인

시작하기 전에 [시스템 요구사항](./prerequisites.md)을 읽고 다음 사항을 확인하세요.

- 현재 환경은 Windows 10 1903+ 또는 Windows 11입니다.
- x64 환경을 사용하세요
- 설치 디렉터리에 대한 쓰기 권한이 있어야 합니다. 서비스 구성이 관련된 경우 관리자 권한을 사용하는 것이 좋습니다
- 온라인 업데이트나 원격 서비스 접속이 필요한 경우 네트워크 환경을 이용할 수 있습니다.

## 방법 1: 설치 패키지를 통해 설치

이는 최종 사용자에게 권장되는 접근 방식입니다.

### 1단계: 설치 패키지 받기

- 팀 배포 채널, 릴리스 페이지 또는 내부 제공 패키지에서 ColorVision 설치 프로그램을 가져옵니다.
- 기본 설치 패키지와 증분 업데이트 패키지가 모두 제공되는 경우, 최초 설치 시 전체 설치 패키지를 우선적으로 사용하시기 바랍니다.

### 2단계: 설치 프로그램 시작

- 설치 프로그램을 두 번 클릭하여 설치 마법사를 시작합니다.
- 시스템에 권한 확인 팝업이 표시되면 필요에 따라 설치를 계속할 수 있습니다.
- 설치 프로그램에서 필수 구성 요소가 누락되었다는 메시지를 표시하는 경우 메시지에 따라 먼저 설치를 완료하십시오.

### 3단계: 설치 위치 및 구성요소 선택

- 설치 마법사를 따라 대상 디렉토리를 선택하십시오.
- 필요에 따라 바탕화면 바로가기, 시작 메뉴 항목 등을 생성할지 여부를 결정합니다.
- 제공 패키지에 플러그인이나 프로젝트 리소스가 포함된 경우 기본 설치 옵션을 유지하는 것이 더 안전한 경우가 많습니다.

### 4단계: 설치 완료

- 파일 복사 및 초기화가 완료될 때까지 기다립니다.
- 완료 후 메인 프로그램 시작을 직접 확인할 수 있습니다.
- 환경에 이전 버전이 있는 경우 설치 프로그램은 계속하기 전에 이전 버전을 업그레이드하거나 덮어쓰거나 제거하라는 메시지를 표시할 수 있습니다.

## 방법 2: 소스 코드에서 빌드 및 실행

이는 개발자 및 디버깅 시나리오를 위한 경로입니다.

### 최소 단계

``파워셸
닷넷 복원
dotnet build .\ColorVision\ColorVision.csproj -p:Platform=x64
dotnet run --project .\ColorVision\ColorVision.csproj -p:Platform=x64
```
### 설명

- 현재 웨어하우스는 주로 Windows WPF 및 x64용입니다. 메인 프로그램을 크로스 플랫폼 프로젝트로 취급하는 것은 권장되지 않습니다.
- 단순히 빌드 성공 여부만 확인하고 싶다면 `dotnet build`를 먼저 실행한 후 running 및 debug로 진입하면 됩니다.
- 출력 디렉터리가 실행 중인 프로그램에 의해 점유된 경우 먼저 이전 프로세스를 닫거나 임시 출력 디렉터리를 사용하여 빌드를 확인합니다.

## 설치 완료 후 최소한의 검증

설치 패키지를 설치하든 소스 코드를 실행하든 최소한 다음 사항을 확인하는 것이 좋습니다.

1. 메인 프로그램이 정상적으로 시작되어 메인 창을 표시할 수 있습니다.
2. 로그 디렉터리나 로그 창에 명백한 시작 수준 오류가 없습니다.
3. 플러그인 스캔이 완료되면 도움말이나 플러그인 관련 메뉴가 정상적으로 열릴 수 있습니다.
4. 사진을 열어 기본 인터페이스의 기본 상호 작용이 정상인지 확인할 수 있습니다.

이 단계가 정상이라면 설치 링크가 기본적으로 제공된다는 의미이며, 계속해서 [첫 실행 가이드](./first-steps.md)와 [빠른 시작](./quick-start.md)을 읽어보실 수 있습니다.

## 일반적인 설치 문제

### 설치 프로그램을 시작할 수 없습니다.

- 먼저 설치 패키지가 손상되지 않았는지 확인하고, 필요한 경우 설치 패키지를 다시 가져옵니다.
- 현재 시스템 버전이 [시스템 요구사항](./prerequisites.md)을 충족하는지 확인하세요.
- 시스템 정책에 의해 차단된 경우, 충분한 권한이 있는 계정을 사용하여 다시 실행해 보세요.

### 설치가 완료되었으나 프로그램 시작에 실패했습니다.

- 로그나 오류 메시지를 먼저 확인하세요.
- 설치 디렉터리와 사용자 문서 디렉터리에 읽기 및 쓰기 권한이 있는지 확인하세요.
- 소스 코드에서 실행하는 경우 먼저 출력 디렉터리에 런타임 종속 파일이 누락되었는지 확인하세요.

### 업그레이드 후 비정상적인 동작

- 잔여 구성이나 이전 버전이 차지하는 파일이 있는지 우선적으로 확인합니다.
- 필요한 경우 기본 프로그램 및 관련 백그라운드 서비스를 종료한 후 다시 설치하거나 다시 시작하세요.

## 더 깊이 들어가고 싶을 때 가는 곳

- 최초 실행 후 기본 동작을 확인하려면 [첫 실행 가이드](./first-steps.md)로 이동하세요.
- 최소한의 폐쇄 루프 경험을 빠르게 완료하려면 [빠른 시작](./quick-start.md)으로 이동하세요.
- 설치 프로그램, 업데이트 프로그램 및 전달 체인의 구현을 보려면 [배포 개요](../02-developer-guide/deployment/overview.md)로 이동하세요.
