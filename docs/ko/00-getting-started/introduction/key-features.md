# 주요특징(구입구)

이 페이지는 이전 링크와의 호환성을 위해 유지됩니다. 현재 문서는 더 이상 소개 장에서 이 긴 "기능 백과사전" 목록을 유지하지 않습니다. 왜냐하면 기록 기능, 프로젝트 사용자 정의 기능 및 현재 웨어하우스 검증 가능 기능을 쉽게 혼합할 수 있기 때문입니다.

## 제품을 빠르게 이해하고 싶다면 어디서부터 살펴봐야 할까요?

- [ColorVision이란 무엇입니까?](../what-is-colorvision.md)
- [시작하기](../README.md)
- [빠른 시작](../quick-start.md)

## 주제별로 계속 읽고 싶다

- 인터페이스 및 일일 작업 확인: [사용자 가이드](../../01-user-guide/README.md)로 이동합니다.
- 디바이스 및 컬렉션 링크를 확인하시려면 [디바이스 서비스 개요](../../01-user-guide/devices/overview.md)로 이동하세요.
- 프로세스 설계 및 실행을 확인하려면: [워크플로 개요](../../01-user-guide/workflow/README.md)로 이동하세요.
- 시스템 디자인 경계를 보고 싶다면: [아키텍처 디자인](../../03-architecture/README.md)으로 이동하세요.

## 여기에는 왜 직접적인 능력 목록이 없나요?

- 기존 기능 목록에 현재 저장소에서 직접 확인하기 어려운 설명이 섞여 있었습니다.
- 일부 기능은 특정 프로젝트, 플러그인 또는 전달 환경에서만 확립되며 일반 진입 약속으로는 적합하지 않습니다.
- 현재 구조는 "설치 및 최초 사용", "일상 사용", "디자인 및 아키텍처", "API 참조"로 분리되어 있습니다. 독자들은 이 입구를 따라 들어가야 합니다.

## 현행판단원칙

현재 주제 페이지, 소스 코드 또는 현재 플러그인 디렉토리에서 특정 기능이 확인되지 않은 경우 현재 문서 및 소스 코드가 이전 기능 목록보다 우선해야 합니다.
