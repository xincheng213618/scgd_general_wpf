# 시작하기

이 장은 후속 장과의 중복을 피하기 위해 ColorVision을 처음 접할 때 가장 필요한 입구만 유지합니다.

## 적용 범위

- 신규 사용자가 설치, 시작 및 기본 확인을 완료하고 싶어합니다.
- 새로운 동료는 주요 프로그램, 장비, 프로세스 및 플러그인이 어디에 있는지 빨리 알고 싶어합니다.
- 개발자가 소스코드 빌드를 위한 최단 경로를 확인하고 싶어함

## 여기서 무엇을 찾을 수 있나요?

- 제품 포지셔닝 및 일반적인 사용 시나리오
- Windows 환경 요구사항 및 설치 전 준비사항
- 메인 프로그램 최초 실행 후 기본 동작 경로
- 소스 코드에서 기본 프로그램을 실행하는 최소한의 단계

## 소스 코드에서 시작

현재 창고는 주로 Windows WPF 및 x64용입니다. 먼저 종속성 복구를 완료한 다음 기본 프로그램을 빌드하는 것이 좋습니다.

``파워셸
닷넷 복원
dotnet build -p:플랫폼=x64
dotnet run --project ColorVision/ColorVision.csproj
```
## 계속 읽기

- 인터페이스 및 일일 작업 확인: [사용자 가이드](../01-user-guide/README.md)로 이동합니다.
- 시스템 설계 및 모듈 경계를 보려면 [아키텍처 설계](../03-architecture/README.md)로 이동하세요.
- 웨어하우스 디렉토리 및 모듈 분업을 확인하려면 [프로젝트 구조 개요](../05-resources/project-structure/README.md)로 이동하세요.
- 2차 개발을 원하시는 경우 : [개발자 가이드](../02-developer-guide/README.md)로 이동하세요.

## 설명

- 이전 버전의 소개 페이지에 있던 아키텍처, 설치 프로그램 구현, 릴리스 스크립트와 관련된 긴 내용이 해당 장으로 압축되어 더 이상 여기에 유지되지 않습니다.
- 문서가 현재 코드 동작과 일치하지 않는 경우 소스 코드와 실제 빌드 결과가 우선합니다.
