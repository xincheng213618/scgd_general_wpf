# 시스템 요구사항

이 페이지에서는 ColorVision을 설치하고 실행하기 전에 준비해야 하는 환경에 대해서만 설명하고 설치 프로그램 구현, 웨어하우스 구조 또는 아키텍처 설계에 대해서는 확장하지 않습니다.

## 적용 가능 객체

- 기성 패키지를 설치하여 사용하고 싶은 최종 사용자
- 메인 프로그램을 소스에서 직접 빌드하고 실행하고 싶은 개발자
- 설치 단계를 계속 읽기 전에 시스템 조건이 충족되는지 확인하고 싶은 동료

## 작동 환경 요구 사항

### 운영 체제

- Windows 10 버전 1903 이상 또는 Windows 11
- x64 시스템 환경 사용을 권장합니다.
- 처음으로 시스템 서비스를 설치, 업그레이드, 구성하는 경우에는 관리자 권한이 있는 계정을 사용하는 것이 좋습니다.

### 하드웨어 및 디스플레이

- 1920x1080 이상의 해상도 사용을 권장합니다.
- 이미지 처리 및 프로세스 실행을 위해 충분한 메모리와 디스크 공간을 확보하는 것이 좋습니다.
- 카메라, 분광계, 모터, 기타 장비를 연결해야 하는 경우 해당 드라이버 및 통신 환경이 준비되어 있는지 미리 확인해야 합니다.

### 네트워크 및 권한

- 최초 설치 후 업데이트 확인, 플러그인 마켓 접속, 원격 서비스 연결이 필요한 경우 네트워크가 가능한지 확인해주세요.
- 배포 환경에 엄격한 제한이 있는 경우 프로그램 디렉터리, 로그 디렉터리, 사용자 문서 디렉터리에 쓰기 권한이 있는지 미리 확인하시기 바랍니다.

## 설치 패키지에서 실행할 때 주의할 점은 무엇인가요?

- 설치 프로그램은 자체 프로세스에 따라 필요한 구성 요소를 확인하고 배포합니다. 필수 구성 요소가 누락되었다는 메시지가 나타나면 설치 프로그램 메시지에 따라 필수 구성 요소를 완료하세요.
- 일부 서비스 관리, 장치 드라이버 구성 또는 시스템 수준 쓰기 작업에는 관리자 권한이 필요할 수 있습니다.
- 환경에 이전 버전이 있는 경우, 새로 설치하기 전에 업그레이드를 완료하거나 제거하는 것이 좋습니다.

## 소스코드로 빌드할 때 준비해야 할 것

현재 창고는 주로 Windows WPF 및 x64용입니다. 다음과 같은 환경을 준비하는 것이 좋습니다.

- .NET 8.0 SDK 또는 .NET 10.0 SDK
- Visual Studio 2022 이상(권장)
- Git 및 PowerShell(리포지토리 가져오기 및 스크립트 실행용)

기본 프로그램을 빌드하려면 x64 플랫폼을 사용하는 것이 좋습니다.

``파워셸
닷넷 복원
dotnet build .\ColorVision\ColorVision.csproj -p:Platform=x64
```
## 런타임 종속성 설명(소스 코드 시나리오)

설치 패키지를 통해 배포하는 대신 소스 코드에서 직접 실행하는 경우 실행 출력에 다음 종속성이 포함되어야 한다는 점에도 유의해야 합니다.

- OpenCvSharp Windows 런타임
-`DLL/CVCommCore.dll`
-`DLL/MQTTMessageLib.dll`
- 메인 프로그램에 필요한 `log4net.config` 및 관련 리소스 파일

이러한 내용은 일반적으로 프로젝트 참조 및 복사 규칙에 따라 처리됩니다. 프로그램을 빌드할 수 있지만 시작에 실패하는 경우 먼저 출력 디렉터리에 위의 종속성이 누락되었는지 확인하세요.
