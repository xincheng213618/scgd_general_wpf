#ColorVision.Database

이 페이지에서는 현재 UI/ColorVision.Database에 구현된 데이터 액세스 및 데이터베이스 검색 기능에 대해서만 설명합니다. 기존 템플릿의 "데이터베이스 튜토리얼 + 샘플 조각 + 구축 검증 기록"의 혼합 작성 방식은 더 이상 유지되지 않습니다.

## 모듈 포지셔닝

`ColorVision.Database`에는 현재 두 가지 유형의 책임이 있습니다:

- 비즈니스 엔터티 및 DAO를 위한 기본 데이터 액세스 계층
- 런타임 유지 관리를 위한 데이터베이스 브라우저 및 공급자 시스템

그 중에서 현재 우선적으로 주목해야 할 주요 라인은 전통적인 엔터티 스캐닝 모드가 아닌 "데이터베이스 우선" 브라우저 체인입니다.

## 현재 가장 중요한 디렉터리 및 파일

프로젝트 디렉터리를 살펴보면 먼저 알아야 할 가장 중요한 사항은 다음과 같습니다.

- `DatabaseBrowserWindow.xaml(.cs)`: 데이터베이스 브라우저 메인 창
- `DatabaseBrowserProviderRegistry.cs`: 공급자 등록 및 지연 로딩 입구
- `IDatabaseBrowserProvider.cs`: 브라우저 공급자 계약
- `DatabaseBrowserModels.cs`: 라이브러리, 테이블, 열, 페이징 모델
- `MySqlControl.cs`: MySQL 구성 및 공급자 생성
- `BaseTableDao.cs`, `EntityBase.cs`, `ViewEntity.cs`: 비즈니스 엔터티 액세스 계층의 기본 유형

## 키 입력 유형

### 데이터베이스브라우저 창

`DatabaseBrowserWindow`는 현재 데이터베이스 유지 관리 경험의 주요 입구입니다. 다음을 담당합니다.

- 데이터 소스, 라이브러리, 테이블의 트리 구조 표시
- 오른쪽의 `DataTable`을 사용하여 결과 집합을 찾아보세요.
- 검색, 페이징, 정렬 지원
- 추가, 업데이트, 삭제 등 일반적인 테이블 수준 작업 수행

주요 기능은 현재 브라우저가 UI를 구동하기 위해 더 이상 C# 엔터티 정의에 의존하지 않는다는 것입니다. 대신 먼저 실제 데이터베이스 연결에서 라이브러리, 테이블 및 열 정보를 가져온 다음 표시하고 다시 쓰는 방법을 결정합니다.

### 데이터베이스브라우저공급자 레지스트리

`DatabaseBrowserProviderRegistry`는 탐색 가능한 데이터 소스의 통합 관리를 담당합니다. 현재는 기본 제공자를 지연 로드하여 브라우저에 노출합니다.

- MySQL 기본 공급자
- 다른 발신자가 직접 등록한 공급자

따라서 이는 현재 데이터베이스 브라우저 시스템의 스케줄링 진입점입니다.

### IDatabaseBrowserProvider

`IDatabaseBrowserProvider`는 데이터베이스 브라우저의 가장 중요한 추상 경계입니다. 현재 다음을 제공하기 위한 구현이 필요합니다.

- 도서관 목록
- 테이블 목록
- 컬럼 정보
- 페이징 쿼리
- 삽입, 업데이트, 삭제

따라서 이 모듈의 핵심 확장 포인트는 "엔티티 클래스 추가"가 아니라 "새 공급자 등록"입니다.

### 마이SQL컨트롤

`MySqlControl`은 현재 연결 구성 개체일 뿐만 아니라 다음도 가정합니다.

- MySQL 구성 지속성
- 연결 문자열 구성
- MySQL 브라우저 공급자 생성

따라서 MySQL 관련 항목은 `BaseTableDao<T>`만 보는 것이 아니라 그에 따라 직접 보아야 합니다.

### BaseTableDao/EntityBase/ViewEntity

이러한 유형은 현재 비즈니스 계층 엔터티 액세스의 기반으로 남아 있습니다.

- `IEntity`는 `Id`를 통합합니다.
- `EntityBase`는 기본 키 매핑 기본 클래스를 제공합니다.
- 바인딩 가능한 엔터티에 대한 'ViewEntity'
- `BaseTableDao<T>`는 기존 비즈니스 코드를 계속 제공합니다.

그러나 더 이상 현재 데이터베이스 UI 검색 체인의 유일한 중심이 아닙니다.

## 현재 런타임 메인 체인

이 모듈 세트는 현재 다음 체인에 더 가깝습니다.

1. `DatabaseBrowserWindow`는 `DatabaseBrowserProviderRegistry`에서 사용 가능한 공급자를 가져옵니다.
2. 공급자는 라이브러리, 테이블 및 열 정보를 반환합니다.
3. 브라우저는 테이블 구조에 따라 'DataTable' 결과를 동적으로 표시합니다.
4. 데이터 추가, 편집 및 삭제는 공급자의 범용 쓰기 인터페이스를 통해 데이터베이스로 반환됩니다.
5. 비즈니스 코드의 경우 엔터티와 DAO 시스템을 계속 병렬로 사용할 수 있지만 브라우저 UI는 더 이상 제어되지 않습니다.

## 현재 구현의 경계는 무엇입니까?

### 브라우저 메인라인은 이미 "데이터베이스 우선"입니다.

이는 현재 가장 중요한 경계 변경입니다. 이전 아이디어는 "엔티티를 먼저 갖고 그 다음 테이블 인터페이스를 갖는" 경향이 있었습니다. 이제는 실제 데이터베이스 구조에서 직접 탐색 및 유지 관리 인터페이스를 생성하는 것이 더 중요합니다.

### 공급자가 엔터티보다 더 중요합니다.

새 데이터베이스 소스를 확장하려는 경우 현재 우선 순위는 엔터티 클래스 배치를 시스템에 추가하는 대신 'IDatabaseBrowserProvider'를 구현하고 등록하는 것입니다.

### DAO 시스템은 여전히 ​​존재하지만 이것이 유일한 입구는 아닙니다.

`BaseTableDao<T>`와 같은 유형은 여전히 ​​기존 비즈니스 코드를 제공하지만 이 모듈을 읽을 때 더 이상 데이터베이스 기능의 유일한 중심으로 작성할 수 없습니다.

## 이 모듈을 읽는 방법이 현재 더 적합합니다.

### 데이터베이스 브라우저 메인링크를 보고싶다

먼저 살펴보세요:

-`DatabaseBrowserWindow.xaml.cs`
-`DatabaseBrowserProviderRegistry.cs`
- `IDatabaseBrowserProvider.cs`

### MySQL에 대한 실제 접근을 보고 싶다

먼저 살펴보세요:

-`MySqlControl.cs`

### 비즈니스 항목 액세스 레이어를 보고 싶습니다.

먼저 살펴보세요:

- `IEntity.cs`
- `EntityBase.cs`
- `ViewEntity.cs`
- `BaseTableDao.cs`

## 이 페이지에서는 더 이상 아무것도 수행하지 않습니다.

이 페이지에서는 더 이상 다음과 같은 고위험 콘텐츠를 유지하지 않습니다.

- 튜토리얼 스타일의 샘플 코드 스택
- "모범 사례" 일반 단락
- 검증 기록을 수동으로 구축
- 데이터베이스 모듈을 엔터티 클래스에서만 작동하는 이전 모델로 작성합니다.
