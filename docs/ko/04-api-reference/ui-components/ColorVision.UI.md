#ColorVision.UI

이 페이지에는 현재 가장 중요한 인프라와 'UI/ColorVision.UI/' 항목 유형만 유지되며 이전 문서의 "버전 목록 + 전체 의사 API + 업데이트 로그" 작성 방법은 더 이상 유지되지 않습니다.

## 모듈 포지셔닝

'ColorVision.UI'는 단일 컨트롤이 아니지만 데스크톱 애플리케이션을 위한 많은 UI 인프라가 있는 곳입니다. 현재 역할은 주로 메인 프로그램, 엔진 및 기타 UI 하위 프로젝트에서 재사용되는 "UI 셸 및 공용 서비스 컬렉션"에 더 가깝습니다.

디렉토리 구조로 볼 때, 여기에 포함되는 내용은 최소한 다음을 포함합니다.

- 읽기, 쓰기 및 환경 경로 구성
- 플러그인 로딩 및 플러그인 패키지 처리
- 메뉴 시스템
- 속성 편집기
- 단축키 시스템
- 다국어 리소스
- 로그 관련 UI 구성
- 셸, 검색, 상태 표시줄, 페이지 및 일반 컨트롤

따라서 단일 "컨트롤 라이브러리"가 아니며 안정적인 공개 API 표면을 갖춘 표준 SDK로 작성하기에 적합하지 않습니다.

## 현재 가장 중요한 디렉토리

빠르게 인지도를 구축하려면 먼저 다음 디렉터리를 살펴보는 것이 좋습니다.

- `플러그인/`: 플러그인 검색, 메타데이터, 종속성 확인, 압축 풀기 및 업데이트
- `PropertyEditor/`: 속성 편집 창, 트리 노드, 편집기 유형 시스템
- `메뉴/`: 메뉴 등록 및 동적 메뉴 새로 고침
- `HotKey/`: 전역 및 창 수준 단축키
- `Languages/`: 언어 및 리소스 전환
- `LogImp/`: 로그 관련 구성 및 창 상태
- `ConfigSetting/` 및 루트 디렉터리 `ConfigHandler.cs`: 시스템 항목 구성
- `Shell/`, `Serach/`, `StatusBar/`: 데스크톱 대화형 보조 기능

## 키 입력 유형

### `구성 핸들러`

`ConfigHandler`는 이 프로젝트의 핵심 인프라 중 하나입니다. 많은 'IConfig' 구성 개체는 결국 해당 개체 또는 관련 구성 서비스 주변에서 읽고, 캐시하고, 저장됩니다.

문제가 "설정이 저장되지 않음", "구성이 로드되지 않음", "기본값 비정상"으로 나타나는 경우 일반적으로 이 링크를 먼저 확인하십시오.

### `플러그인로더`

'PluginLoader'는 현재 플러그인 런타임 로딩을 담당합니다. 이것이 수행하는 작업은 "DLL 검색"뿐만 아니라 다음과 같습니다.

- `Plugins/` 디렉토리를 검색하세요.
-`manifest.json`을 읽어보세요.
- 선택적인 `.deps.json` 구문 분석
- `ColorVision.*` 종속성 버전을 확인하세요.
- 플러그인 어셈블리의 최종 로딩

플러그인 관련 문서에 "반사 스캐닝 플러그인 유형"만 읽으면 왜곡되는 경우가 많은 이유가 여기에 있습니다.

### `메뉴관리자`

`MenuManager`는 메뉴 시스템의 중심 개체입니다. 많은 동적 메뉴, 최근 파일 새로 고침 및 플러그인 메뉴 항목은 결국 등록 또는 새로 고침 체인에 속하게 됩니다.

따라서 이 부분은 정적 XAML 메뉴 정의 집합이라기보다는 애플리케이션 셸의 메뉴 코디네이터에 더 가깝습니다.

### `속성 편집기`

`PropertyEditor/`는 현재 속성 편집 경험을 담당하는 메인 체인입니다.

- `PropertyEditorWindow`
- `PropertyTreeNode`
- 에디터 종류 및 보조 클래스

웨어하우스의 'Category', 'DisplayName', 'Description' 및 'PropertyEditorType'과 같은 속성을 가진 다수의 개체와 함께 사용되는 이 시스템은 현재 동적 속성 편집 환경의 기초입니다.

### `핫키`

단축키 시스템은 현재 단일 지점으로 구현되지 않고 다음과 같이 구분됩니다.

- `글로벌핫키/`
- `WindowHotKey/`
- 'HotKeys' 및 해당 구성 및 설정 창

따라서 단축키를 변경할 때 일반적으로 시스템 수준 단축키를 변경하는지 창 전체 단축키를 변경하는지 구별해야 합니다.

### `언어`

다국어 리소스 및 UI 문화 전환 관련 기능을 중앙에서 관리합니다. 기본 프로그램의 시작 단계에서 UI Culture를 설정한 후 많은 인터페이스 리소스 로딩이 이로 인해 영향을 받습니다.

### `LogImp`

로그 관련 UI 구성과 로컬 로그 창 상태도 이 프로젝트에 배치됩니다. 완전한 로그 백엔드 자체가 아니라 "로깅 표시 및 구성 일치"에 관한 것입니다.

## 이 프로젝트는 현재 잘못 작성되었을 가능성이 가장 높습니다.

### 단일 컨트롤 라이브러리가 아닙니다.

오래된 문서에서는 `ColorVision.UI`를 "핵심 UI 제어 패키지"로 작성하는 것을 좋아합니다. 현재 코드는 이보다 훨씬 더 복잡합니다. 또한 플러그인, 구성, 메뉴, 단축키, 속성 편집 및 다중 언어와 같은 교차 기능을 수행합니다.

### 플러그인 시스템은 확장점 정의 자체와 동일하지 않습니다.

'PluginLoader'는 여기에 있지만 플러그인이 실제로 확장하는 기능은 여전히 각 플러그인 어셈블리와 구현된 메뉴, 템플릿, 서비스 및 결과 보기 인터페이스에 따라 다릅니다.

### 이 페이지의 권한은 "글로벌 RBAC 센터"로 일반화되어서는 안 됩니다.

전역적으로 대략적으로 세분화된 권한은 현재 `Authorization.Instance.PermissionMode`에서 나오는 반면, 세부적으로 세분화된 로컬 RBAC 하위 시스템은 주로 `UI/ColorVision.Solution/Rbac/`에 있습니다. 'ColorVision.UI'는 인증 인프라와 공개 종속성을 제공하므로 여기에서 전체 권한 플랫폼으로 작성해서는 안 됩니다.

## 현재 이 프로젝트를 어떻게 읽는 것이 더 적합합니까?

### 구성 및 글로벌 서비스를 보고 싶습니다.

먼저 살펴보세요:

-`ConfigHandler.cs`
-`Environments.cs`
-`FileProcessorFactory.cs`

### 플러그인 실행시간을 보고싶다

먼저 살펴보세요:

- `플러그인/PluginLoader.cs`
- `플러그인/PluginManifest.cs`
- `플러그인/PluginInfo.cs`

### 속성 편집 시스템을 보고싶다

먼저 살펴보세요:

- `PropertyEditor/PropertyEditorWindow.xaml(.cs)`
- `PropertyEditor/PropertyTreeNode.cs`
-`PropertyEditor/PropertyEditors.cs`

### 메뉴와 단축키를 보고싶다

먼저 살펴보세요:

-`메뉴/MenuManager.cs`
- `HotKey/HotKeys.cs`
- `핫키/글로벌핫키/`
-`핫키/WindowHotKey/`

## 이 페이지에서는 더 이상 아무것도 수행하지 않습니다.

이 페이지에서는 더 이상 다음과 같은 고위험 콘텐츠를 유지하지 않습니다.

- 오래된 버전 번호 및 대상 프레임워크 목록
- 반원들을 위한 검증되지 않은 의사코드의 큰 부분
- `ColorVision.UI`를 안정적인 공개 SDK로 만듭니다.
- 권한, 플러그인, 로그 및 기타 교차 기능을 자체 완전한 플랫폼으로 전환

향후 특정 하위 시스템을 보완해야 하는 경우 여기에 "크고 포괄적인" 지침을 계속 쌓아두는 대신 해당 주제 페이지로 직접 이동해야 합니다.
