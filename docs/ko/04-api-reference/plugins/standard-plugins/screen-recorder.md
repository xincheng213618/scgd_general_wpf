# 스크린 레코더 상태 설명

해당 소스 코드 프로젝트를 현재 `scgd_general_wpf` 작업 공간에서 더 이상 찾을 수 없기 때문에 이 페이지에서는 더 이상 ScreenRecorder를 현재 웨어하우스의 표준 플러그인 구현으로 작성하지 않습니다.

## 현재 작업공간의 실제 상태

현재 창고 구조에 따라 확인하십시오.

- `Plugins/` 디렉토리 아래에는 `ScreenRecorder/` 소스 코드 디렉토리가 없습니다.
- 작업공간에 해당 플러그인 프로젝트 파일이 없습니다.
- 현재 플러그인 색인 페이지 [Plugins/README.md](../../../../Plugins/README.md)에는 기존 플러그인 디렉토리로 나열되어 있지 않습니다.
- 해당 플러그인 구현이 현재 웨어하우스에 존재함을 나타내기보다는 기록 상태 설명 페이지에 계속 액세스할 수 있도록 하기 위해 현재 문서의 사이드바에 유지됩니다.

따라서 이 페이지는 "고성능 화면 녹화 플러그인 API 매뉴얼"의 이전 버전을 계속 유지할 수 없습니다. 그렇지 않으면 기록 설명이 현재 구현으로 잘못 작성됩니다.

## 현재 이 페이지에는 어떤 정보가 보관되나요?

현재 남은 결론은 하나뿐입니다.

ScreenRecorder 관련 문서는 현재 소스 코드를 기반으로 확인할 수 있는 API 참조 페이지가 아닌 이 작업 공간의 기록 잔여 페이지입니다.

이 플러그인이 나중에 다시 도입되면 최소한 다음 실제 앵커를 기반으로 새 문서를 다시 작성해야 합니다.

- 플러그인 디렉터리 및 프로젝트 파일
-`manifest.json`
- 메뉴 또는 공급자 액세스 포인트
- 녹화창 및 녹화소스 관리 구현
- 구성 및 출력 드롭 포인트

인코딩 형식, 녹음 소스 유형, 오버레이 API 등에 대한 설명은 해당 코드가 다시 나타날 때까지 추가하면 안 됩니다.

## 오래된 원고를 계속 유지하는 것은 어떨까요?

이전 페이지에서는 ScreenRecorder를 현재 실제 화면 녹화 플러그인으로 설명하고 녹화 소스, 인코더, 오버레이 및 고급 기능 목록을 제공합니다. 그러나 현재 소스 코드 트리에는 이러한 내용에 대한 검증 가능한 구현이 없습니다.

오래된 초안을 계속 다듬으면 문서가 더욱 완벽해 보일 뿐이지만 현재 창고와는 점점 더 단절됩니다.
