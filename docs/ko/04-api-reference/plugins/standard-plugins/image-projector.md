# ImageProjector 상태 설명

해당 소스 코드 프로젝트가 현재 `scgd_general_wpf` 작업 공간에서 더 이상 찾을 수 없기 때문에 이 페이지에서는 더 이상 ImageProjector를 현재 웨어하우스의 표준 플러그인 구현으로 작성하지 않습니다.

## 현재 작업공간의 실제 상태

현재 창고 구조에 따라 확인하십시오.

- `Plugins/` 디렉토리 아래에는 `ImageProjector/` 소스 코드 디렉토리가 없습니다.
- 작업공간에 해당 플러그인 프로젝트 파일이 없습니다.
- 현재 플러그인 색인 페이지 [Plugins/README.md](../../../../Plugins/README.md)에는 기존 플러그인 디렉토리로 나열되어 있지 않습니다.
- 해당 플러그인 구현이 현재 웨어하우스에 존재함을 나타내기보다는 기록 상태 설명 페이지에 계속 액세스할 수 있도록 하기 위해 현재 문서의 사이드바에 유지됩니다.

따라서 이 페이지는 "다중 모니터 프로젝션 도구 전체 매뉴얼"의 이전 버전을 계속 유지할 수 없습니다. 그렇지 않으면 기록 기능이 현재 소스 코드 사실로 기록됩니다.

## 현재 이 페이지에는 어떤 정보가 보관되나요?

현재 남은 결론은 하나뿐입니다.

ImageProjector 관련 문서는 현재 소스 코드를 기반으로 확인할 수 있는 API 참조 페이지가 아닌 이 작업 공간의 기록 잔여 페이지입니다.

이 플러그인이 나중에 다시 도입되면 최소한 다음 실제 앵커를 기반으로 새 문서를 다시 작성해야 합니다.

- 플러그인 디렉터리 및 프로젝트 파일
-`manifest.json`
- 메뉴 또는 공급자 액세스 포인트
- 메인창 또는 프로젝션창 구현
- 구성 배치

이 코드가 다시 나타날 때까지 기능 설명, 구성 테이블 또는 API 목록을 더 이상 추가해서는 안 됩니다.

## 오래된 원고를 계속 유지하는 것은 어떨까요?

이전 페이지에서는 ImageProjector를 현재 존재하는 플러그인으로 설명하고 전체 기능 목록, 디스플레이 모드 설명 및 구성 요소 구조를 제공합니다. 그러나 현재 소스 코드 트리에서는 이러한 설명에 더 이상 검증 가능한 구현 전달자가 없습니다.

문서가 계속해서 이렇게 작성되면 "과거에 존재했을 수도 있는 기능"을 "현재 창고 사실"로 위장하게 됩니다. 이것이 바로 이번 정리가 피하려고 하는 것입니다.
