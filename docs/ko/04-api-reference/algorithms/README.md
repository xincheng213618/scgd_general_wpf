# 알고리즘 및 템플릿 개요

이 장은 이제 "템플릿 시스템 및 알고리즘 액세스 체인"에 대한 소개로 압축되었으며, 모든 이미지 처리 방법을 백과사전 카탈로그로 평면화하는 기존 작성 방법은 더 이상 유지되지 않습니다.

## 이 장에서는 무엇을 다루고 있나요?

여기서 "알고리즘"은 웨어하우스에 있는 모든 기본 이미지 처리 코드의 전체 목록이 아니라 주로 `Engine/ColorVision.Engine/Templates/` 및 주변 액세스 체인에 해당합니다. 현재 우선순위는 다음과 같습니다:

- 템플릿을 검색, 로드, 관리 및 편집하는 방법.
- Flow 템플릿을 `FlowEngineLib`에 연결하는 방법.
- 특수 편집기를 통해 JSON 템플릿을 시스템에 입력하는 방법.
- ARVR, POI 등 비즈니스 템플릿 패밀리를 알고리즘 서비스와 연결하는 방법.

OpenCV 레벨에서 기본 처리 기능을 찾고 있다면 진입점은 일반적으로 이 장에 없지만 `Engine/cvColorVision/`, `UI/ColorVision.Core/` 또는 기본 DLL 측에 더 가깝습니다.

## 현재 챕터 구조

### 참가 페이지

- [알고리즘 시스템 개요](./overview.md): 현재 구현 체인에 대한 전반적인 설명입니다. 이 페이지를 먼저 읽으면 시간이 절약됩니다.

### 특별 카탈로그

- `템플릿/`: 템플릿 관리, 프로세스 템플릿, JSON 템플릿, POI/ARVR 및 기타 특수 페이지.
- `Detectors/`: 소수의 결함/감지 주제.
- `프리미티브/`: 소수의 기본 구성 요소에 대한 설명입니다.

이 디렉토리에는 여전히 일부 역사적인 페이지가 있지만 더 이상 이 장의 홈페이지에 안정적인 입구로 배치되지 않습니다.

## 가장 먼저 알아야 할 코드 앵커

현재 상황으로 볼 때 이러한 유형의 파일은 템플릿 및 알고리즘 링크에 대해 가장 알아야 할 가치가 있습니다.

- `Templates/TemplateContorl.cs`: 템플릿 검색 및 등록 입구.
- `Templates/TemplateManagerWindow.xaml.cs`: 템플릿 관리 창입니다.
- `Templates/TemplateEditorWindow.xaml.cs`: 템플릿 편집 창입니다.
- `Templates/Flow/TemplateFlow.cs`: 프로세스 템플릿 및 프로세스 편집기에 대한 액세스 지점입니다.
- `Templates/Jsons/ITemplateJson.cs`: JSON 템플릿의 일반적인 로드/가져오기 및 내보내기 논리입니다.
- `Templates/Jsons/EditTemplateJson.xaml(.cs)`: 텍스트/속성의 두 가지 편집 모드를 담당하는 JSON 템플릿 편집 컨트롤입니다.
- `Templates/POI/AlgorithmImp/AlgorithmPOI.cs`, `Templates/ARVR/*/Algorithm*.cs`: 일반적인 비즈니스 알고리즘 UI 및 메시지 어셈블리 입구.

## 현재의 여러 키 경계

- 많은 '알고리즘*' 클래스 자체가 최종 컴퓨팅 코어가 아닙니다. 현재는 템플릿 매개변수, 파일 경로, 장치 정보를 수집한 다음 MQTT/서비스 체인을 통해 실행 요청을 발행하는 일을 더 많이 담당합니다.
- 'POI'는 고립된 주제가 아니며, 여전히 현재 코드의 여러 알고리즘 계열이 공유하는 업스트림 템플릿이자 매개변수 소스입니다.
- 'Flow' 템플릿은 표현 형식이 다르지만 여전히 동일한 템플릿 시스템의 일부이므로 일반 템플릿 체인과 완전히 분리되어서는 안 됩니다.
- JSON 템플릿과 기존의 강력한 형식의 템플릿이 현재 공존합니다. 읽을 때 시스템이 하나의 템플릿 정의 방법만 예약한다고 가정하지 마십시오.

## 추천읽기순서

1. 먼저 [알고리즘 시스템 개요](./overview.md)를 읽고 런타임 시 메인 체인에 대한 인식을 설정합니다.
2. 그런 다음 [템플릿 모듈 분석](../../03-architecture/comComponents/templates/analytic.md)과 비교하여 디렉토리 및 등록 항목을 이해합니다.
3. 프로세스 템플릿이 걱정된다면 [FlowEngineLib 아키텍처](../../03-architecture/comComponents/engine/flow-engine.md)를 살펴보세요.
4. 마지막으로 특정 비즈니스 도메인에 따라 'templates/' 아래에 단일 페이지를 입력하고 항상 소스 코드와 대조하여 읽습니다.
