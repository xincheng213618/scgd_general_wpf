## 현재 구성 원칙

- 최상위 홈페이지에는 요약된 개요 페이지만 표시되며, 더 이상 모든 단일 페이지를 첫 번째 화면 진입으로 간주하지 않습니다.
- 분할된 주제 페이지는 여전히 해당 디렉토리에 유지되지만 기본적으로 소스 코드에 대해 읽어야 합니다.
- 문서가 구현과 일치하지 않는 경우 소스 코드, XML 주석 및 실제 실행 동작이 우선합니다.

## 현재 챕터 경계

- `ui-components/`는 주로 WPF UI 측면 모듈과 데스크톱 셸을 다룹니다.
- `engine-components/`는 전체 알고리즘 백과사전보다는 주로 Engine 디렉토리의 런타임 모듈을 다룹니다.
- 'algorithms/'는 모든 기본 이미지 연산자 디렉터리보다는 주로 템플릿 시스템과 알고리즘 액세스 체인을 다룹니다.
- `plugins/`는 현재 `Plugins/` 디렉터리에 실제로 존재하는 플러그인만 현재 기능으로 다룹니다.
- `projects/`는 `Projects/`에 실제로 존재하는 고객 프로젝트, 연동 데모, 인수인계 매트릭스, 현장 조사 플레이북을 다룹니다.
- `extensions/`는 현재 실제 코드와 직접적으로 일치할 수 있는 Flow 노드 확장 및 확장 지점 주제를 주로 보유하고 있습니다.

## 보충 독서 방법

- `plugins/` 아래 페이지에는 플러그인 항목 "현재 소스 코드 일치 가능"과 "기록 상태 설명" 페이지도 포함되어 있습니다. 단일 페이지에 들어가기 전에 장 개요를 읽어야 합니다.
- '확장/'은 현재 매우 좁은 범위를 다루고 있으며 주로 코드 앵커와 직접 일치할 수 있는 흐름 노드 확장과 같은 주제를 다루며 완전한 확장 메커니즘 백과사전은 아닙니다.
- `projects/`는 고객 납품, 현장 프로토콜, 결과 출력을 확인하는 진입점입니다. 범용 플러그인 기능과 분리해서 읽으세요.

## 권장 읽기 순서

1. 먼저 [UI 구성요소 개요](./ui-components/README.md)를 읽고 클라이언트 셸 및 UI 인프라를 이해하세요.
2. 서비스, 템플릿, 프로세스 런타임을 이해하려면 [엔진 구성요소 개요](./engine-components/README.md)를 다시 살펴보세요.
3. 마지막으로 [알고리즘 및 템플릿 개요](./algorithms/README.md)와 [알고리즘 시스템 개요](./algorithms/overview.md)를 살펴보고 템플릿과 알고리즘 액세스 링크를 연결하세요.
4. 고객 프로젝트를 확인해야 할 경우, [프로젝트 패키지 개요](./projects/README.md)와 [현재 프로젝트 문서 커버리지](./projects/current-project-coverage.md)에 들어가세요.
5. 플러그인이나 확장 포인트의 상태를 확인해야 할 경우, [플러그인 및 상태 페이지](./plugins/README.md) 및 [확장 포인트 개요](./extensions/README.md)에 들어가세요.
