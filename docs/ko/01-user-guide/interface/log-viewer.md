# 로그 뷰어

로그 뷰어는 시작 예외, 장치 통신 문제, 프로세스 실행 오류 및 플러그인 로드 실패를 찾는 데 사용됩니다. "인터페이스가 응답하지 않고, 기능이 작동하지 않고, 장치를 연결할 수 없습니다"와 같은 문제가 발생하면 일반적으로 맹목적으로 추측하는 것보다 로그를 먼저 보는 것이 더 빠릅니다.

## 언제 켜야 하나요?

- 메인창이나 특정 기능이 열리지 않습니다.
- 비정상적인 기기 연결 상태
- 프로세스 실행 중에 오류가 보고되지만 인터페이스 프롬프트가 충분히 명확하지 않습니다.
- 플러그인, 서비스, 템플릿의 비정상적인 로딩 동작

## 평소 사용하는 기능

- 실시간으로 새로운 로그 보기
- 검색 키워드
- 로그 수준으로 필터링
- 특정 범위의 기록 로그 보기
- 자동 스크롤과 수동 읽기 간 전환

## 기본 사용법

1. 도움말 메뉴 또는 해당 항목에서 로그 창을 엽니다.
2. 먼저 가장 최근의 로그를 읽고 오류가 발생한 대략적인 시간을 확인합니다.
3. 모듈 이름, 장치 이름, 예외 키워드 또는 파일 이름을 검색하려면 키워드를 사용하십시오.
4. 로그가 너무 많으면 '경고' 또는 '오류'로 수준을 높여 범위를 좁힙니다.

## 검색을 보다 효율적으로 수행하는 방법

- 기기 이름, 템플릿 이름, `error`, `timeout` 등 간단한 키워드를 먼저 사용하세요.
- 결과가 너무 많을 경우 여러 키워드를 결합하세요.
- 꼭 필요할 때만 정규식을 사용하세요.

입력 후 검색 상자가 빨간색으로 바뀌면 일반적으로 정규식이 잘못 작성되었음을 의미합니다. 이 경우 먼저 공통 키워드 검색으로 돌아가는 것이 더 안정적입니다.

## FAQ

### 로그창을 열면 내용이 거의 없습니다

- 먼저 자동 새로고침이 켜져 있는지 확인하세요.
- 현재 로그 레벨이 너무 높은지 다시 확인하세요.
- 새 로그만 보려면 현재 로드 전략이 비어 있지 않은지 확인하세요.

### 로그가 많고 인터페이스가 읽기 쉽지 않습니다.

- 먼저 레벨을 '경고' 또는 '오류'로 전환하세요.
- 키워드를 사용하여 범위를 좁히세요.
- 일시적인 문제 해결인 경우 아직 모든 기록 로그를 유지하려고 하지 마세요.

### 잘못된 검색결과

- 먼저 여러 키워드가 동시에 일치해야 하는지 확인하세요.
- 정규식을 사용하는 경우 특수문자를 이스케이프해야 하는지 먼저 확인하세요.
- 검색 결과를 찾을 수 없는 경우, 필터링되지 않은 보기로 돌아가서 해당 시점을 다시 찾아보세요.

## 문제 해결 제안

### 장비 문제

- 기기명, 카메라코드, 서비스명을 먼저 검색하세요.
- 그런 다음 [장치 서비스 개요](../devices/overview.md)를 보려면 링크하세요.

### 프로세스 문제

- 템플릿 이름, 노드 이름 또는 실행 실패 키워드를 먼저 검색하세요.
- 그런 다음 링크하여 [프로세스 실행 및 디버깅](../workflow/execution.md)을 확인하세요.

### 플러그인 또는 시작 문제

- 애플리케이션 시작 단계에서 최신 로그를 먼저 확인하세요.
- DLL, 버전, 경로 또는 로딩 실패 메시지가 있는지 주의하세요.
