# 터미널

터미널 패널은 두 가지 유형의 작업, 즉 스크립트 실행과 명령줄 출력 직접 관찰에 적합합니다. 별도의 개발 문서 입구라기보다는 메인 프로그램의 작업 보조 영역에 가깝습니다.

## 언제 켜야 하나요?

- PowerShell 또는 CMD 명령을 직접 실행해야 함
- 현재 프로젝트 컨텍스트에서 스크립트를 실행하고 싶습니다.
- 스크립트 출력, 오류 보고 또는 대화형 프롬프트를 관찰해야 합니다.
- 메인 프로그램을 종료하지 않고 빠르게 명령줄 확인을 하고 싶습니다.

## 평소에는 어떻게 사용하시나요?

### 쉘 선택

터미널은 일반적으로 두 개 이상의 공통 셸을 지원합니다.

- 파워셸
-CMD

현재 명령이 PowerShell 구문을 사용하는 경우 CMD에서 직접 다시 시도하지 마세요. 실행하기 전에 먼저 쉘 유형을 확인하십시오.

### 스크립트 실행

일반적인 관행은 다음과 같습니다.

- 스크립트를 실행하려면 명령을 직접 입력하세요.
- 리소스 관리 또는 관련 포털에서 "터미널에서 실행" 트리거
- 문제 해결 시 최소 명령을 먼저 실행한 후 점차적으로 매개 변수를 추가하십시오.

### 출력 및 오류 보고서를 확인하세요.

- 첫 번째 오류를 먼저 살펴보세요.
- 마지막 줄에만 집중하지 마세요
- 스크립트가 계속해서 출력된다면, 멈췄다고 가정하기보다는 먼저 스크립트가 계속 실행되고 있는지 확인하세요.

## 권장사용순서

1. 먼저 PowerShell을 사용할지, CMD를 사용할지 확인하세요.
2. 현재 작업 디렉터리가 예상한 프로젝트 위치인지 확인합니다.
3. 최소 명령을 먼저 실행하여 환경, 경로, 권한이 정상인지 확인합니다.
4. 명령이 실패하면 먼저 [로그 뷰어](./log-viewer.md)와 터미널 출력을 사용하여 판단합니다.

## FAQ

### 명령을 실행할 수 있지만 결과가 예상과 일치하지 않습니다.

- 먼저 현재 쉘 유형을 확인하십시오.
- 현재 디렉터리, 환경 변수, 스크립트 매개변수를 다시 확인하세요.
- 스크립트가 외부 프로그램에 의존하는 경우 먼저 해당 프로그램이 실제로 사용 가능한지 확인하십시오.

### 터미널에 출력이 없거나, 응답이 없는 것 같습니다.

- 먼저 명령이 아직 입력을 기다리고 있는지 확인하십시오.
- 잘못된 Shell로 전환했는지 다시 확인하세요.
- 긴 작업이라면 첫 번째 결과를 출력할 때까지 시간을 좀 주세요. 즉시 명령을 다시 제출하지 마십시오.

### 스크립트 실행 시 경로 또는 권한 오류가 발생합니다.

- 먼저 경로가 올바른지 확인하세요.
- 스크립트 파일과 종속 파일이 존재하는지 다시 확인하세요.
- PowerShell 스크립트를 사용하는 경우 먼저 실행 전략 및 호출 방법이 올바른지 확인하십시오.

### 대화형 명령이 비정상적으로 작동합니다.

- 새 터미널 세션을 생성하고 다시 시도하세요.
- 대화형 명령 자체에 문제가 있는지 확인하려면 더 간단한 명령을 다시 사용해 보십시오.
- 결과만 보고 싶다면 터미널에서 모든 상호작용을 완료할 필요는 없습니다.
