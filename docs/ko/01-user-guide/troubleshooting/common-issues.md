# FAQ

이 페이지는 코드 분석이 아니라 문제 해결 포털입니다. 문제가 발생하면 먼저 그것이 속한 카테고리를 확인한 다음 해당 페이지로 이동하여 계속해서 범위를 좁혀보세요.

## 먼저 이 순서를 확인해 보겠습니다.

1. 먼저 [로그 뷰어](../interface/log-viewer.md)를 살펴보고 시작 단계, 장치 단계, 프로세스 단계 또는 플러그인 단계에서 오류가 발생하는지 확인하세요.
2. 현재 프로젝트, 디바이스 인스턴스, 템플릿, 윈도우가 자신이 생각하는 객체인지 다시 확인하세요.
3. 경로, 라이센스, 데이터베이스 연결, 서비스 상태 등 외부 조건을 확인합니다.
4. 마지막으로 창을 다시 열어야 하는지, 구성을 다시 저장해야 하는지, 관련 서비스를 다시 시작해야 하는지 고려하세요.

## 일반적인 시나리오

### 메인 창에서 기능을 찾을 수 없습니다

- 먼저 [메인 창 탐색](../interface/main-window.md)으로 돌아갑니다.
- 검색창을 이용해 우선적으로 입구를 찾으세요.
- 플러그인 기능인 경우 플러그인이 제대로 로딩되었는지 확인하세요.

### 장치가 존재하지만 작동하지 않습니다

- 먼저 [장치 서비스 개요](../devices/overview.md)를 살펴보세요.
- 계속해서 카메라 장면을 감상하세요 [카메라 서비스](../devices/camera.md)
- 바인딩이나 하드웨어 접근 문제인 경우 [물리적 카메라 관리](../devices/camera-management.md)를 참고하세요.

### 매개변수가 변경되었으나 효과는 변경되지 않았습니다.

- 먼저 현재 사용 중인 개체를 변경하고 있는지 확인하세요.
- 저장되었는지, 해당 창이나 서비스가 다시 로드되었는지 다시 확인하세요.
- 속성 및 템플릿과 관련된 시나리오의 경우 먼저 [속성 편집기](../interface/property-editor.md) 또는 해당 장치 매개 변수 페이지로 돌아가서 검토하세요.

### 프로세스 실행 실패

- 먼저 [프로세스 실행 및 디버깅](../workflow/execution.md)을 살펴보세요.
- 로그에서 노드 이름, 템플릿 이름, 실패 키워드를 우선적으로 검색합니다.
- 전체 프로세스를 처음부터 의심하지 말고, 먼저 실패한 노드부터 축소하세요.

### JSON 또는 템플릿 콘텐츠 유효성 검사에 실패했습니다.

- 우선 에디터의 체크버튼을 이용해 포맷에 문제가 있는지 확인해주세요.
- 주석 텍스트, 설명 텍스트 또는 이전 템플릿 콘텐츠가 실수로 JSON에 붙여넣어졌는지 다시 확인하세요.
- 다른 편집 모드로 전환하는 경우, 먼저 현재 저장된 JSON으로 돌아가서 판단하시기 바랍니다.

### 로그가 너무 많아서 핵심을 볼 수 없습니다.

- 먼저 로그 수준을 `Warn` 또는 `Error`로 전환하세요.
- 키워드를 사용하여 범위를 좁히세요.
- 검색창이 빨간색으로 변하면 일반 키워드를 먼저 포기하고 일반 키워드로 돌아가세요.

## 아직 찾지 못한 경우

- 문제가 발생한 시간을 기록해 두세요.
- 현재 프로젝트, 장치 이름, 템플릿 이름 또는 플러그인 이름을 기록해 두세요.
- 해당 로그 조각을 첨부하고 문제 해결을 계속하면 효율성이 훨씬 높아집니다.
