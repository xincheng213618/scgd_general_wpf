# 데이터 관리

이 장에서는 세 가지 질문, 즉 데이터가 일반적으로 어디에 있는지, 언제 데이터베이스 페이지에 들어가야 하는지, 언제 가져오고 내보내야 하는지에 대한 답변만 제공합니다.

## 이 장을 읽기에 적절한 시기는 언제입니까?

- 테스트 결과나 비즈니스 데이터가 어디에 저장되어 있는지 확인하고 싶으신가요?
- 데이터베이스 내용을 보거나 정리하거나 정리해야 합니다.
- 일괄 가져오기 또는 내보내기를 수행해야 합니다.
- "데이터가 저장되지 않음", "내보내기 실패", "이력 기록이 보이지 않음" 등의 문제를 해결하고 계시나요?

## 이 장을 읽는 방법

### 먼저 데이터베이스를 살펴보겠습니다

- [데이터베이스 작업](./database.md)

적합한 시나리오:

- 실제로 데이터베이스에 데이터가 저장되어 있는지 확인하고 싶은 경우
- 기록별, 시간별, 조건별 결과를 확인하고 싶은 경우
- 연결, 쓰기 또는 쿼리 문제를 해결하고 싶습니다.

### 가져오기 및 내보내기를 다시 살펴보겠습니다.

- [데이터 내보내기 및 가져오기](./export-import.md)

적합한 시나리오:

- 결과를 외부 시스템에 제출하거나 수동으로 분석해야 함
- 일괄 구성 또는 데이터를 마이그레이션해야 하는 경우
- 현재 지원되는 내보내기 형식과 처리 순서를 확인하고 싶습니다.

## 일반적인 사용 순서

1. 먼저 현재 프로젝트, 장비 또는 프로세스가 올바르게 실행되었는지 확인하십시오.
2. 데이터가 저장되지 않은 것으로 의심되면 먼저 [데이터베이스 작업](./database.md)을 입력하여 기록을 확인하세요.
3. 데이터 존재 여부를 확인한 후 [데이터 내보내기 및 가져오기](./export-import.md)를 입력하여 후속 처리를 진행합니다.
4. 내보낸 내용이 올바르지 않은 경우 내보내기 버튼을 반복적으로 누르지 말고 돌아가서 원본 데이터를 확인하세요.

## FAQ

### 원하는 과거 데이터가 보이지 않습니다

- 먼저 현재 연결된 데이터베이스 인스턴스가 올바른지 확인하세요.
- 필터 조건, 시간 범위, 프로젝트 대상이 올바르게 선택되었는지 다시 확인하세요.
- 프로세스가 방금 실행된 경우 먼저 [로그 뷰어](../interface/log-viewer.md)와 결합하여 실제로 쓰기가 성공했는지 확인합니다.

### 데이터는 존재하지만 내보내기 결과가 올바르지 않습니다.

- 먼저 내보낸 개체가 현재 보고 있는 데이터의 일괄처리인지 확인합니다.
- 내보내기 형식이 후속 사용 시나리오를 충족하는지 다시 확인하세요.
- 필드가 누락된 경우 내보내기 기능이 비정상이라고 직접 판단하기보다는 원본 데이터 페이지로 돌아가서 확인하세요.

### 문제가 프로세스에 있는지 데이터 레이어에 있는지 확실하지 않음

- 먼저 [프로세스 실행 및 디버깅](../workflow/execution.md)을 살펴보세요.
- 그런 다음 [로그 뷰어](../interface/log-viewer.md)를 사용하여 실행, 쓰기 또는 내보내기 단계에서 실패가 발생했는지 확인합니다.
