# 데이터베이스 작업

현재 가장 명확하고 안정적인 데이터베이스 항목은 "데이터베이스 브라우저"입니다. 테이블 수준의 탐색, 검색, 페이징 보기, 조건 충족 시 추가, 수정, 삭제에 적합합니다.

## 이 페이지는 언제 읽어야 할까요?

- 특정 배치의 데이터가 실제로 데이터베이스에 저장되어 있는지 확인하고 싶습니다.
- MySQL이나 SQLite에서 실제 테이블 데이터를 보고 싶은 경우
- 기록을 키워드로 검색해야 합니다.
- 소량의 테이블 데이터 유지 관리가 필요한 경우
- "데이터가 기록되었나요?"와 같은 문제를 해결하고 있습니까? 그리고 "왜 목록에서 볼 수 없나요?"

## 현재 실제입장

데이터베이스 브라우저에는 현재 명시적인 메뉴 항목이 있습니다.

- 도구 → 데이터베이스 브라우저

현재 구현에서는 SQL 편집기가 아니라 테이블 탐색 및 기본 유지 관리에 중점을 둔 창입니다.

## 데이터베이스 브라우저에서는 무엇을 할 수 있나요?

### 데이터 소스, 라이브러리 및 테이블 찾아보기

왼쪽의 트리는 먼저 데이터 소스를 표시한 후 라이브러리와 테이블을 입력합니다. 현재 기본적으로 등록 및 표시할 수 있는 소스는 다음과 같습니다.

-MySQL
- SQLite 로깅 라이브러리

### 테이블별로 데이터 보기

테이블을 선택하면 오른쪽에 다음이 표시됩니다.

- 표 제목 및 출처
- 컬럼 정보
- 데이터 그리드
- 검색창
- 페이지 매김 제어
- 상태 표시줄

### 검색 및 페이징

현재 창에서는 테이블 데이터가 많을 때 키워드 검색과 페이지 탐색을 지원합니다. "분명히 데이터가 있지만 보이지 않습니다"라는 메시지가 나타나면 직접 작성되지 않았다고 판단하기보다는 먼저 검색 조건과 페이지 번호를 확인하십시오.

### 기본 쓰기 작업

현재 데이터 소스와 테이블 모두 쓰기를 허용하고 테이블에 기본 키가 있는 경우 인터페이스는 다음 작업을 제공합니다.

- 신규
- 저장
- 취소
- 삭제
- 새로 고침

테이블에 기본 키가 없거나 현재 데이터 소스에서 쓰기가 허용되지 않는 경우 모든 편집 버튼을 사용할 수 있을 것이라고 기대하지 마세요.

## 일반적인 사용 순서

1. 먼저 도구 메뉴에서 데이터베이스 브라우저를 엽니다.
2. 왼쪽에서 올바른 데이터 소스, 라이브러리 및 테이블을 확장합니다.
3. 먼저 테이블 이름, 소스, 레코드 개수를 확인하여 잘못된 데이터베이스에 입력되지 않았는지 확인하세요.
4. 데이터가 너무 많으면 검색을 먼저 한 후 페이지를 넘기세요.
5. 유지보수가 필요한 경우 현재 테이블에서 쓰기가 가능한지 확인합니다.

## 사용 시 특별히 주의해야 할 점은 무엇인가요?

### 현재 창은 범용 쿼리 플랫폼이 아닌 테이블 브라우저입니다.

테이블 보기, 레코드 필터링 및 간단한 유지 관리에 적합하지만 현재 문서에서는 더 이상 통합 SQL 콘솔로 설명하지 않습니다. 모듈에 특별한 쿼리 창이 있는 경우 해당 모듈 자체의 입구를 먼저 사용해야 합니다.

### 기본 키가 없는 테이블은 임시 수정에 적합하지 않습니다.

현재 구현에서는 기본 키를 사용하여 수정 및 삭제의 신뢰성을 결정합니다. 기본 키가 없는 테이블은 읽기 전용 탐색이나 신중한 새 추가에 더 적합합니다.

### 쓰기 기능은 데이터 소스와 테이블에 따라 다릅니다.

데이터베이스 브라우저를 열었다고 해서 모든 테이블을 쓸 수 있는 것은 아닙니다. 버튼을 사용할 수 없는 경우 먼저 현재 테이블이 쓰기 가능하지 않은지, 아니면 데이터 소스 자체가 읽기 전용인지 확인하세요.

## FAQ

### 보고 싶은 데이터 테이블을 찾을 수 없습니다

- 먼저 현재 확장된 데이터 소스와 라이브러리가 올바른지 확인하세요.
- 데이터 소스 트리를 다시 새로 고칩니다.
- MySQL인 경우 연결 구성 및 데이터베이스 이름이 올바른지 확인

### 테이블을 열 수 있지만 데이터가 비어 있습니다.

- 현재 검색 조건이 필터링 되었는지 먼저 확인하세요.
- 페이지 번호와 페이지 크기를 다시 확인하세요.
- 아직 비어 있는 경우 [프로세스 실행 및 디버깅](../workflow/execution.md) 또는 [로그 뷰어](../interface/log-viewer.md)로 돌아가서 실제로 작성이 성공했는지 확인하세요.

### 버튼이 회색으로 표시되어 편집할 수 없습니다.

- 우선 현재 테이블이 쓰기를 허용하는지 확인하세요.
- 테이블에 기본 키가 있는지 다시 확인
- 기본 키가 없는 경우 "편집 불가능"을 데이터베이스 예외로 오인하지 마십시오.

### 수정 후 결과가 올바르지 않습니다.

- 먼저 저장이 성공했는지 확인하세요.
- 현재 테이블을 새로 고치고 다시 봅니다.
- 주요 비즈니스 데이터의 경우 읽기 경로가 올바른지 먼저 확인한 후 수동으로 수정하세요.
