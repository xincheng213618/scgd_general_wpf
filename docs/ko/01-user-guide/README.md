## 추천 독서 경로

1. 어떤 페이지를 볼지 모르겠다면 [사용자 작업 워크플로 매트릭스](./operation-workflow-matrix.md)부터 시작합니다.
2. 현장 납품 또는 재테스트 때는 [현장 작업 검수 체크리스트](./field-operation-acceptance.md)에 따라 확인합니다.
3. 먼저 [메인 창 탐색](./interface/main-window.md)을 읽고 메인 인터페이스의 레이아웃을 이해하세요.
4. 다음으로 [UI 구성요소 사용 설명서](./interface/ui-component-handbook.md)에서 각 창/구성요소의 입구, 완료 기준, 점검 방향을 확인합니다.
5. [속성 편집기](./interface/property-editor.md) 및 [이미지 편집기 개요](./image-editor/overview.md)를 살펴보고 기본 작업 경로를 설정합니다.
6. 하드웨어 관련인 경우 [장치 서비스 개요](./devices/overview.md) 및 해당 장치 페이지로 이동합니다.
7. 자동화가 필요한 경우 [워크플로 개요](./workflow/README.md)로 이동합니다.
8. 예외가 발생하는 경우 먼저 [FAQ](./troubleshooting/common-issues.md)를 확인하세요.

## 장 경계

- 부분 구현 및 확장 메커니즘의 내용이 [개발자 가이드](../02-developer-guide/README.md)로 이동되었습니다.
- 부분 클래스 라이브러리, 인터페이스, 모듈 수준 설명의 내용이 [API 참조](../04-api-reference/README.md)로 이동되었습니다.
- 시스템 설계 전반에 대한 이해가 필요한 경우에는 [아키텍처 설계](../03-architecture/README.md)로 바로 이동하세요.
