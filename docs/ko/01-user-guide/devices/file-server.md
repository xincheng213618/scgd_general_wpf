# 파일 서비스

파일 서비스는 원격 파일 목록, 파일 다운로드 및 파일 업로드를 통합 장치 시스템에 통합하는 데 사용됩니다. "결과 파일이 서버 측에 있다", "이미지를 장치 측에서 가져와야 한다", "파일을 원격 끝으로 보내야 한다"와 같은 시나리오에 사용하기에 적합합니다.

## 이 페이지는 언제 읽어야 할까요?

- 파일 서비스를 추가하거나 구성하려는 경우
- 원격지에서 사용 가능한 파일 목록을 보고 싶습니다.
- 소스 파일을 현재 시스템에 다운로드하고 싶습니다.
- 원격 서비스에 파일을 업로드하고 싶습니다.
- '파일은 보이지만 열 수 없습니다', '업로드 후 응답이 없습니다' 등의 문제를 해결하고 계시나요?

## 평소 이 서비스를 이용하는 방법

### 먼저 구성을 완료하세요.

먼저 장치 코드의 기본 구성, 송신 주제 및 구독 주제가 올바른지 확인하십시오. 파일 서비스와 관련된 많은 문제는 "파일 작업 실패"처럼 보이지만 실제로는 연결 또는 테마 구성 초기에 중단됩니다.

### 파일 목록을 다시 살펴보세요

현재 원격지에 어떤 파일이 있는지 모른다면 먼저 파일 목록을 가져온 후 다운로드할지 업로드할지 결정하세요. 파일 이름을 직접 추측하지 마세요.

### 콘텐츠가 필요할 때 열거나 다운로드하세요.

현재 구현의 일반적인 파일 작업은 다음과 같습니다.

- 모든 파일 목록 가져오기
- 지정된 파일을 열거나 다운로드합니다.
- 지정된 파일 업로드

## 일반적인 사용 순서

1. 먼저 장치 목록에 파일 서비스를 생성하고 저장합니다.
2. 연결 관련 구성이 올바른지 확인하세요.
3. 먼저 파일 목록을 가져와 실제로 원격 끝에 대상 파일이 있는지 확인합니다.
4. 다운로드 또는 열기 작업을 다시 수행합니다.
5. 파일을 다시 업로드해야 하는 경우 업로드를 다시 수행하세요.

## 사용 시 주의할 점은 무엇인가요?

### 먼저 파일 이름을 확인해야 합니다.

파일 서비스는 파일 이름을 기반으로 작업을 시작합니다. 파일명이 틀리면 나중에 표시창을 보면 별 의미가 없습니다.

### 목록과 콘텐츠는 두 단계로 구성됩니다.

목록을 볼 수 있다고 해서 특정 파일을 정상적으로 열 수 있다는 의미는 아닙니다. 마찬가지로 업로드가 성공했다고 해서 올바른 디렉토리나 올바른 유형에서 본 것을 의미하는 것은 아닙니다.

### 테마 및 장치 코드 문제를 먼저 해결하세요.

현재 구현에는 장치 코드와 구독 주제가 명시적으로 일치합니다. 응답이 없는 경우 먼저 이러한 기본 조건을 확인하십시오.

## FAQ

### 파일 목록을 되돌릴 수 없습니다.

- 먼저 보내는 주제와 구독하는 주제가 맞는지 확인하세요.
- 현재 장치 코드가 반환된 메시지와 일치하는지 다시 확인하세요.
- 서버에 하트비트는 있지만 파일 이벤트가 없는 경우 먼저 특정 파일 동작 구성을 확인하세요.

### 파일을 나열할 수는 있지만 열 수는 없습니다.

- 먼저 파일명이 맞는지 확인하세요.
- 요청한 파일 형식이 대상 파일 형식인지 다시 확인하세요.
- 문제가 지속되면 [로그 뷰어](../interface/log-viewer.md)를 사용하여 반환 메시지에 더 명확한 오류 메시지가 있는지 확인하세요.

### 업로드 후 파일을 볼 수 없습니다

- 먼저 업로드 작업이 실제로 전송되고 완료되었는지 확인하세요.
- 이전 목록에 의존하는 대신 파일 목록을 다시 가져옵니다.
- 원격 측에 디렉토리 또는 유형 구별이 있는 경우 올바른 범위를 보고 있는지 확인하십시오.

### 진행중인 파일 서비스가 불안정합니다

- 먼저 파일 목록, 다운로드 또는 업로드가 성공했는지 수동으로 확인하세요.
- [프로세스 실행 및 디버깅](../workflow/execution.md)으로 돌아가 해당 프로세스에서 동일한 파일 서비스가 사용되는지 확인합니다.
- 필요한 경우 장치 코드와 테마 구성을 모두 확인하여 수동 테스트와 일치하는지 확인하십시오.
