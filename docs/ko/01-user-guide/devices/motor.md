# 모터 서비스

모터 서비스는 모터 장비를 ColorVision의 통합 장치 시스템에 통합하는 데 사용되며 열림, 닫힘, 이동, 원점 복귀, 현재 위치 읽기 등 일반적인 제어 동작을 제공합니다. 수동 디버깅과 자동화 프로세스 모두에서 위치 제어가 필요한 시나리오에 사용하기에 적합합니다.

## 이 페이지는 언제 읽어야 할까요?

- 모터 서비스를 추가하거나 구성하려는 경우
- 모터를 지정된 위치로 수동으로 이동시켜야 합니다.
- 현재 위치가 업데이트 되었는지 확인이 필요합니다.
- "모터는 보이지만 움직이지 않는다", "위치가 새로 고쳐지지 않는다"는 문제를 해결해야 합니다.

## 일반적인 사용 순서

1. 먼저 장치 목록에 모터 서비스를 생성하고 저장합니다.
2. 통신 구성 및 시간 초과 매개변수가 적절한지 확인하십시오.
3. 먼저 열기 작업을 수행하여 장치가 작동 가능한 상태인지 확인하세요.
4. 그런 다음 이동, 원점 복귀, 위치 읽기 등의 동작을 수행합니다.
5. 완료 후 업무상 장치 종료가 필요한 경우 종료 작업을 다시 수행합니다.

## 이 서비스의 일반적인 작업

### 열고 닫기

이후의 많은 작업에서는 장치가 이미 작동 중이라고 가정합니다. "명령을 보냈으나 응답이 없습니다"라는 메시지가 나타나면 먼저 이동 명령을 직접 의심하기보다는 성공적으로 열렸는지 확인하십시오.

### 지정된 위치로 이동

다음 매개변수에 주의하는 것이 일반적입니다.

- 대상 위치
- 절대 또는 상대 이동
- 타임아웃 기간

디버깅 단계인 경우 먼저 작은 범위의 이동을 사용하여 방향과 동작을 확인한 다음 큰 단계 크기 조정을 수행합니다.

### 원점복귀

현재 위치를 신뢰할 수 없거나 통일된 출발점을 재설정해야 하는 경우 후속 작업을 계속하기 전에 원점으로 돌아가는 것이 우선입니다.

### 현재 위치 가져오기

현재 위치는 추측할 수 없으므로 주도적으로 다시 읽고 확인해야 합니다. 특히 자동화된 프로세스나 여러 번의 수동 이동 후에는 위치를 먼저 읽고 다음 단계를 수행하는 것이 더 안정적인 경우가 많습니다.

## FAQ

### 모터 서비스가 존재하지만 열 수 없습니다.

- 먼저 통신 구성이 올바른지 확인하십시오.
- 기기가 온라인 상태인지 다시 확인하세요.
- 열기 작업이 실패하면 후속 이동 및 읽기 위치는 일반적으로 신뢰할 수 없습니다.

### 이동 명령을 보냈으나 위치가 바뀌지 않았습니다.

- 먼저 장치가 성공적으로 열렸는지 확인하십시오.
- 목표 위치, 절대/상대 모드 및 타임아웃을 다시 확인하세요.
- 단순히 인터페이스가 새로고침되었는지 확인하지 말고, 작업 후 현재 위치를 적극적으로 읽어보세요.

### 현재 위치가 잘못 표시됩니다

- 위치를 먼저 얻으려면 다시 실행하십시오.
- 마지막 이동이 실패했지만 인터페이스는 여전히 이전 값을 유지하는지 다시 확인합니다.
- 원점복귀를 포함하는 경우에는 원점복귀가 실제로 완료되었는지 먼저 확인하시기 바랍니다.

### 프로세스에서 모터 동작을 호출하지 못했습니다.

- 먼저 모터가 열리고 정상적으로 움직일 수 있는지 수동으로 확인하십시오.
- 프로세스의 매개 변수가 수동 테스트와 일치하는지 다시 확인하십시오.
- 필요한 경우 [프로세스 실행 및 디버깅](../workflow/execution.md) 및 [로그 뷰어](../interface/log-viewer.md) 보기 링크
