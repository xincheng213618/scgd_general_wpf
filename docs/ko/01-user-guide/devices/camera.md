#카메라서비스

이 페이지에서는 일상적인 사용에서 "카메라 서비스" 계층의 역할과 물리적 카메라, 매개변수 구성 및 보정 템플릿과의 관계를 설명합니다.

## 카메라 서비스와 실제 카메라는 같은 것이 아닙니다

- 물리적 카메라: 실제 하드웨어, 라이센스, 리소스 및 기본 기능
- 카메라 서비스: 프로젝트 내 장치 목록, 보기 창 및 프로세스에서 실제로 호출되는 논리적 장치
- 카메라 서비스는 일반적으로 이미지 캡처, 비디오, 보정 및 템플릿 기능을 안정적으로 사용하기 전에 물리적 카메라에 바인딩되어야 합니다.

## 여기서 보통 무엇을 하시나요?

- 카메라 서비스 구성 편집
- 카메라 보기 또는 라이브 피드 열기
- 물리적 카메라 바인딩 또는 전환
- 자동 노출, 자동 초점 및 카메라 매개변수 템플릿 입력
- 교정 매개변수 편집 창을 엽니다.
- 로컬 디버깅 포털 및 카메라 로그 보기

## 권장사용순서

1. 먼저 [실제 카메라 관리](./camera-management.md)에서 하드웨어, 라이선스, 리소스가 사용 가능한지 확인하세요.
2. 카메라 서비스로 돌아가서 현재 서비스 인스턴스를 올바른 물리적 카메라에 바인딩합니다.
3. [카메라 매개변수 구성](./camera-configuration.md)에서 노출, 게인, 채널 및 모드를 조정합니다.
4. 필요한 경우 자동 노출, 자동 초점 또는 보정 템플릿을 입력합니다.
5. 실시간 화면, 로컬 창, 프로세스를 이용하여 검증 결과를 실행합니다.

## 이 페이지에 해당하는 능력 범위

현재 코드에서 카메라 서비스 레이어에는 이미 몇 가지 공통 입구가 포함되어 있습니다.

- 카메라 구성 편집
- 장치 ID 새로 고침 및 물리적 카메라 바인딩
- 자동 노출 템플릿
- 자동 초점 템플릿
- 카메라 매개변수 템플릿
- 교정 템플릿
- 카메라 로그 및 로컬 창

따라서 "시스템에서 카메라를 정상적으로 사용할 수 있는지 여부"라는 문제가 발생하면 일반적으로 기본 카메라 리소스를 직접 보는 대신 카메라 서비스를 먼저 살펴봅니다.

## FAQ

### 카메라 서비스가 존재하지만 열 수 없거나 이미지를 캡처할 수 없습니다.

- 먼저 물리적 카메라가 바인딩되었는지 확인하세요.
- 라이선스, 연결 상태, 카메라 로그를 다시 확인하세요.
- 방금 구성을 변경한 경우 해당 창을 다시 열거나 서비스를 다시 시작한 후 다시 시도해 보세요.

### 물리적 카메라 구성을 변경한 후 카메라 서비스가 동기화되지 않습니다.

- 저장 후 카메라 서비스 구성을 다시 입력하세요.
- 현재 서비스가 예상되는 물리적 카메라에 바인딩되어 있는지 확인하세요.
- 장치 ID를 새로 고치거나 필요한 경우 다시 바인딩하십시오.

### 비디오 또는 로컬 모드 예외

- 현재 영상이나 실시간 모드를 먼저 종료한 후 다시 연결하거나 다시 시작하세요.
- 연결 문제인지, 파라미터 문제인지 우선순위를 정하세요.
- 필요 시 카메라 로그를 확인하여 백그라운드 서비스 상태를 확인하세요.
