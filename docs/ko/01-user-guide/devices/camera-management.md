#물리적 카메라 관리

이 페이지에서는 물리적 카메라 계층이 담당하는 작업을 설명합니다. 이는 카메라 서비스를 직접 교체하는 것이 아니라 실제 카메라를 검색, 승인 및 유지 관리하는 "하드웨어 리소스 원장"에 가깝습니다.

## 언제 먼저 와야하나요?

- 새 카메라를 처음 연결하는 경우
- 라이센스 가져오기 또는 업데이트
- 생성 가능한 카메라 ID 새로고침
- 교정 리소스, 그룹화 및 물리적 구성 관리
- 논리 카메라 서비스가 바인딩 가능한 개체를 찾을 수 없습니다.

## 물리적 카메라 레이어는 무엇을 담당하나요?

- 실제 카메라 객체 생성
- `.lic` 또는 `.zip` 라이센스 가져오기
- 물리적 카메라 구성 및 리소스 트리 유지
- 카메라 서비스를 위한 바인딩 가능한 카메라 코드 및 기본 매개변수 제공

즉, "하드웨어가 실제로 시스템에서 인식되는지 여부", "라이센스가 유효한지 여부"와 같은 문제는 일반적으로 이 수준에서 먼저 확인해야 한다는 의미입니다.

## 일반적인 작업 순서

1. 장치를 연결하고 시스템이 유휴 카메라 ID를 볼 수 있는지 확인합니다.
2. 물리적 카메라 관리에서 카메라를 생성하거나 라이선스를 가져옵니다.
3. 카메라 이름, 코드, 모드, 기본 구성을 확인하세요.
4. 필요한 경우 교정, 그룹화 및 관련 리소스를 준비합니다.
5. [카메라 서비스](./camera.md)로 돌아가서 논리적 서비스를 이 물리적 카메라에 바인딩합니다.

## 구성 시 확인해야 할 사항은 무엇인가요?

- 라이센스가 유효한가요?
- 정말 생성되지 않은 카메라 ID가 있는 걸까요?
- 물리적 카메라 자원의 완성 여부
- 카메라 모드가 실제 하드웨어와 일치하나요?
- 수정 후 다시 저장하거나 새로고침하려면 논리카메라 서비스로 돌아가야 하나요?

## FAQ

### 생성되지 않은 카메라를 찾을 수 없습니다.

- 하드웨어가 연결되어 있고 시스템에서 인식되는지 확인하세요.
- 먼저 장치 ID를 새로 고치고 다시 시도하십시오.
- 방금 장치를 연결했다가 분리했다면 창을 다시 열고 다시 확인하세요.

### 라이센스를 가져온 후에도 여전히 사용할 수 없습니다.

- 가져온 파일 형식이 올바른지 확인하세요.
- 라이센스에 해당하는 카메라 코드가 현재 하드웨어와 일치하는지 확인
- 데이터베이스 또는 로컬 리소스가 동기화되었는지 확인하세요.

### 물리적 카메라 구성이 변경되어 기존 카메라 서비스가 비정상적으로 동작합니다.

- 논리 서비스를 다시 저장하거나 다시 바인딩해야 하는지 확인하세요.
- 구성 변경 후 관련 서비스를 다시 시작해야 하는지 주의하세요.
- 경로 및 리소스 조정이 포함된 경우 관련 파일이 새 위치로 이동되었는지 확인합니다.
