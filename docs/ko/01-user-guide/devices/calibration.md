# 교정 서비스

교정 서비스는 교정 프로세스를 실제 카메라에 바인딩하고, 교정 템플릿을 관리하고, 교정 데이터 생성을 트리거하고, 필요할 때 원시 파일 및 캐시를 처리하는 데 사용됩니다. 단순한 "매개변수 페이지"가 ​​아니라 물리적 카메라, 템플릿 편집 및 교정 결과를 연결하는 서비스 계층입니다.

## 이 페이지는 언제 읽어야 할까요?

- 교정 서비스를 생성하거나 구성하려는 경우
- 보정 서비스를 실제 카메라에 바인딩해야 합니다.
- 교정 템플릿을 편집하고 싶습니다.
- 교정 관련 데이터 수집을 트리거하고 싶습니다.
- "왜 교정을 시작할 수 없나요?"와 같은 문제를 해결하고 있습니까? "왜 템플릿을 열 수 없나요?"

## 사용 전 꼭 확인하세요.

### 실제 카메라를 먼저 바인딩합니다.

현재 구현에서 교정 서비스는 실제 카메라 코드에 직접적으로 의존합니다. 해당 실제 카메라를 먼저 구성하지 않으면 많은 보정 관련 작업이 전혀 계속되지 않습니다.

### 데이터베이스를 다시 사용할 수 있는지 확인

현재 환경에서 MySQL이 활성화되어 있으나 데이터베이스가 연결되지 않은 경우에는 캘리브레이션 템플릿 편집이 정상적으로 입력되지 않습니다. 이 경우 먼저 데이터베이스 연결을 수정하고 템플릿 창을 반복적으로 클릭하지 마십시오.

## 일반적인 사용 순서

1. 먼저 장치 목록에서 교정 서비스를 생성하거나 확인합니다.
2. 올바른 실제 카메라에 바인딩합니다.
3. 보정 템플릿 편집 포털을 열고 템플릿 내용이 현재 카메라와 일치하는지 확인합니다.
4. 교정 데이터를 얻어야 할 경우 관련 교정 조치를 수행하십시오.
5. 실제 카메라 구성이 변경되면 돌아가서 보정 서비스가 여전히 올바르게 바인딩되어 있는지 확인해야 합니다.

## 이 서비스에서 가장 일반적인 작업

### 실제 카메라 관리

현재 교정 서비스에 해당하는 카메라가 아직 준비되지 않은 경우 문제 해결을 위해 교정 창에만 머무르지 말고 실제 카메라 관리 페이지로 이동하여 먼저 처리하십시오.

계속해서 읽을 수 있습니다:

- [카메라 관리](./camera-management.md)
- [카메라 파라미터 설정](./camera-configuration.md)

### 보정 템플릿 편집

보정 템플릿 편집은 현재 바인딩된 물리적 카메라에 따라 다릅니다. 템플릿 창을 정상적으로 열 수 있는지 여부는 그 자체로 중요한 상태 확인입니다.

- 열리지 않는 경우 카메라 바인딩 및 데이터베이스 연결을 먼저 확인합니다.
- 열 수는 있으나 내용이 잘못된 경우 현재 카메라 및 템플릿 개체가 올바른지 확인하는 것이 우선적으로 적용됩니다.

### 원본 파일 및 캐시 처리

교정 서비스에는 원시 파일 목록, 파일 다운로드 및 캐시 정리도 포함됩니다. "현재 교정을 방해하는 오래된 데이터"와 같은 문제가 발생하면 전체 서비스를 사용할 수 없다고 직접 의심하는 대신 캐시 및 원본 파일 범위부터 시작하십시오.

## FAQ

### 교정 템플릿 편집을 클릭했지만 편집기에 들어가지 않았습니다.

- 먼저 해당 물리적 카메라가 바인딩되어 있는지 확인하세요.
- MySQL이 활성화된 경우 데이터베이스 연결이 정상인지 확인
- 현재 올바른 교정 서비스 인스턴스가 열려 있는지 재확인하세요.

### 카메라를 교체했는데도 캘리브레이션 결과가 여전히 낡아 보입니다.

- 먼저 캘리브레이션 서비스에 바인딩된 'CameraCode'가 업데이트되었는지 확인하세요.
- 물리적 카메라 전환 후 현재 구성이 다시 저장되었는지 다시 확인하세요.
- 필요한 경우 현재 템플릿 개체를 확인하기 위해 템플릿 편집기를 다시 입력하세요.

### 보정 과정에서 얻은 원본 파일이 올바르지 않습니다.

- 먼저 현재 장치 코드와 장치 유형이 대상 장치인지 확인하십시오.
- 이전 캐시가 아닌 원본 파일 목록이 요청되었는지 다시 확인하세요.
- 캐시 간섭이 의심되는 경우 먼저 캐시를 정리한 후 다시 확인하세요.

### 교정 서비스가 존재하지만 프로세스에서는 적용되지 않습니다.

- 바인딩된 카메라와 템플릿이 유효한지 먼저 확인하세요.
- 다시 연결된 검사 [워크플로 개요](../workflow/README.md) 및 [프로세스 실행 및 디버깅](../workflow/execution.md)
- 동시에 [로그 뷰어](../interface/log-viewer.md)와 결합하여 구체적인 실패 단계를 확인합니다.
