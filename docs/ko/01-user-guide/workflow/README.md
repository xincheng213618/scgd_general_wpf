# 작업 흐름

이 장은 매일 사용하기 위한 것입니다. FlowEngine 아키텍처 자체를 설명하지는 않지만 프로세스를 설계할 시기와 실행 및 디버그할 시기를 결정하는 데 도움이 됩니다.

## 워크플로는 어떤 문제를 해결하는 데 적합합니까?

ColorVision에서 워크플로는 일반적으로 장치 작업, 템플릿 처리 및 결과 출력을 반복 가능한 비즈니스 링크로 묶는 데 사용됩니다.

일반적인 시나리오는 다음과 같습니다.

- 고정된 단계로 여러 장치 또는 서비스 호출
- 일련의 탐지 또는 처리 시퀀스를 재사용합니다.
- 실행 실패 시 어떤 노드가 멈춰 있는지 빠르게 찾아보세요.
- 수동 작업을 저장 가능하고 반복 가능한 프로세스로 최대한 융합합니다.

## 이 장을 읽는 방법

### 과정을 수강하고 싶을 때

- [프로세스 디자인](./design.md)

적합한 시나리오:

- 새로운 프로세스 생성
- 노드 순서, 연결 관계 또는 매개변수 조정
- 특정 단계를 어디에 배치해야 하는지 확인하고 싶습니다.

### 프로세스를 실행하거나 문제를 해결하고 싶을 때

- [프로세스 실행 및 디버깅](./execution.md)

적합한 시나리오:

- 기존 프로세스를 수동으로 실행
- 실행현황 및 결과 관찰
- 첫 번째 실패한 노드를 찾습니다.

## 권장사용순서

1. 먼저 장비, 템플릿 또는 데이터 처리 단계를 함께 연결할지 여부를 명확히 하십시오.
2. [Process Design](./design.md)을 입력하여 최소 실행 가능 버전을 빌드합니다.
3. [프로세스 실행 및 디버깅](./execution.md)으로 이동하여 각 단계가 실제로 예상대로 실행되는지 확인합니다.
4. 프로세스가 장치 서비스와 관련된 경우 프로세스 창에서 다시 시도하는 대신 돌아가서 해당 장치 페이지를 확인하십시오.

## FAQ

### 프로세스를 열 수는 있지만 실행할 수는 없습니다.

- 먼저 동시에 많은 노드를 변경하지 마십시오
- 먼저 실패한 첫 번째 노드를 찾습니다.
- 그런 다음 노드가 의존하는 장치, 템플릿 또는 입력 데이터를 확인하십시오.

### 프로세스 실행 결과가 기대와 일치하지 않습니다.

- 먼저 현재 올바른 프로세스 버전이 실행되고 있는지 확인하세요.
- 노드 매개변수와 입력 객체가 현재 프로젝트와 일치하는지 다시 확인하세요.
- 결과가 라이브러리에 드랍되거나 비정상적으로 내보내진 경우 [데이터 관리](../data-management/README.md)에 링크하세요.

### 문제가 공정에 있는지 장비에 있는지 잘 모르겠습니다.

- 먼저 [프로세스 실행 및 디버깅](./execution.md)을 살펴보세요.
- 서비스 상태 및 구성을 확인하려면 해당 장치 페이지로 돌아갑니다.
- [로그 뷰어](../interface/log-viewer.md)와 결합하여 범위를 좁힐 수도 있습니다.
