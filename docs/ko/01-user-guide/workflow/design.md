# 프로세스 설계

프로세스 디자인 페이지의 초점은 "비주얼 편집기란 무엇인가"를 추상적으로 소개하는 것이 아니라 프로세스를 구축하고 저장하고 변경 후 빠르게 실행 가능한 상태로 되돌릴 수 있도록 돕는 것입니다.

## 현재 디자이너에서 직접 할 수 있는 작업은 무엇인가요?

현재 구현에서 프로세스 디자이너는 최소한 다음과 같은 일반적인 작업을 지원합니다.

- 새로운 프로세스 생성
- 기존 프로세스 템플릿 열기
- 저장 과정
- 프로세스 삭제
- 수입과정
- 수출과정
- 모듈 가져오기
- 자동 레이아웃
- 캔버스에 자동으로 적응
- 실행 취소 및 다시 실행
- 속성 패널을 통해 노드 관련 콘텐츠 편집

## 일반적인 디자인 순서

1. 먼저 프로세스를 생성하거나 엽니다.
2. 필요한 노드를 캔버스에 배치합니다.
3. 노드 관계를 연결하고 속성 패널에서 주요 매개변수를 완성합니다.
4. 자동 레이아웃이나 위치 미세 조정을 하기 전에 한번 저장해 보세요.
5. 주요 변경 후 다시 저장하고 [프로세스 실행 및 디버깅](./execution.md)에 들어가 확인합니다.

## 디자인에서 가장 일반적으로 사용되는 작업 중 일부

### 새로 만들기, 열기 및 저장

프로세스 디자인에서 가장 흔한 문제는 "노드를 드래그할 수 없는 것"이 아니라 오랜 시간 동안 변경한 후 올바른 템플릿을 저장하지 못하는 것입니다. 디자인할 때 현재 편집 중인 프로세스를 먼저 확인한 후 저장하세요.

### 자동 레이아웃 및 자동 적응

노드가 점점 많아지고 캔버스가 점점 더 지저분해지면 먼저 자동 레이아웃과 자동 적응을 사용하여 구조를 읽을 수 있는 상태로 되돌린 다음 계속해서 매개변수를 조정하세요. 이는 일반적으로 수동 끌기보다 훨씬 안정적입니다.

### 실행 취소 및 다시 실행

현재 구현에는 명시적인 실행 취소 및 다시 실행 스택이 있습니다. 큰 변화를 가할 때 먼저 시도하는 것을 두려워하지 마십시오. 그러나 시도해 보고 방향이 틀렸다고 판단되면 잘못된 상태에서 계속해서 만회하기보다는 가능한 한 빨리 철회해야 합니다.

### 가져오기, 내보내기 및 모듈 재사용

처음부터 프로세스를 구축하지 않는 경우 먼저 재사용 가능한 프로세스나 모듈이 있는지 확인하세요. 미리 만들어진 콘텐츠를 가져온 다음 약간만 변경하는 것이 일반적으로 완전히 수동으로 재구성하는 것보다 빠르며 매개변수를 놓칠 가능성이 적습니다.

## 디자인할 때 특히 주의해야 할 점은 무엇인가요?

### 캔버스의 콘텐츠가 저장된 콘텐츠와 동일하지 않습니다.

저장되지 않은 경우 디자이너에 표시되는 콘텐츠는 후속 실행에서 실제로 실행될 버전이 아닐 수도 있습니다. 실행하기 전에 반드시 저장하세요.

### 선택한 프로세스 템플릿은 편집 대상과 일치해야 합니다.

현재 구현에서는 현재 선택된 프로세스 템플릿을 유지합니다. 선택한 개체가 생각한 것과 일치하지 않으면 이후 저장, 삭제, 내보내기가 잘못된 개체로 종료됩니다.

### 매개변수는 속성 패널에서 먼저 확인됩니다.

노드가 연결된 후 매개변수가 정확하다고 직접 가정하지 마십시오. 많은 문제는 프로세스 구조가 잘못된 것이 아니라 노드 매개변수가 여전히 이전 템플릿 값을 사용하고 있다는 것입니다.

## FAQ

### 프로세스를 열 수 있지만 캔버스가 비어 있습니다.

- 먼저 현재 선택한 템플릿이 올바른지 확인하세요.
- 템플릿에 실제로 프로세스 데이터가 있는지 다시 확인하세요.
- 템플릿 자체에 콘텐츠가 없는 경우 해당 프로세스를 생성한 후 진행하세요.

### 프로세스는 변경됐지만 실행 결과는 그대로인 것 같습니다.

- 먼저 저장되었는지 확인하세요.
- 실행 페이지에서 동일한 프로세스 템플릿이 선택되었는지 다시 확인하세요.
- 모듈이나 프로세스를 import한 경우 잘못된 객체가 수정되었는지 확인한다.

### 캔버스가 너무 지저분하고 관계가 명확하게 보이지 않습니다.

- 자동 레이아웃이나 자동 적응을 먼저 사용하세요.
-키 노드 이름 바꾸기 또는 재구성
- 구조를 명확하게 보지 않고 문제 해결에 뛰어들지 마십시오.

### 문제가 설계에 있는지 실행에 있는지 확실하지 않습니다.

- 아직 노드 관계 및 매개변수를 확인하지 않았다면 디자인 페이지에 머물면서 정리하세요.
- 구조가 안정적이라면 [프로세스 실행 및 디버깅](./execution.md)으로 이동하여 런타임 문제를 찾아보세요.
