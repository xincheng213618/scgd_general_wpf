# 플러그인 개발 개요

이 페이지에서는 현재 웨어하우스에서 실제로 사용할 수 있는 플러그인 개발 모델만 설명하고, 기존의 일반화된 인터페이스 예제를 계속 사용하지 않습니다.

## 현재 플러그인 모델

ColorVision의 플러그인은 기본 프로그램 실행 디렉토리 아래 `Plugins/`의 별도 디렉토리에 배포됩니다. 기본 프로그램이 시작되면 각 플러그인 디렉터리를 검색하고 `manifest.json`을 읽은 다음 매니페스트에 지정된 DLL에 따라 어셈블리를 로드합니다.

현재 코드에서 직접 확인할 수 있는 몇 가지 핵심 사항은 다음과 같습니다.

- 플러그인의 기본 인터페이스는 `UI/ColorVision.Common/Interfaces/IPlugin.cs`에 있습니다.
- 플러그인 매니페스트는 `UI/ColorVision.UI/Plugins/PluginManifest.cs`에 있습니다.
- 플러그인 로딩 로직은 `UI/ColorVision.UI/Plugins/PluginLoader.cs`에 있습니다.

## 주요 구성 요소

### 1. 플러그인 인터페이스

현재 저장소에 표시되는 기본 인터페이스는 매우 가볍습니다.

```csharp
공개 인터페이스 IPlugin
{
    문자열 헤더 { get; }
    문자열 설명 { get; }
    무효 실행();
}
```
간단한 플러그인 항목만 만드는 경우 일반적으로 'IPluginBase'로 시작하는 것이 더 편리합니다.

### 2. 매니페스트.json

플러그인 디렉터리는 일반적으로 `manifest.json`을 제공해야 합니다. 현재 매니페스트 객체에는 최소한 다음 필드가 포함되어 있습니다.

- `id`
- `매니페스트_버전`
- `이름`
- `버전`
- '필요하다'
-`설명`
-`dll 경로`
- `저자`
- `url`
- `진입_포인트`
-`아이콘`

가장 핵심적인 것은 플러그인 식별, 설명 및 DLL 경로입니다. 'entry_point'는 항목 유형을 명시적으로 지정해야 할 때 사용됩니다.

### 3. 로딩 과정

메인 프로그램이 시작된 후 `PluginLoader`는 다음을 수행합니다.

1. `Plugins/` 아래의 플러그인 디렉터리를 검색합니다.
2. 각 디렉터리의 `manifest.json`을 읽습니다.
3. 매니페스트를 기반으로 DLL 경로를 계산합니다.
4. 종속성과 버전을 확인합니다.
5. 어셈블리를 로드하고 플러그인 정보를 내부 캐시에 씁니다.

플러그인 디렉터리에 매니페스트가 없는 경우에도 플랫폼은 "디렉터리 이름과 동일한 이름의 DLL"로 이를 로드하려고 시도하지만 이는 더 이상 권장되는 형식이 아닙니다.

## 권장 디렉터리 구조

``텍스트
플러그인/
└── 마이플러그인/
    ├── 매니페스트.json
    ├── MyPlugin.csproj
    ├── MyPlugin.dll
    ├── 읽어보세요.md
    ├── CHANGELOG.md
    ├── 자산/
    └── 소스/ 또는 *.cs/*.xaml
```
## 개발 제안

### 플랫폼 내 개발

- 웨어하우스에 새로운 `Plugins/<PluginId>/` 프로젝트를 생성합니다.
- 빌드 후 메인 프로그램의 출력 디렉터리에 있는 `Plugins/<PluginId>/`에 출력을 복사합니다.
- 기존 표준 플러그인의 디렉토리 및 패키징 방식을 우선시합니다.

### 외부 독립 개발

- 플러그인의 최종 결과물은 직접 복사할 수 있는 완전한 디렉터리로 유지되어야 합니다.
- 플랫폼의 기본 프로그램과 함께 제공되는 `ColorVision.*.dll`을 다시 패키지하지 마십시오.
- 타사 런타임 종속성은 플러그인 제품과 함께 릴리스되어야 합니다.

## 설명

- 이전 문서에 등장하는 'IPluginContext', 비동기 라이프사이클 인터페이스, 독립 플러그인 호스트 모델은 현재 웨어하우스 메인 경로에서 직접 볼 수 있는 기본 인터페이스가 아니다.
- 문서와 코드가 일치하지 않는 경우 `UI/ColorVision.Common/Interfaces/IPlugin.cs`, `UI/ColorVision.UI/Plugins/PluginLoader.cs` 및 기존 플러그인 프로젝트가 우선합니다.
