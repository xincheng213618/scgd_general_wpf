# 배포 개요

이 페이지에는 Windows 데스크톱 애플리케이션, 설치 프로그램 및 업데이트 메커니즘을 다루는 데 중점을 두고 현재 저장소에서 아직 사용 중인 배포 항목만 유지됩니다.

## 현재 배포 객체

- `ColorVision/`: 메인 프로그램 본문
- `ColorVisionSetup/`: 관련 프로그램 설치 및 업데이트
- `Scripts/`: 빌드, 패키징 및 게시를 위한 보조 스크립트
- `Plugins/`: 런타임 시 로드되는 플러그인 디렉터리

## 현재 권장 경로

### 개발 또는 테스트 환경

소스에서 직접 기본 프로그램을 빌드하고 실행합니다.

``파워셸
닷넷 복원
dotnet build -p:플랫폼=x64
dotnet run --project ColorVision/ColorVision.csproj
```
### 전달 환경

- 설치 프로그램을 사용하여 완전한 데스크톱 프로그램 제공
- 필요에 따라 플러그인 디렉터리 및 런타임 종속성을 수행합니다.
- 온라인 업데이트인 경우 [자동 업데이트 시스템](./auto-update.md)을 계속해서 읽어보세요.

## 배포 전 확인사항

- 대상 환경은 Windows
- 메인 앱은 x64용으로 제작되었습니다.
- 런타임 종속성 및 로컬 DLL이 패키지와 함께 올바르게 내보내집니다.
- 필수 구성 파일이 출력 디렉터리에 복사되었습니다.

## 설명

- 이전 Docker, 클라우드 배포, 프로덕션 클러스터 등 지침은 더 이상 기본 배포 경로가 아닙니다.
- 프로젝트에 특별한 전달 방법이 있는 경우 일반 배포 페이지에 계속 쌓여 있지 않고 해당 프로젝트 디렉터리나 프로젝트 문서에 별도로 보관해야 합니다.
